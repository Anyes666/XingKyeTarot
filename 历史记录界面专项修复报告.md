## 3. 历史筛选修复

### 3.1 筛选项确认

五个筛选项完整：全部 / 单牌 / 圣三角 / 关系 / 牌轮

`filterToSpreadType()` 映射逻辑：
- '单牌' → `SpreadType.SINGLE`
- '圣三角' → `SpreadType.TRIANGLE`
- '关系' → `SpreadType.RELATION`
- '牌轮' → `SpreadType.CARD_WHEEL`

### 3.2 chips 横向滑动

使用 `Scroll(ScrollDirection.Horizontal)` + 内部 `Row` 实现横向滑动，`scrollBar(BarState.Off)` 隐藏滚动条。不改为多行瀑布流。

### 3.3 chips 样式

- 选中态：金色背景 `Theme.GOLD_AMBER`(#CA8A04)，深色文字 `Theme.BG_DEEP_INDIGO`(#0F0F23)，无描边
- 未选中态：深蓝背景 `Theme.BG_CARD_DARK`(#1B1B30)，金色文字 `Theme.GOLD_AMBER`，金色细描边 `rgba(202,138,4,0.25)`
- 圆角：`Theme.RADIUS_FULL`(999) pill 形状
- 高度：38vp，左右 padding：18vp
- 字体：`Theme.FONT_BODY`(15fp)，`FontWeight.Medium`

### 3.4 小屏适配

横向 Scroll 确保小屏设备也能滑动查看全部五个筛选项，文案不会被裁切。

## 4. 旋转牌轮记录归类

### 4.1 数据流确认

旋转牌轮抽一张：
```
CardWheelDrawPage → router.replaceUrl(Routes.RESULT_GOLDEN, { entryMode: 'CARD_WHEEL', ... })
→ ResultGoldenPage.saveHistory()
  → entryMode === 'CARD_WHEEL' → spreadType = SpreadType.CARD_WHEEL, modeTitle = '旋转牌轮'
```

旋转牌轮抽三张：
```
CardWheelDrawPage → router.replaceUrl(Routes.TRIANGLE_RESULT, { source: 'card_wheel', ... })
→ TriangleResultPage.saveHistory()
  → source === 'card_wheel' → spreadType = SpreadType.CARD_WHEEL, modeTitle = '旋转牌轮'
```

以上数据流已正确实现，无需修改 CardWheelDrawPage 或两个结果页。

### 4.2 筛选逻辑

`HistoryPage.getFilteredRecords()` 按 `record.spreadType === SpreadType.CARD_WHEEL` 筛选旋转牌轮记录，两张和三张都会被正确筛出。

### 4.3 显示文案

- `ResultGoldenPage` 保存时 `modeTitle = '旋转牌轮'`（单牌来自牌轮）
- `TriangleResultPage` 保存时 `modeTitle = '旋转牌轮'`（三牌来自牌轮）
- 历史列表卡片展示 `record.modeTitle`，即"旋转牌轮"

## 6. 历史详情页修复

### 6.1 兼容的记录类型

详情页通过 `record.modeTitle` 显示记录类型，通过 `record.cards` 展示卡牌，通过 `record.spreadType` 判断牌阵。以下类型均可正常展示：

- 单牌（`SpreadType.SINGLE`）→ 显示 1 张牌 + 解读
- 圣三角（`SpreadType.TRIANGLE`）→ 显示 3 张牌（过去/现在/未来）+ 解读
- 关系探索（`SpreadType.RELATION`）→ 显示多张牌（我/对方/走向）+ 解读
- 旋转牌轮（`SpreadType.CARD_WHEEL`）→ 根据 cards 数量显示 1 或 3 张牌 + 解读

### 6.2 详情页展示内容

- 记录类型（`modeTitle`）
- 时间（`formatDate`）
- 卡牌区（`buildCardsSection`，使用 `TarotCardImage` 组件，5:8 比例）
- 综合解读（`summaryText`）
- 行动建议（`adviceText`）
- 删除按钮 + 合规页脚

### 6.3 滚动与安全空间

- 长内容使用 `Scroll` 包裹，可完整滚动
- 底部安全空间：128vp（避开手势条）
- 返回按钮使用 `AppBackButton({ fallbackUrl: Routes.HISTORY })`，优先 `router.back()`
- 详情页返回后不影响底部导航

### 6.4 删除按钮

- 颜色 `#E5998A`（柔和暖红）
- 背景 `rgba(229,153,138,0.08)`（极淡暖红）

## 7. 返回路径验收

### 7.1 入口方式

- MainFramePage（底部导航）→ `router.pushUrl(Routes.HISTORY)` → HistoryPage
- DivinationCenterPage（占卜中心）→ `router.pushUrl(Routes.HISTORY)` → HistoryPage

两者都使用 `pushUrl`，因此 `router.back()` 可正常返回源页面。

### 7.2 返回路径验证

| 路径 | 方式 | 结果 |
|------|------|------|
| HistoryPage → 返回 | `router.back()` | 回到 MainFramePage 或 DivinationCenterPage |
| HistoryPage → 空状态按钮 | `router.replaceUrl()` | 替换当前页为对应占卜入口 |
| HistoryPage → 卡片点击 | `router.pushUrl(HISTORY_DETAIL)` | 进入详情页 |
| HistoryDetailPage → 返回 | `router.back()` | 回到 HistoryPage |
| HistoryDetailPage → 删除 | `router.back()` | 回到 HistoryPage |

底部 5 个 tab 在返回后仍然可点击，不影响导航。
