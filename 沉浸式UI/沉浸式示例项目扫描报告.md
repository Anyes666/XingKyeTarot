1. `new WindowUtil(windowStage.getMainWindowSync())`
2. `AppStorage.setOrCreate('windowUtil', this.windowUtil)`

## 4. 页面级沉浸式实现

示例项目共演示 **3 种**页面级写法，分布在 4 个示例页面：

### 4.1 方案 A：纯组件背景沉浸（`Shopping.ets`）

```typescript
NavDestination() {
  Column() {
    Home()
  }
  .background('#F1F3F5')   // ← 唯一关键行
}
.hideTitleBar(true)
```

- 不调用任何 Window API
- 不调用 `ignoreLayoutSafeArea`
- 不设置 padding
- 背景色通过 `background()` 涂到根 Column，由系统自动延伸到状态栏 / 导航栏区域
- **内容仍留在安全区内**，不会顶到状态栏

✅ 这是最贴合星钥塔罗"深蓝黑背景延伸 + 内容不被遮挡 + 不改全局"目标的写法。

### 4.2 方案 B + E：组件页面沉浸 + 动态避让（`ShoppingAvoid.ets`）

```typescript
NavDestination() {
  Column() {
    Home()
  }
  .ignoreLayoutSafeArea()                    // ← 关键 1：忽略安全区
  .height(LayoutPolicy.matchParent)          // ← 关键 2：撑满窗口
  .padding({
    top: this.windowUtil?.mainWindowInfo.avoidSystem?.topRect.height + 'px',
    bottom: this.windowUtil?.mainWindowInfo.avoidNavigationIndicator?.bottomRect.height + 'px'
  })
  .backgroundColor($r('app.color.page_background'))
}
.hideTitleBar(true)
```

- 用 `ignoreLayoutSafeArea()` + `LayoutPolicy.matchParent` 让 Column 铺满整个窗口（包括状态栏、导航条区域）
- 用 `padding` 把内容推回安全区
- `top` 取 `avoidSystem.topRect.height`（状态栏高度）
- `bottom` 取 `avoidNavigationIndicator.bottomRect.height`（手势条高度）
- ⚠️ **注意单位**：这里直接 `+ 'px'` 拼字符串，未走 `px2vp`。在不同 dpi 设备上可能有偏差。星钥塔罗迁移时建议改用 `px2vp()` 转换后拼 `'vp'`，或直接用 `Padding` 对象传 number。

### 4.3 方案 B + E + 临时隐藏系统栏（`MiniGame.ets`）

```typescript
NavDestination() {
  Column() {
    Header()
    Content({ mainWindowInfo: this.windowUtil?.mainWindowInfo })
    Footer()
  }
  .ignoreLayoutSafeArea()
  .height(LayoutPolicy.matchParent)
  .width(CommonConstants.FULL_PERCENT)
  .backgroundImage($r('app.media.game_background'))
  .backgroundImageSize(ImageSize.FILL)
  .padding(this.mainWindowInfo?.avoidAreaPadding)   // ← 用 WindowUtil 算好的 Padding 对象
}
.hideTitleBar(true)
.onShown(() => this.onShown())     // → setWindowSystemBarEnable([]) 隐藏状态栏+导航栏
.onHidden(() => this.onHidden())   // → setWindowSystemBarEnable(['status','navigation']) 恢复
```

- 这是"游戏式全屏"：进入页面隐藏系统栏，离开恢复
- `avoidAreaPadding` 由 `WindowUtil.updateAvoidPadding()` 计算得出（已 `px2vp`）
- ❌ **不适合星钥塔罗**：塔罗需求明确"不隐藏状态栏、不隐藏导航条"

## 5. 避让区处理方式

集中在 `WindowUtil.ets`，是示例项目的**核心工具类**：

### 5.1 一次性获取（`updateWindowInfo()`，第 192–230 行）

```typescript
this.mainWindowInfo.avoidSystem =
  this.mainWindow.getWindowAvoidArea(window.AvoidAreaType.TYPE_SYSTEM);
this.mainWindowInfo.avoidNavigationIndicator =
  this.mainWindow.getWindowAvoidArea(window.AvoidAreaType.TYPE_NAVIGATION_INDICATOR);
this.mainWindowInfo.avoidCutout =
  this.mainWindow.getWindowAvoidArea(window.AvoidAreaType.TYPE_CUTOUT);
this.mainWindowInfo.avoidSystemGesture =
  this.mainWindow.getWindowAvoidArea(window.AvoidAreaType.TYPE_SYSTEM_GESTURE);
this.mainWindowInfo.avoidKeyboard =
  this.mainWindow.getWindowAvoidArea(window.AvoidAreaType.TYPE_KEYBOARD);
```

### 5.2 动态监听（`onAvoidAreaChange`，第 59–78 行）

```typescript
this.mainWindow.on('avoidAreaChange', this.onAvoidAreaChange);
```

监听回调内根据 `avoidOptions.type` 分发到 `avoidSystem / avoidCutout / avoidSystemGesture / avoidKeyboard / avoidNavigationIndicator`。注意 **CUTOUT 类型** 还会触发 `updateAvoidPadding()` 重新计算 padding。

### 5.3 padding 计算（`updateAvoidPadding`，第 83–107 行）

```typescript
updateAvoidPadding(area: window.AvoidArea) {
  const avoidAreaPadding: Padding = {};
  try {
    if (area.topRect.height > 0) {
      avoidAreaPadding.top = this.uiContext?.px2vp(area.topRect.height + area.topRect.top);
    }
    if (area.leftRect.width > 0) {
      avoidAreaPadding.left = this.uiContext?.px2vp(area.leftRect.left + area.leftRect.width);
    }
    if (area.rightRect.width > 0) {
      avoidAreaPadding.right = this.uiContext?.px2vp(display.getDefaultDisplaySync().width - area.rightRect.left);
    }
    if (area.bottomRect.height > 0) {
      avoidAreaPadding.bottom = this.uiContext?.px2vp(display.getDefaultDisplaySync().height - area.bottomRect.top);
    }
  } catch (error) {
    let err = error as BusinessError;
    hilog.error(0x0000, 'TestLog', `Failed to set avoid padding. Code: ${err.code}, message: ${err.message}`);
  }
  this.mainWindowInfo.avoidAreaPadding = avoidAreaPadding;
}
```

- ✅ 正确使用 `px2vp` 转换
- ✅ try/catch 包裹
- ⚠️ `Padding` 类型用 `{}` 初始化，部分字段 undefined——ArkTS 严格模式下需确认 `Padding` 允许部分字段缺省
- ⚠️ `avoidAreaPadding` 只在 CUTOUT 变化时更新，**SYSTEM / NAVIGATION_INDICATOR 变化时未重算**——这是示例项目的一个已知简化

### 5.4 使用方式

| 页面 | 取值方式 | 单位 |
|---|---|---|
| ShoppingAvoid | `avoidSystem.topRect.height + 'px'` | px（未转换，有 dpi 风险） |
| ShoppingAvoid | `avoidNavigationIndicator.bottomRect.height + 'px'` | px（未转换） |
| MiniGame | `mainWindowInfo.avoidAreaPadding`（Padding 对象） | vp（已转换） |

## 8. 对星钥塔罗的最佳方案建议

### 推荐方案：**方案 A（组件背景沉浸）为主 + 受限使用方案 E（动态避让区）兜底**

| 项 | 内容 |
|---|---|
| **方案名称** | 组件背景沉浸 + 可选动态避让 |
| **推荐等级** | ★★★★★ |
| **为什么适合** | 1. 塔罗几乎所有页面都是 `#0F0F23` 深蓝黑背景，正是方案 A 的最佳场景<br>2. 方案 A 零 Window API，不改 EntryAbility 全局行为<br>3. 状态栏 / 导航栏正常显示，符合"系统时间电量可见 + 手势条可见"<br>4. 内容自动留在安全区，不需要每页算 padding<br>5. 单行 `.background('#0F0F23')` 即可落地，改动最小<br>6. 对 Scroll 页面无副作用（背景在根容器，Scroll 在内层） |
| **需要改哪些类型的文件** | 仅页面根容器（每个 `@Entry` 或 `NavDestination` 内的根 `Column` / `Stack`） |
| **是否需要改所有页面** | 否。可逐页落地，优先改有白边问题的页面。未改的页面维持原状不影响 |
| **是否适合 Scroll 页面** | ✅ 适合。背景涂在 Scroll 的父容器上，Scroll 内部布局不受影响 |
| **是否会影响保存分享卡截图** | ❌ 不影响。截图通常是组件级 `componentSnapshot`，与窗口背景无关 |
| **是否会影响服务卡片** | ❌ 不影响。服务卡片是独立 FormAbility，与主 Ability 窗口无关 |
| **是否会影响隐私门禁（LaunchPage）** | ❌ 不影响。LaunchPage 若是独立 Ability，其窗口行为独立；若是主 Ability 内页面，方案 A 不改 EntryAbility，门禁逻辑不动 |

### 备选方案对比

| 方案 | 适合塔罗？ | 理由 |
|---|---|---|
| 1. 只用 EntryAbility 设置窗口背景色 | ⚠️ 可作补充 | `setWindowBackgroundColor` 可设全局兜底色，但示例项目未用此 API。可作为方案 A 的双保险 |
| 2. `setWindowLayoutFullScreen(true)` + 页面 padding | ❌ 不推荐 | 全局强制全屏，所有页面都要手动避让，改动面大，Scroll 页面风险高 |
| 3. 页面根容器 `background()` | ✅ **首选** | 即方案 A |
| 4. `expandSafeArea` | ⚠️ 示例未用 | 示例项目零命中，且滚动容器中可能无效，塔罗多 Scroll 页面不建议 |
| 5. `ignoreLayoutSafeArea` + WindowUtil 避让 | ⚠️ 仅特殊页 | 即方案 B+E，适合"需要内容真正铺到状态栏下"的特殊页（如启动页全屏图），普通页面不必 |
| 6. 混合方案 | ✅ **最终建议** | 普通页面用方案 A；启动页 / 牌阵展示页等需要全屏图的用方案 B+E |

## 9. 风险清单

| 风险 | 出现场景 | 严重程度 | 规避建议 |
|---|---|---|---|
| 顶部内容顶到状态栏 | 方案 B 未设 `padding.top` | 高 | 方案 A 不存在此问题；方案 B 必须设 top padding |
| 底部按钮被手势条遮挡 | 方案 B 未设 `padding.bottom` | 高 | 方案 A 不存在；方案 B 必须取 `avoidNavigationIndicator.bottomRect.height` |
| Scroll 页面避让无效 | `expandSafeArea` 在滚动容器内 | 中 | 不用 `expandSafeArea`，改用方案 A 或在 Scroll 外层做 padding |
| 弹窗 / Toast 异常 | 弹窗用了全屏 Stack 但未避让 | 中 | 弹窗组件单独处理，不依赖页面级沉浸式 |
| 保存分享卡截图范围错 | 截图取了含状态栏的全屏 | 低 | 截图用组件级 API，限定到目标组件 |
| 服务卡片入口异常 | 卡片 Ability 误用主 Ability 窗口设置 | 低 | 卡片是独立 Ability，不共享主窗口 |
| LaunchPage 协议门禁异常 | 门禁逻辑依赖窗口状态 | 低 | 方案 A 不改 EntryAbility，门禁逻辑零影响 |
| 全项目页面都要改 | 方案 D（全局全屏） | 高 | 不选方案 D；方案 A 可逐页改 |
| SDK 不兼容 API | `ignoreLayoutSafeArea` / `LayoutPolicy.matchParent` / `TYPE_NAVIGATION_INDICATOR` 需 API 20 | 高 | 先核对塔罗 `build-profile.json5` 的 `compatibleSdkVersion`；若低于 20，方案 B 不可用，只能用方案 A |
| ArkTS 编译错误 | `Padding = {}` 空对象 / `as BusinessError` / `uiContext!` 非空断言 | 中 | 塔罗严格模式下 `Padding = {}` 可能告警，建议显式列出字段或用 `Optional` |
| `px + 'px'` 字符串拼接 | `ShoppingAvoid.ets` 第 42–43 行 | 中 | 不同 dpi 偏移；改用 `px2vp() + 'vp'` |
| `avoidAreaPadding` 不随系统栏变化更新 | `WindowUtil.ets` 仅 CUTOUT 触发 update | 低 | 塔罗若复用 WindowUtil，应在 SYSTEM / NAVIGATION_INDICATOR 分支也调用 `updateAvoidPadding` |

## 10. 下一轮改星钥塔罗的最小落地建议

> 本轮不改代码，只给方向。

### 第一步：核对 SDK 版本 + 选定主方案
- 打开星钥塔罗 `build-profile.json5`，确认 `compatibleSdkVersion`
- 若 ≥ `6.0.0(20)`：主方案 A + 备选 B+E 都可用
- 若 < 20：**只能用方案 A**（`background()` 是老 API，全版本可用）
- **验证**：记录版本号，决定是否引入 `ignoreLayoutSafeArea`

### 第二步：在一个最简单的页面验证方案 A
- 选一个白边最明显的页面（如首页或某个纯色背景页）
- 在根 `Column` / `Stack` 上加一行 `.background('#0F0F23')`
- 不动 EntryAbility，不动其他页面
- **验证**：真机运行，观察状态栏区域是否变深蓝黑、内容是否仍正常显示、底部手势条区域是否变深蓝黑

### 第三步：若方案 A 不足以覆盖（如启动页需要全屏图），再引入方案 B+E
- 仅对需要"内容真正铺到状态栏下"的特殊页用 `ignoreLayoutSafeArea + matchParent + padding`
- padding 必须用 `px2vp` 转换，不要直接拼 `'px'`
- 可参考 `WindowUtil.ets` 的避让区获取逻辑，但裁剪掉传感器 / 分屏 / 自由窗口等无关功能
- **验证**：真机旋转屏幕、切换挖孔机型，观察 padding 是否随避让区变化

## 12. 一句话总结

示例项目用**最干净的 EntryAbility**（不动窗口全局行为）+ **三种页面级方案**（背景沉浸 / 页面沉浸+避让 / 自由窗口）演示沉浸式。对星钥塔罗最稳妥的路径是 **方案 A：页面根容器 `.background('#0F0F23')`** —— 单行 API、零 Window 改动、内容自动避让、Scroll 无副作用、可逐页落地。仅当个别页面需要内容真正铺到状态栏下时，才按 `ShoppingAvoid.ets` 的写法引入 `ignoreLayoutSafeArea + matchParent + padding`，且 padding 必须用 `px2vp` 转换。
