# 星钥塔罗 · HarmonyOS 设计理念适配分析（四）：风险、优先级、实施提示词

---

## 9. 不建议采用的设计理念

| 理念 | 原因 |
|:---|:---|
| **全新色彩（蓝白体系）** | 与星钥"深蓝夜空+金色星钥"视觉调性根本冲突。蓝白=科技/效率/纯净；星钥需要神秘/安静/月光。 |
| **过强动态视觉** | 星澜是"安静陪伴"，不是"炫酷展示"。大面积视差、粒子爆发、光效闪烁都会破坏安静感。 |
| **过多数据可视化** | 星钥塔罗不是数据工具。仅情绪星图需要轻量可视化，不要图表/仪表盘。 |
| **过度立体化** | 轻度 Z 轴层次可以，但不要 3D 旋转卡片、视差滚动、复杂阴影。 |
| **过度悬浮** | 卡片悬浮仅用于"选择"类场景。语录页大面积悬浮导致阅读疲劳。 |
| **过度卡片化** | 卡片化仅限于信息容器。对话、长文本、合规声明不需要卡片包裹。 |
| **赌场化/游戏化动效** | 绝对禁止：闪烁边框、金币特效、中奖音效、"恭喜"文案、概率展示、转盘无限旋转。 |
| **万能卡片换肤** | 当前阶段资源优先投入核心体验。 |
| **跨设备流转** | 星澜聊天涉及用户隐私情绪，不适合跨设备无缝流转。 |

## 10. 最终优先级计划

### P0：必须马上做（2-3 个迭代，共约 13h）

| # | 设计理念 | 涉及页面 | 具体改动 | 预期收益 | 风险 | 工时 |
|:---|:---|:---|:---|:---|:---|:---|
| 1 | 沉浸光感 | 启动页、语录页、结果页、卡牌详情 | 叠加 radialGradient 微光；优化语录页渐变遮罩；全屏预览背景加光感 | 品牌质感显著提升；纯色背景不再单调 | 低端设备性能 | 4h |
| 2 | 动效物理动力学 | 牌轮页、翻牌页、聊天页、导航 | 牌轮摩擦减速+弹性吸附；翻牌缓出曲线；气泡弹性出现；导航微缩放 | 核心交互手感大幅提升 | 参数需反复调试 | 6h |
| 3 | 悬浮+吸引 | 占卜中心、牌轮页 | 入口卡片微悬浮+光晕；牌轮中心吸附增强 | 入口引导更清晰；牌轮体验更流畅 | 阴影在暗色背景不明显 | 3h |

### P1：适合近期做（3-5 个迭代，共约 15h）

| # | 设计理念 | 涉及页面 | 具体改动 | 预期收益 | 风险 | 工时 |
|:---|:---|:---|:---|:---|:---|:---|
| 4 | 日食 | 结果页、语录页 | 翻牌明暗过渡；语录切换停顿+微闪 | 仪式感增强；切换更温柔 | 三牌揭示总时长偏长 | 4h |
| 5 | 轻拟物 | 全页面卡片、按钮 | 卡片金色描边+柔阴影；按钮按压反馈；气泡纸张质感 | UI 品质感提升 | 改动面大 | 5h |
| 6 | 平衡 | 圣三角结果页、关系探索页 | 非对称空间平衡排列 | 牌阵视觉更有层次 | 机械对称风险 | 3h |
| 7 | 空间感 | 启动页、语录页、卡牌总览 | 多层 Stack 景深；列表渐隐 | 星空主题深度感增强 | 多层渲染负担 | 3h |

### P2：后续增强（6+ 迭代，共约 18h）

| # | 设计理念 | 涉及页面 | 具体改动 | 工时 |
|:---|:---|:---|:---|:---|
| 8 | 图标卡片形状 | 全页面 | 统一圆角体系 | 4h |
| 9 | HarmonyOS Sans | 全局 | 字体替换 | 2h |
| 10 | 多设备适配 | 全页面 | 折叠屏/平板布局 | 12h+ |

### P3：暂不建议

| 理念 | 原因 |
|:---|:---|
| 全新色彩（蓝白） | 与品牌调性冲突 |
| 万能卡片换肤 | 非 MVP 需求 |
| 跨设备流转 | 隐私考量 + 非 MVP |
| 数据可视化（过度） | 不符合产品定位 |

## 11. Agent 后续执行提示词

以下提示词可直接交给 CodeBuddy / Agent 执行 UI 优化：

```text
请执行「星钥塔罗 HarmonyOS 设计理念 UI 优化专项 — P0 批次」。

## 约束条件
- 只做 UI 和交互优化，不改核心逻辑
- 不改星澜聊天核心逻辑（引擎/安全/路由/Composer/Session 均不动）
- 不改抽牌算法
- 不改结果页解读逻辑
- 不接入大模型
- 完成后必须 assembleHap，ERROR=0
- 遵守工程铁律（ArkTS 严格类型、禁止 any、禁止大规模重构）
- 不要引入新依赖

## P0-1：沉浸光感优化

### LaunchPage.ets
- 加载状态背景（纯色 BG_DEEP_INDIGO Column）叠加 radialGradient：
  中心色 `rgba(245, 215, 122, 0.03)` (GOLD_PALE 3%)，向外渐变到透明，覆盖全屏
- 品牌文字区域（APP_NAME + 副标题）下方加金色光晕 shadow

### HomePage.ets / MainFramePage.ets XinglanQuotesTabContent
- 优化 9 段渐变遮罩：顶部过渡从 `#030716` 改为从 BG_DEEP_INDIGO 开始，过渡区 48vp
- 底部过渡区 24vp
- 中间区域的遮罩密度降低（减少渐变段数或降低 opacity）
- 背景星空图 xinglan_home_main 的 opacity 从 0.82 提升到 0.88

### ResultGoldenPage.ets
- 结果卡片区域（翻牌后显示牌面+解读的 Column）叠加 radialGradient：
  中心色 `rgba(202, 138, 4, 0.04)` (GOLD_AMBER 4%)，半径约 200vp，向外透明

### TarotCardDetailPage.ets
- 全屏预览背景从 `rgba(5,7,20,0.96)` 改为 radialGradient：
  中心微亮 `rgba(20, 20, 40, 0.96)`，边缘深暗 `rgba(5, 7, 20, 0.96)`

## P0-2：动效物理动力学

### CardWheelDrawPage.ets
- handleDragEnd() 中松手后前 200ms 使用较高的摩擦系数（模拟静摩擦→动摩擦）
- 吸附动画使用 animateTo + curve: Curve.customCubic(0.34, 1.56, 0.64, 1)
- 吸附后中心牌金色光晕渐显（200ms）
- 当前已有的 PanGesture distance:0 和增量累加保持不动

### ResultGoldenPage.ets
- 翻牌动画 duration 从 600ms 改为 450ms
- 动画曲线从默认改为 Curve.EaseOut
- 翻牌时牌面 scale 从 0.95 过渡到 1.0

### XinglanChatPage.ets
- 新消息气泡出现动画：
  scale(0.92→1.0) + opacity(0→1)，duration 350ms
  curve: Curve.customCubic(0.34, 1.56, 0.64, 1)
- typing 指示器三点弹动（每个点 offset 2vp 上下交替，周期 600ms）

### CompanionBottomNav.ets
- 图标点击反馈：scale(0.9→1.0)，duration 200ms，Curve.EaseOut
- 不做旋转、不做弹跳

## P0-3：悬浮 + 吸引

### MainFramePage.ets — DivinationCenterTabContent
- 5 个入口卡片添加 shadow({ radius: 6, color: 'rgba(0,0,0,0.15)', offsetY: 1 })
- 按压态：scale(0.98) + shadow radius 缩小到 2
- 第 2 个卡片（圣三角牌阵）背景色比其余亮 5%：rgba(36,36,64,0.72)
- 卡片列表区域中央叠加 radialGradient 微光（GOLD_PALE 5%，不遮挡卡片文字）

### CardWheelDrawPage.ets
- 中心牌（currentIndex 对应的牌）比两侧牌 translateY 上浮 4vp
- 中心牌金色描边宽度 2vp（其余牌 1vp），颜色 GOLD_LIGHT
- 两侧牌（currentIndex±2, ±3）opacity 从 1.0 降到 0.7

## 验证
- 每个文件修改完成后检查 linter
- 全部完成后 assembleHap
- 确认 ERROR=0
- 在模拟器 127.0.0.1:5555 上验证交互手感
```
