## 〇、计划使用说明

### 0.1 文档定位

- 本文档是给**开发团队**的可执行施工手册，不是给 AI Agent 的提示词。
- 每个阶段都包含：**目标 / 改动文件清单（精确到行号锚点）/ 技术实现要点 / 验收清单 / 风险**。
- 团队按阶段顺序执行即可，每阶段独立可交付，每阶段后必须 `hvigor assembleHap` 且 `ERROR=0`。

### 0.2 全局工程铁律（每阶段都适用，不重复写）

1. **ArkTS 严格类型**：禁用 `any` / `as any` / `Record<string, Object>` / 匿名对象 / `Array<{...}>`；所有对象用显式 `interface` 或 `class`。
2. **路由**：跳转参数必须先声明变量，再构造 `router.RouterOptions`，`try/catch` 包裹。
3. **UI 闭包**：`Column/Row/Scroll/Stack` 内不写 `const/let/赋值`，逻辑放方法里。
4. **异步**：所有 `async` 必须 `try/catch`，错误用 `hilog` 记录。
5. **不扩散**：每阶段只改指定文件，不重构、不替换 API、不改业务逻辑。
6. **编译验收**：每阶段后 `hvigor assembleHap`，`ERROR=0` 才进下一阶段；`WARN` 可暂留。
7. **产品底线**：所有新增文案不得出现"预言/治愈/疗愈/诊断/必须/一定/注定/建议你"等词，必须符合"不预言、不承诺、不治疗、不替用户决定"。

### 2.3 精确改动点

#### ① 新增状态字段（HomePage.ets 第 1057-1062 行 @State 区块附近）

```typescript
@State isXinglanResponding: boolean = false;   // 星澜是否正在回应（防抖+时序控制）
@State isPlayingHomeGuide: boolean = false;    // 首页首次引导是否正在播放（与点击互斥，避免首次体验被打断）
@State hasScheduledHomeGuide: boolean = false; // 首页引导是否已调度（防 aboutToAppear 重复触发，本生命周期内不重置）
```

> **互斥规则（产品评审建议 3）**：首页首次引导播放期间，点击星澜不追加随机文案。
> 这样首次体验的对话流是干净的：初始文案 → 引导句 1 → 引导句 2，不会被"星澜安静地听着。"+随机文案打断。
> `addNextXinglanQuote()` 开头需同时判断两个标志（见下文 ③）。
>
> **防重复调度（产品评审建议 3）**：`hasScheduledHomeGuide` 防止 `aboutToAppear` 短时间多次触发导致重复 setTimeout，本字段在页面生命周期内不重置。详见阶段 2 ②。

#### ② 新增临时文案常量（XINGLAN_QUOTES 定义之前，约第 16 行）

```typescript
const XINGLAN_LISTENING_HINT: string = '星澜安静地听着。';
```

#### ③ 改造 `addNextXinglanQuote()` 方法（第 1079-1088 行）

**保留**：原循环索引逻辑、scrollEdge 逻辑。
**新增**：防抖判断、临时句追加、延迟、临时句移除、正式句追加。

伪代码结构（团队按 ArkTS 规范实现）：

```typescript
private async addNextXinglanQuote(): Promise<void> {
  // 1. 防抖 + 引导互斥：正在回应或正在播放首页引导时直接返回
  if (this.isXinglanResponding || this.isPlayingHomeGuide) {
    return;
  }
  this.isXinglanResponding = true;

  try {
    // 2. 先追加临时句
    const listWithHint: string[] = [...this.quoteList, XINGLAN_LISTENING_HINT];
    this.quoteList = listWithHint;
    this.scrollToBottom();

    // 3. 延迟 800~1500ms
    const delay: number = 800 + Math.floor(Math.random() * 700);
    await this.sleep(delay);

    // 4. 移除临时句（保守版：只删尾部确为临时句的那条，避免误删并发追加的引导文案）
    const baseList: string[] = this.removeListeningHintFromTail();

    // 5. 追加正式句（保留原循环逻辑）
    this.quoteIndex = (this.quoteIndex + 1) % XINGLAN_QUOTES.length;
    const finalList: string[] = [...baseList, XINGLAN_QUOTES[this.quoteIndex]];
    this.quoteList = finalList;
    this.scrollToBottom();
  } catch (err) {
    hilog.error(0x0000, 'HomePage', 'addNextXinglanQuote error: %{public}s', JSON.stringify(err));
  } finally {
    this.isXinglanResponding = false;
  }
}

/**
 * 保守移除临时句：仅当尾部确实等于 XINGLAN_LISTENING_HINT 时才删，
 * 否则原样返回。避免与首页首次引导并发时误删引导文案。
 */
private removeListeningHintFromTail(): string[] {
  if (this.quoteList.length === 0) {
    return [];
  }
  if (this.quoteList[this.quoteList.length - 1] === XINGLAN_LISTENING_HINT) {
    return this.quoteList.slice(0, this.quoteList.length - 1);
  }
  return this.quoteList;
}

private sleep(ms: number): Promise<void> {
  return new Promise<void>((resolve: () => void) => {
    setTimeout(() => { resolve(); }, ms);
  });
}

private scrollToBottom(): void {
  try {
    this.scroller.scrollEdge(Edge.Bottom);
  } catch (_err) {
    // 忽略
  }
}
```

#### ④ 调用点改造（第 1272、1281 行）

#### ② 新增页面生命周期方法（HomePage.ets 第 1062 行附近）

> **ArkUI 生命周期规范（产品评审建议 2）**：`aboutToAppear` 保持普通 `void`，异步逻辑放私有方法里，稳定性和可读性更好。阶段 3、5 同理。
>
> **防重复调度（产品评审建议 3）**：`aboutToAppear` 理论上可能短时间内触发多次，而 `hasSeenHomeGuide` 还没来得及写入，导致重复安排 `setTimeout`。增加 `hasScheduledHomeGuide` 标志保险，本标志无需重置（页面生命周期内只需调度一次）。

```typescript
aboutToAppear(): void {
  this.prepareHomeGuide();
}

private async prepareHomeGuide(): Promise<void> {
  try {
    // 防重复调度：已安排过或正在播放，直接返回
    if (this.hasScheduledHomeGuide || this.isPlayingHomeGuide) {
      return;
    }
    const hasSeen: boolean = await PreferenceStore.getBoolean('hasSeenHomeGuide', false, getContext(this));
    if (!hasSeen) {
      // 确认要播放，置位调度标志（本生命周期内不重置）
      this.hasScheduledHomeGuide = true;
      // 延迟到页面渲染完成后追加，避免与初始 quoteList[0] 冲突
      setTimeout(() => { this.playFirstLaunchGuide(); }, 600);
    }
  } catch (err) {
    hilog.error(0x0000, 'HomePage', 'prepareHomeGuide error: %{public}s', JSON.stringify(err));
  }
}

private async playFirstLaunchGuide(): Promise<void> {
  // 置位引导播放标志，与点击星澜互斥（见阶段 1 ③）
  this.isPlayingHomeGuide = true;
  try {
    // 第一句
    this.quoteList = [...this.quoteList, HOME_FIRST_GUIDE_LINE_1];
    this.scrollToBottom();
    await this.sleep(HOME_GUIDE_DELAY_MS);
    // 第二句
    this.quoteList = [...this.quoteList, HOME_FIRST_GUIDE_LINE_2];
    this.scrollToBottom();
    // 标记已看过
    await PreferenceStore.putBoolean('hasSeenHomeGuide', true, getContext(this));
  } catch (err) {
    hilog.error(0x0000, 'HomePage', 'playFirstLaunchGuide error: %{public}s', JSON.stringify(err));
  } finally {
    // 引导结束，恢复点击星澜能力
    this.isPlayingHomeGuide = false;
    // 注意：hasScheduledHomeGuide 不重置，本页面生命周期内只调度一次
  }
}
```

> **注意**：`sleep` 方法在阶段 1 已定义，复用即可。如果阶段 2 先于阶段 1 执行，需在本阶段同步定义 `sleep`。
>
> `hasScheduledHomeGuide` 字段需在阶段 1 ① 的 @State 区块附近一起声明（见下文 ③ 补充）。

#### ③ 初始 quoteList 处理

### 8.5 抽取工具方法

```typescript
// 从指定场景池中随机抽取一条，支持最近去重（可选）
export function pickLineByScene(
  pool: CompanionLine[],
  scene: CompanionScene,
  recentTexts: string[]  // 最近用过的文案，用于去重
): string {
  const filtered: CompanionLine[] = pool.filter((item: CompanionLine) => item.scene === scene);
  if (filtered.length === 0) {
    return '';
  }
  // 优先选不在 recentTexts 中的
  const available: CompanionLine[] = filtered.filter((item: CompanionLine) => recentTexts.indexOf(item.text) === -1);
  const candidates: CompanionLine[] = available.length > 0 ? available : filtered;
  const idx: number = Math.floor(Math.random() * candidates.length);
  return candidates[idx].text;
}
```

## 十四、一句话总结

> 本次交互优化的核心不是"增加更多功能"，而是：
