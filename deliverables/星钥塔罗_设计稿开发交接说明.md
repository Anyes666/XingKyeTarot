## 第 1 章：交接总览

### 1.1 项目背景

星钥塔罗是一款 HarmonyOS 本地离线塔罗式情绪陪伴 App。核心理念：**陪伴式自我整理**——让抽牌结果像一次"被看见"，而非算命预言。所有功能本地运行，不需要联网，不收集用户数据。

### 1.2 设计阶段成果

| 阶段 | 内容 | 产出 |
|------|------|------|
| Phase 1 | 需求发现 | 用户画像、场景分析、功能优先级排序 |
| Phase 2 | 设计系统 | 三套视觉方案定义 + 完整 Token 体系（design-system-tokens.md + proposals-v2.md） |
| Phase 3 | 原型生成 | 14 个高保真 HTML 原型，覆盖全部核心页面 |
| Phase 4 | 质量审查 | 原型逐页审查通过，符合四条铁律 |
| **Phase 5** | **导出交付** | **本文档——将 14 个原型落地为 ArkTS 页面的完整交接说明** |

### 1.3 开发团队任务

拿到 14 个 HTML 原型 + 本交接说明后，开发团队应能**独立完成**以下工作，不需要反问设计师：

1. 理解每个原型使用了哪套设计方案（B/C/A），对应哪些 Token
2. 找到每个原型对应的 ArkTS 页面文件和路由常量
3. 识别需要新建的页面、组件，以及需要视觉迁移的已有页面
4. 理解状态机设计、文案接入点、动效实现方式
5. 遵守 ArkTS 开发铁律，确保编译通过

### 3.3 路由注册现状

当前 `main_pages.json` 共注册 **38 个页面**。上述 14 个原型中：

- **11 个原型对应已注册页面**（#2, #3, #4, #6, #7, #9, #10, #11, #12, #13, #14）

- **1 个原型为组件级**（#8 LevelUpPopup，不注册路由）

需新增的 Routes 常量（在 `Routes.ets` 中添加）：

```arkts
// ── Phase 5 新增路由 ──
static readonly XINGLAN_CHAT_ENTRANCE: string = 'pages/XinglanChatEntrancePage';
static readonly STAR_CALENDAR: string = 'pages/StarCalendarPage';
```

需在 `main_pages.json` 的 `"src"` 数组中添加：

```json
"pages/XinglanChatEntrancePage",
"pages/StarCalendarPage"
```

### 3.4 ★ 令牌基准页说明

原型 #4（卡牌结果页 `04-card-result.html`）被标记为**令牌基准页**——它是 C 方案 + B 月光氛围的混合典型，包含了：

- C 方案完整色彩体系（bg-base-C / bg-card-C / gold-gilt-C / spine-line-C）
- C 方案完整字体体系（serif 标题 + sans 正文 + mono 编号）
- B 方案月光光晕（借用氛围）
- C 方案烫金线装饰
- 标准卡片结构（牌面 + 牌名 + 关键词 + 金句 + 深度解读 + 免责声明）

开发团队应以 #4 为 C 方案页面的实现基准，其他 C 方案页面的 Token 用法参考 #4。

## 第 7 章：动效实现建议（CSS → ArkTS 映射）

### 7.1 动效映射总表

| CSS 原型写法 | ArkTS 实现方式 | 说明 |
|-------------|---------------|------|
| `animation: pageEnter 300ms ease-out` | `.animation({duration:300, curve:Curve.EaseOut})` + `@State opacity/translateY` | 页面进入 |
| `@keyframes moonlightBreath 6s` | `animateTo({duration:6000, iterations:-1, curve:Curve.EaseInOut})` + `@State opacity` | 月光呼吸（无限循环） |
| `transform: rotateY(180deg)` | `.rotate({ y: flipAngle, centerX:'50%' })` + `animateTo` | 翻牌动画 |
| `animation: cardFlipIn 400ms` | `animateTo({duration:400, curve:Curve.EaseOut})` + `@State rotateY/opacity` | 牌面入场 |
| `@keyframes fadeIn 500ms` | `.animation({duration:500, curve:Curve.EaseOut})` + `@State opacity` | 淡入 |
| `transform: scale(0.97)` pressed | `@State pressedIndex` + `.scale({x:0.97,y:0.97})` + `.animation({duration:140})` | 按压态 |
| `LoadingProgress` 旋转 | ArkTS `<LoadingProgress>()` 组件原生支持 | 加载指示 |
| `@media (prefers-reduced-motion)` | ArkTS 无原生等价，需用 `featureConfiguration` 判断 | 降级 |
| stagger 入场（delay 100ms 递增） | 多个 `animateTo` + `setTimeout` 或 `delay` 参数 | 卡片错落入场 |

### 7.2 关键动效实现示例

#### 月光呼吸（B 方案背景光晕）

```arkts
@State moonlightOpacity: number = 0.6;

aboutToAppear() {
  // 无限循环呼吸动画
  animateTo({
    duration: 6000,
    iterations: -1,
    curve: Curve.EaseInOut
  }, () => {
    this.moonlightOpacity = 1.0;
  });
}
```

> **提醒**：`iterations: -1` 表示无限循环。不同页面的呼吸周期可略有差异（5-8s），避免全局同步。

#### 翻牌动画（#12 抽牌页）

```arkts
@State flipAngle: number = 0;
@State cardOpacity: number = 0;

startFlip() {
  // 阶段 1：牌背淡入
  animateTo({ duration: 300, curve: Curve.EaseOut }, () => {
    this.cardOpacity = 1;
  });
  // 阶段 2：翻牌
  setTimeout(() => {
    animateTo({ duration: 600, curve: Curve.EaseOut }, () => {
      this.flipAngle = 180;
    });
  }, 300);
}
```

UI 中使用：

```arkts
Image(this.cardBack)
  .width(200).height(320)
  .rotate({ y: this.flipAngle, centerX: '50%' })
  .opacity(this.cardOpacity)
```

> **提醒**：翻牌动画必须用 `.rotate({ y: flipAngle })` 不是 `.rotateAngle`。ArkTS 的 `.rotate()` 接受 `{ y: number }` 参数表示绕 Y 轴旋转，这与 CSS 的 `transform: rotateY(180deg)` 语义对应。

#### 按压态（#11 问心选项卡 / #10 占卜入口卡）

```arkts
@State pressedIndex: number = -1;

// Touch 事件处理
Touch() {
  .onTouch((event: TouchEvent) => {
    if (event.type === TouchType.Down) {
      this.pressedIndex = index;
    } else if (event.type === TouchType.Up || event.type === TouchType.Cancel) {
      this.pressedIndex = -1;
    }
  })
  .scale({
    x: this.pressedIndex === index ? 0.97 : 1.0,
    y: this.pressedIndex === index ? 0.97 : 1.0
  })
  .animation({ duration: 140, curve: Curve.EaseOut })
}
```

> **提醒**：按压态必须用 `@State pressedIndex` + `onTouch` 事件实现，不要用 CSS 的 `:active` 伪类（ArkTS 不支持伪类）。

### 7.3 降级处理

原型中通过 `@media (prefers-reduced-motion: reduce)` 实现动效降级。ArkTS 无原生等价方案，建议：

1. 使用 `featureConfiguration` 获取系统无障碍设置
2. 定义全局降级开关：`static readonly REDUCED_MOTION: boolean`
3. 降级时：关闭 `iterations: -1` 循环动画，关闭粒子效果，仅保留必要的淡入淡出

降级版月光层（无动画静态光斑）：

```arkts
// 降级：使用 Theme.v2bMoonlightGlowDim() 静态渲染
// 正常：使用 animateTo 循环动画
```

## 第 9 章：ArkTS 开发铁律（★ 最高优先级，必须遵守）

> ⚠️ **以下规则来自项目 MEMORY，违反将导致编译失败或运行崩溃。所有开发人员在动手前必须通读本章节。**

### 9.1 严格类型规则

#### ⚠️ 禁止操作

1. **禁止匿名对象类型**：不得使用 `{ a: string }` 作为类型注解
2. **禁止 `Array<{ ... }>`**：不得使用内联匿名类型的数组
3. **禁止 `Record<string, Object>`**：不得使用 Object 作为 Record 的值类型
4. **禁止 `any`**：不得使用 `any` 类型
5. **禁止 `as` 强行类型断言**：不得使用 `as` 进行类型转换
6. **禁止直接返回匿名对象**：不得 `return { ... }` 返回未声明类型的对象
7. **禁止路由内联匿名参数**：不得 `router.pushUrl({ url: xxx, params: { ... } })`
8. **禁止无 RouterOptions 的路由**：不得 `router.replaceUrl({ url: xxx })` 不带类型
9. **禁止 UI 闭包内声明变量**：Column/Row/Scroll/Stack 内不得写 `const`/`let`/赋值
10. **禁止 UI 闭包内裸调用函数**：UI 结构内不得写普通函数调用

#### 必须操作

1. **必须显式 interface 或 class 声明**：所有对象结构用 `export interface` 或 `export class` 声明
2. **必须先声明变量再 return**：`return` 前先显式类型变量
3. **必须先声明 params 变量**：路由跳转先声明 `params` 变量，再声明 `router.RouterOptions`

   ```arkts
   // ✅ 正确写法
   let params: Record<string, string> = { cardId: cardId };
   let options: router.RouterOptions = { url: Routes.TAROT_CARD_DETAIL, params: params };
   try {
     router.pushUrl(options);
   } catch (e) {
     console.error('路由跳转失败: ' + e);
   }
   ```

4. **必须 try/catch 路由操作**：router 操作需 try/catch 或安全封装
5. **必须安全解析 getParams**：`getParams` 后需字段校验
6. **必须校验 JSON.parse 结果**：`JSON.parse` 后需校验字段存在性
7. **必须注册新页面**：新增页面需 `@Entry` + `@Component` + `main_pages.json` 注册
8. **组件不需注册路由**：新增组件（非页面）不需要在 `main_pages.json` 注册
9. **只改指定文件**：每个任务只改指定文件，不扩散到无关文件

### 9.2 编译铁律

⚠️ **任何任务完成后必须 `assembleHap` 编译验证。**

- `ERROR > 0` → 先修 ERROR，禁止扩功能
- `WARN` 分阶段处理，不为清 WARN 破坏主流程
- 以 `assembleHap` 编译结果为准，`ERROR = 0` 为第一验收标准
- 编译不过的代码不允许合入

### 9.3 API 调用铁律

⚠️ **禁止猜 HarmonyOS API。**

- 禁止调用当前 SDK 不存在的 API
- 保存图片/相册/权限/组件截图以当前 SDK 编译通过为准
- 功能暂未适配 → 明确提示失败，**绝不假成功**

### 10.4 风险提示

1. **C 方案衬线字体**：HarmonyOS Serif 在部分设备上可能无中文衬线字形，需确认是否需要打包 Noto Serif SC（约 4MB）
2. **月光呼吸性能**：多个页面同时运行 `iterations: -1` 动画可能影响性能，建议页面 `aboutToDisappear` 时停止动画
3. **翻牌动画兼容性**：`.rotate({ y: angle })` 在低版本 ArkUI 可能无 3D 透视效果，需验证
4. **78 张牌图加载**：卡牌总览页一次加载 78 张图可能内存压力大，建议用 LazyForEach + 异步加载
5. **历史记录存储**：结果页自动保存历史，需确认 DivinationHistoryService 的存储上限和清理策略
