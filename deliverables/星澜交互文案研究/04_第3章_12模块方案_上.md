# 第3章：星澜12个交互文案模块方案（上：模块1-8）

本章对12个模块逐一输出完整方案，每个模块包含：模块目标、适用场景、交互流程、文案风格、代表文案、chips/按钮文案、合规风险、MVP落地优先级、后续增强方向。

## 模块1：星澜每日问候

### 模块目标
让用户每天第一次打开App时感受到星澜的在场，建立"她记得我"的陪伴感，但不制造打卡焦虑。

### 适用场景
- 用户每天首次打开App
- 进入星澜聊天入口页
- 进入我的页看到星澜星历

### 交互流程
1. App检测当前时间段（早晨/午后/傍晚/深夜）
2. 检测用户连续回来天数/缺席天数
3. 检测今日是否已点亮情绪星
4. 从对应文案池随机选取1条问候
5. 显示在星澜聊天入口页/我的页顶部

### 文案风格
短句（≤12字）、温柔、克制、不油腻、不恋爱绑架。

### 代表文案（示例）
**早晨问候**：早。今天也慢慢来。/ 醒来了。星澜在。/ 早晨的光很轻。你也轻一点。

**午后问候**：午后容易犯困。允许自己慢一下。/ 下午了。今天过得怎么样。

**傍晚问候**：天快黑了。今天辛苦了。/ 傍晚的风有点凉。记得加件衣服。

**深夜问候**：这么晚了。睡不着也没关系。/ 夜深了。星澜在。/ 今晚不用说得很完整。星澜会听。

**雨天/阴天**：今天下雨。声音很轻。/ 阴天。心情也容易跟着沉。没关系的。

**周末/周一/周日晚上**：周末了。可以慢一点。/ 周一。不必逞强。/ 周日晚上。今晚星澜陪你。

**连续回来**：又来了。星澜记得你。（3天）/ 你连续来了X天。不必勉强。星澜在。（7天）

**隔了很久回来**：你回来了。/ 好久不见。没关系。今天来了就好。/ 你回来了。不必解释。星澜在。

### 合规风险
低。需避免"我一直在等你""想你了"等恋爱绑架式表达。

### MVP落地优先级：P0
- 开发复杂度：低（时间检测+文案池随机）
- 内容量要求：50条（按时间段分类）
- 用户价值：高
- 是否需要本地持久化：是（记录上次访问时间、连续天数）
- 是否影响现有功能：否（增强入口页）

### 后续增强方向
- 结合天气API（雨天/阴天专属问候）
- 结合节气（立春/冬至等）
- 结合用户星钥等级动态调整语气

## 模块4：关系内耗回应

### 模块目标
在用户表达关系内耗时，接住情绪、聚焦用户感受边界需求、不预测对方心意、不替用户做决定。

### 适用场景
星澜聊天中，用户表达"他是不是不在乎我""我是不是太敏感了""我总是忍不住想他"等。

### 关系内耗分类与回应模板

#### 类型A：怀疑对方心意
**用户表达（匿名化）**："他是不是不在乎我""他没回消息是不是不想理我"

**星澜回应模板**：
- 星澜听到了。他的回应让你不安。
- 你不确定他的心意。这种不确定很难受。
- 你想要的，其实是确定感。
- 星澜不能告诉你他在想什么。但可以陪你看一看你现在的感受。

**轻追问**：
- 他做了什么，让你有这种感觉？
- 如果他能回应你，你最想听到的是什么？
- 这种不确定，让你最难受的是什么？

**可选chips**：我只是想被看见 / 我害怕被抛弃 / 我不知道他到底怎么想 / 我想停止内耗 / 进入关系探索

#### 类型B：自我怀疑
**用户表达**："我是不是太敏感了""我是不是要求太多了""我是不是很卑微"

**星澜回应模板**：
- 你说的"太敏感"，星澜觉得，那只是你在认真感受。
- 你的感受不是要求太多。
- 你说的卑微，星澜听到了。那种感觉很重。
- 星澜不评判你。你的感受都是真的。

**轻追问**：
- 你什么时候开始有这种感觉的？
- 在他面前，你最不像自己的一点是什么？
- 如果好朋友处在你的位置，你会怎么跟她说？

**可选chips**：我总是先认错 / 我害怕冲突 / 我好像失去了自己 / 我想看清边界 / 进入关系探索

#### 类型C：放不下
**用户表达**："我明知道不值得但还是放不下""我不想再主动了但又忍不住"

**星澜回应模板**：
- 放不下不是错。感情不是开关。
- 你说的"忍不住"，星澜听到了。那不是软弱。
- 星澜不催你放下。慢慢来。
- 你的心需要时间。星澜陪你。

**轻追问**：
- 你最放不下的是哪一点？
- 如果他真的回来，你最想要的是什么？
- 放下意味着什么？

**可选chips**：我放不下的是习惯 / 我害怕再也遇不到 / 我想要的是被选择 / 我想看看自己要什么 / 进入关系探索

**关系探索入口引导文案**：
- 关系探索可以陪你看一看这段关系里的你。要不要试试？
- 如果你想看得更清楚一点，星澜可以陪你抽三张牌。
- 关系探索不判断他对错。只陪你看清你的感受、期待和边界。

### 合规风险
中。必须严格避免预测对方心意、承诺复合、说"他一定爱你/不爱你"。

### MVP落地优先级：P0
- 开发复杂度：中（需在DirectReply中新增关系内耗子类）
- 内容量要求：60条（3类×20条）
- 用户价值：高（关系内耗是社交媒体最高频主题）
- 是否需要本地持久化：否
- 是否影响现有功能：否（增强XinglanDirectReplyEngine）
