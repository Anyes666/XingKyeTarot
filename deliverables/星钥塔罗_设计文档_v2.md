## 1. 设计总览

### 1.1 产品定位

**星钥塔罗**（XingKey Tarot）是 HarmonyOS NEXT 本地离线塔罗式情绪陪伴 App。核心角色**星澜**是"月光下的倾听者"——不预言未来，不替用户做决定，只陪用户看见此刻的自己。

**品牌核心表达**：不是预测未来，而是陪你看见此刻的自己。

### 1.2 设计目标

将 App 从旧版"黑金星月"单一风格，升级为**品牌级场景分层混合视觉系统**，建立统一、成熟、有辨识度的视觉风格，同时保留产品功能稳定性。

### 1.3 设计成果

| 产出 | 数量 | 说明 |
|------|------|------|
| 视觉方向方案 | 3 套 | A 星图档案 / B 月湖映心 / C 星钥之书 |
| 设计系统 Token | 1658 行 / 10 章节 | 完整色彩/字体/间距/圆角/阴影/图标/组件/动效/降级规范 |
| 高保真原型 | 9 个页面 | HTML/CSS 纯静态原型，390×844 基准 |
| Theme.ets v2 Token | 100+ 常量 | 三套色板+间距+圆角+字号+阴影+动效+工具方法 |
| 已迁移页面 | 14 个 | 从旧 Token 迁移到 v2 Token，编译通过 |

## 2. 品牌哲学与设计原则

### 2.1 星澜人格六大原则

| 原则 | 设计转译 |
|------|---------|
| 温柔但不黏腻 | 像月光而非阳光，不冷不热；深色背景是"包裹"而非"黑暗" |
| 具体但不说教 | 文案卡片短、轻、留白；不大段心灵鸡汤 |
| 接住但不替代现实支持 | 安全边界温柔可见；危机提示清晰但不恐吓 |
| 有角色感但不恋爱绑定 | 星澜符号化（月光环/剪影/星钥），禁止乙游立绘 |
| 有仪式感但不打卡焦虑 | 星历像"情绪旅程"非"签到任务"；无连签/断签/积分 |
| 不预言、不承诺、不制造依赖 | 所有表达聚焦"看见此刻"，无预测/命运/复合暗示 |

### 2.2 六条设计红线

| 红线 | 禁止 |
|------|------|
| 算命感 | 命运/预测/注定/运势/魔法阵/水晶球 |
| 恋爱绑定感 | 暧昧/亲密/好感度/乙游立绘 |
| 博彩感 | 大面积金色/赌场金/金光闪闪 |
| 恐怖玄学感 | 骷髅/死神恐怖意象/神秘光效 |
| AI 工具感 | "正在分析"/"生成中"/客服腔 |
| 强打卡感 | 签到/连签/断签/积分/排行榜 |

### 2.3 四条修正铁律

1. **星澜形象**：可用头像/月光环/半透明剪影/星钥符号，禁止全屏立绘/恋爱姿态/好感度
2. **卡牌总览**：78 张牌完整开放，禁止锁图标/付费解锁
3. **星钥等级阈值**：0/1/7/21/30/90/180/365 天 = 初见/星火初亮/七日星轨/月下自省者/星钥守望者/星河同行者/半年听心者/长夜星澜
4. **动效降级**：标准版（月光呼吸+星点+涟漪）+ 降级版（静态光斑+无粒子+无连续动画）

## 3. 场景分层混合策略

### 3.1 核心策略

不采用单一风格，而是按使用场景分三层，各层使用不同方案，共享基础色板：

```
┌─────────────────────────────────────────────┐
│  聊天陪伴层（B 月湖映心）                      │
│  入口页 / 聊天页 / 情绪星点亮                  │
│  关键词：月光、涟漪、光晕、呼吸、被接住          │
├─────────────────────────────────────────────┤
│  卡牌内容层（C 星钥之书）                      │
│  占卜中心 / 抽牌 / 结果页 / 卡牌总览 / 详情     │
│  关键词：翻阅、深度、烫金、书脊、被理解          │
├─────────────────────────────────────────────┤
│  时间记录层（A 星图档案）                      │
│  星历 / 等级 / 我的页面                        │
│  关键词：档案、克制、记录、星轨、被记住          │
└─────────────────────────────────────────────┘
```

### 3.2 三套方案对比

| 维度 | A 星图档案 | B 月湖映心 | C 星钥之书 |
|------|-----------|-----------|-----------|
| 核心隐喻 | 天文台观测日志 | 深夜湖边月光 | 古籍图书馆 |
| 主情绪 | 被认真对待 | 被温柔包裹 | 被深度理解 |
| 字体 DNA | 几何无衬线+等宽 | 人文无衬线 | 衬线标题+无衬线正文 |
| 金色占比 | ≤3% | ≤5%（含光晕） | ≤4%（含烫金线） |
| 圆角倾向 | 12-16px | 16-24px | 8-12px |
| 层次手段 | 明度差 | 光晕+阴影 | 阴影+边框 |
| 动效节奏 | 精准、克制 | 流动、呼吸 | 翻页、层叠 |
| 聊天场景适配 | ★★★ | ★★★★★ | ★★★ |
| 卡牌场景适配 | ★★★★ | ★★★★ | ★★★★★ |

### 3.3 共享基础

| 维度 | 规则 |
|------|------|
| 深蓝背景区间 | `#060B1A` ~ `#0A0D1A` |
| 暖金点缀区间 | `#C5A050` ~ `#D4AC4D` |
| 月光白文字区间 | `#EBE3CD` ~ `#F2EDDD` |
| 金色使用规则 | 无渐变、无大面积使用、无发光 |
| 正文对比度 | ≥ 4.5:1（WCAG AA） |
| 标题对比度 | ≥ 3:1（WCAG AA） |

| 星点眨眼 | 4-6s 非同步 opacity 0.2↔0.5 | 固定 opacity 0.3 |
| 气泡出现 | opacity+translateY 400ms ease-out | opacity only 150ms |
| Typing dots | 三点依次 opacity 1200ms 循环 | 三个静态圆点 |
| 按钮按压 | scale 0.98 + 光晕收缩 100ms | scale 0.98 80ms linear |
| 弹窗出现 | translateY+overshoot 220ms | translateY 150ms 无overshoot |
| 星星点亮 | scale+opacity+光晕扩散 650ms 三阶段 | opacity only 200ms |
| 涟漪扩散 | 圆形扩散 400ms | 无 |
| 翻页 | 3D rotateY 400ms | opacity 150ms |

### 9.2 降级触发条件

- 系统开启 `prefers-reduced-motion`
- 低端设备检测（帧率 < 50fps）
- 电量低于 20%
- HarmonyOS ArkTS 默认优先稳定

### 9.3 降级实现

所有页面包含 `@media (prefers-reduced-motion: reduce)` 媒体查询，统一移除循环动画。

// 阴影
