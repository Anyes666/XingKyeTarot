# 星钥塔罗 · 单牌快占积极方向与文案增强专项 — 交付报告

## 1. 专项概述

本专项为单牌快占功能新增 6 个积极方向，在原有"情绪接住"方向基础上，提供"看见光"方向的温柔选择，让用户不仅可以被理解，还能被点亮。核心原则：不预言未来、不承诺好运、不制造宿命感；积极感 = 温柔、有光、克制、具体、不夸张、不鸡血。

**编译状态**：BUILD SUCCESSFUL（2026-06-30，ERROR=0，WARN=115，与基线一致）。

## 3. 积极方向设计详情

### 3.1 6 个积极方向选项

| 序号 | 标签 | EmotionTag | 选项标题 | 描述文案 |
|------|------|------------|---------|---------|
| 1 | 给我鼓励 | `给我鼓励` | 给我一点鼓励 | 今天有点灰，想被一句话接住，哪怕只是温柔的肯定 |
| 2 | 今日微光 | `今日微光` | 看看今天的微光 | 不需要大道理，只想看见今天有什么小小的光亮 |
| 3 | 找回力量 | `找回力量` | 找回我的力量 | 感觉自己被掏空了，想知道我其实还有余力 |
| 4 | 一点勇气 | `一点勇气` | 给我一点勇气 | 在犹豫要不要迈出那一步，想被轻轻推一下 |
| 5 | 积极一面 | `积极一面` | 看看积极的那一面 | 我知道事情没那么糟，但想有人帮我指出来 |
| 6 | 一句提醒 | `一句提醒` | 给我一句温柔的提醒 | 在忙乱中快忘了自己是谁，想要一句小声的提醒 |

### 3.2 加权抽牌矩阵

8 张 MVP 卡（愚者、魔术师、女祭司、力量、隐士、星星、太阳、月亮）× 6 个积极方向 = 48 条加权配置。每个方向每张卡有 0-3 的权重值，高权重卡在对应方向下更容易被抽中。

示例（"给我鼓励"方向）：
- 力量（major_08_strength）：权重 3 — 最核心匹配
- 太阳（major_19_sun）：权重 3
- 星星（major_17_star）：权重 2
- 魔术师（major_01_magician）：权重 2

### 3.3 场景化文案结构

每条文案复用现有 ScenarioContent 接口，包含三个字段：
- **goldenSentence**：一句温柔、有光、克制的金句（不含感叹号、不含绝对化表述）
- **emotionKeywords**：该方向 × 该卡牌的关键情绪词（2-3 个）
- **todayAction**：今天可以尝试的一件小事（具体、可执行、不鸡汤）

## 4. UI 交互流程

### 4.1 AskHeartPage 双模式切换

单牌快占页面新增两个 Tab：
- **"今天我想被接住"**（原有情绪方向，6 个选项）
- **"今天我想看见光"**（新增积极方向，6 个选项）

Tab 切换器仅在"单牌快占"模式下显示（首次体验模式不显示），活跃 Tab 使用金色填充（GOLD_AMBER）。

页面标题和副标题根据模式动态变化：
- 原有模式：标题"今天，你更想靠近哪种感受？"，副标题"选一种你此刻最贴切的感受，让一张牌陪你说说话。"
- 积极模式：标题"今天，你想看见什么？"，副标题"选一个你此刻需要的方向，让一张牌给你温柔的提醒。"

### 4.2 CardDrawPage 抽牌分支

- 判断 emotionTag 是否为积极方向（检查是否为 6 个 PositiveEmotionTag 之一）
- 积极方向：调用 `drawCardByPositiveWeight()` 进行加权抽牌，调用 `getPositiveGoldenSentence()` 获取金句
- 原有方向：保持原有 `drawCardByWeight()` 和 `getGoldenSentence()` 逻辑不变

### 4.3 ResultGoldenPage 结果展示

- 根据 emotionTag 判断模式
- 积极方向：调用 `getPositiveScenarioContent()` 加载积极版场景化文案（金句 + 情绪关键词 + 今日行动建议）
- 原有方向：保持原有 `getScenarioContent()` 逻辑不变

### 4.4 HistoryDetailPage 方向展示

历史详情页新增所选方向标签显示，位于时间戳下方，使用 caption 字号 + secondary 文字颜色，单行省略溢出。

## 5. 数据流转

```
用户选择积极方向
  → AskHeartPage.onConfirm() 设置 emotionTag = PositiveEmotionTag, questionText = 选项标签
  → 路由传递 emotionTag + questionText 到 CardDrawPage
  → CardDrawPage.isPositiveEmotionTag() 判断 → isPositiveMode = true
  → CardDrawPage.performDraw() 调用 drawCardByPositiveWeight(emotionTag) → DrawCardResult
  → 路由传递 emotionTag + cardId + isReversed 到 ResultGoldenPage
  → ResultGoldenPage.isPositiveEmotionTag() 判断 → isPositiveMode = true
  → ResultGoldenPage.loadData() 调用 getPositiveScenarioContent(emotionTag, cardId) → 场景化文案
  → 保存历史记录时，scenarioTag = emotionTag, questionText = 选项标签
  → HistoryDetailPage 读取 questionText 显示方向标签
```
