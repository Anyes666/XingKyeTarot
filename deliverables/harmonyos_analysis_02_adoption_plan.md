# 星钥塔罗 · HarmonyOS 设计理念适配分析（二）：采用清单与页面方案

---

## 3. 可直接采用的 10 个设计理念

### 3.1 沉浸光感 [P0 — 最高优先级]

**为什么适合**：星空主题与"光感"天然融合。当前纯色背景 `#0F0F23` 较平，加光感后层次感显著提升。"月光透过云层"的柔光直接提升品牌质感。

**用在哪些页面**：启动页（星空动态微光）、星澜语录页（背景图边缘柔光扩散）、结果页（金色柔光漫反射）、卡牌详情全屏预览（微光背景）、聊天页（Header 下方光感过渡）。

**具体怎么做**：
- 用 `linearGradient` + `radialGradient` 模拟光感扩散（API 12 兼容方案，不用 `systemMaterialEffect`）
- 光感色温保持暖金色（`GOLD_PALE` 透明度 5%-15%），不用冷白光
- 启动页：纯色背景叠加 `radialGradient`（中心 `GOLD_PALE` 3%，向外透明）
- 语录页：优化 9 段渐变遮罩——顶部过渡 48vp、底部 24vp，中间区降低遮罩密度
- 结果页：结果卡片区域叠加金色柔光 `radialGradient`

**风险**：渐变叠加过多影响低端设备性能；需确保 AMOLED 屏幕不过曝。

---

### 3.2 动效物理动力学 [P0 — 最高优先级]

**为什么适合**：牌轮惯性滚动是核心交互；翻牌需要"轻柔翻开"手感；当前牌轮已有基础惯性但缺乏弹性吸附和摩擦力模拟。

**用在哪些页面**：旋转牌轮页（惯性+弹性吸附+摩擦力）、单牌结果页（翻牌缓出曲线）、星澜聊天页（气泡弹性放置）、底部导航（微缩放回弹）。

**具体怎么做**：
- 牌轮：松手后前 200ms 摩擦系数 0.95，之后 0.85；弹性吸附 `cubic-bezier(0.34, 1.56, 0.64, 1)`
- 翻牌：`ANIM_FLIP_DURATION` 600ms→450ms，曲线改为 `ease-out`
- 气泡：`scale(0.92→1.0)` + `opacity(0→1)`，`cubic-bezier(0.34, 1.56, 0.64, 1)`，350ms
- 导航：图标 `scale(0.9→1.0)`，200ms，`ease-out`

**风险**：弹性过大产生"果冻效应"；牌轮物理参数需真机反复测试。

---

### 3.3 悬浮 [P0 — 最高优先级]

**为什么适合**：天然适合"选择"类场景——星钥塔罗的核心交互就是"选择一张牌"。与"星空"主题完美融合（星体悬浮）。

**用在哪些页面**：占卜中心（入口卡片微悬浮）、旋转牌轮（当前牌高亮悬浮）、星澜语录（当前语录微悬浮）、卡牌总览（选中标签悬浮）。

**具体怎么做**：
- 悬浮用 `shadow` + 轻微 `translate` 实现，不用 `position` 硬定位
- 阴影色值：`rgba(202, 138, 4, 0.08)`（金色微影，不用黑色硬影）
- 悬浮高度差：2-4vp，不超 6vp
- 占卜中心：5 个入口卡片 `shadow({ radius: 6, offsetY: 1 })`；按压时下沉 2vp+阴影缩小
- 牌轮：中心 7 张可见牌中，最中间牌比两侧高 4vp + 金色描边发光；两侧微透明 `opacity(0.7)`

**风险**：大面积悬浮导致视觉疲劳；阴影在 AMOLED 暗色背景上可能不明显。

---

### 3.4 吸引 [P0 — 最高优先级]

**为什么适合**：牌轮中心吸附是核心 UX——"牌自动滑到手指下方"；占卜入口需要"引力汇聚"的视觉引导。

**用在哪些页面**：旋转牌轮（中心牌吸附+光晕反馈）、占卜中心（入口卡片微光脉动）、星澜聊天（发送按钮微光呼吸）。

**具体怎么做**：
- 牌轮吸附已实现，增强：吸附后中心牌金色光晕从无到有（200ms 渐显）
- 入口卡片：中心卡片（圣三角）比其余略亮（`rgba(36,36,64,0.72)`），列表中央叠加 `radialGradient` 微光（`GOLD_PALE` 5%）
- 发送按钮：`opacity(1.0↔0.7)` 呼吸动画，周期 2s

**风险**：脉动频率太高分散注意力；吸附太快有"跳帧"感。

---

### 3.5 轻拟物 [P1]

**为什么适合**：卡片是核心 UI 元素，需区分于纯扁平；塔罗牌背纹理需要"纸牌"触感；按钮需要按压质感。

**用在哪些页面**：全页面卡片（统一阴影+金色描边）、按钮（按压缩放+亮度变化）。

**具体怎么做**：
- 卡片：`border({ width: 1, color: 'rgba(202,138,4,0.12)' })` + `shadow({ radius: 12, color: 'rgba(0,0,0,0.3)' })` + 顶部高光线 `border({ width: { top: 1 }, color: 'rgba(255,255,255,0.05)' })`
- 按钮按压态：`scale(0.97)` + `brightness(1.1)`，100ms
- 语录气泡：`linearGradient(180deg, rgba(27,27,48,0.72), rgba(27,27,48,0.64))` + 上边缘微亮

**风险**：阴影在深色背景上可能不明显；改动面大。

---

### 3.6 日食 [P1]

**为什么适合**：翻牌揭示是核心仪式感——"从黑暗到光明"；语录切换可用"月相"式淡入淡出。

**用在哪些页面**：单牌结果页（翻牌动效）、圣三角结果页（三牌依次揭示）、星澜语录（切换明暗过渡）。

**具体怎么做**：
- 翻牌：牌背 opacity 1→0，牌面 opacity 0→1，同时轻微 `scale(0.95→1.0)`，时长 400ms
- 语录切换：当前 1→0（300ms）+ 停顿 200ms（"星澜安静地听着"）+ 新 0→1（400ms）
- 翻牌时背景微暗 `brightness(0.85)` 100ms 后恢复
- 停顿期间背景微暗+星光粒子微闪

**风险**：三牌依次揭示总时长 1.2s 可能偏长；背景变暗不能太明显否则像"闪屏"。

---

### 3.7 平衡 [P1]

**为什么适合**：圣三角牌阵是时间平衡；关系探索是双人平衡；对话是陪伴平衡（不越界、不依赖）。

**用在哪些页面**：圣三角结果页（非对称空间平衡）、关系探索页（双人视觉权重微调）、星澜聊天（气泡节奏一致）。

**具体怎么做**：
- 圣三角：过去牌略偏左后、现在牌居中略前、未来牌略偏右后——非对称平衡
- 关系探索：`SpaceEvenly` 但视觉重量通过透明度微调（避免绝对均等暗示）
- 聊天：星澜气泡与用户气泡的间距节奏保持一致

**风险**：机械对称失去温暖感；关系探索不要暗示"平等=完美"。

---

### 3.8 空间感 [P1]

**为什么适合**：深蓝夜空天然需要空间纵深；启动页湖面、星空、月亮、星澜背影是多层空间。

**用在哪些页面**：启动页（前景湖面→中景星澜→远景月亮→深空）、语录页（前景遮罩→中景气泡→远景星空图）、卡牌总览（列表顶部/底部渐隐）。

**具体怎么做**：
- 启动页：四层 Stack，每层 `opacity` 和 `scale` 微差
- 语录页：当前已有三层，优化渐变遮罩过渡区
- 卡牌总览：列表顶部 `linearGradient` 从 `BG_DEEP_INDIGO` 到透明（16vp 过渡）

**风险**：多层 Stack 增加渲染负担；渐变过渡区太短显得生硬。

---

### 3.9 图标卡片形状 [P2]

**为什么适合**：当前圆角体系不统一（6/10/12/16/20/999），需标准化。

**用在哪些页面**：全页面。

**统一圆角体系**：

| 元素 | 当前 | 建议 | 理由 |
|:---|:---|:---|:---|
| 大卡片 | 12vp | 20vp | 对齐 HarmonyOS 舒适圆角 |
| 小卡片/标签 | 16vp | 12vp | 小元素小圆角更协调 |
| 塔罗牌 | 10vp | 10vp（保持） | 接近真实牌比例 |
| 按钮 | 999 | 999（保持） | 胶囊按钮已是标准 |
| 气泡 | 16vp | 20vp | 统一卡片体系 |

**风险**：改动量大；部分页面圆角改动可能影响布局。

---

### 3.10 全场景 HarmonyOS Sans [P2]

**为什么适合**：统一字体提升品牌质感；书法美学与东方幻想契合；无级字重匹配星钥字号层级。

**用在哪些页面**：全局。

**具体怎么做**：Theme 中定义 `FONT_FAMILY = 'HarmonyOS Sans'`；确认 DevEco Studio 引用方式；逐页替换。

**风险**：需确认字体在目标 HarmonyOS 版本的可用性。

## 4. 页面级落地方案

### 4.1 启动页（LaunchPage）

| 理念 | 落地方案 |
|:---|:---|
| **沉浸光感** | 纯色背景上叠加 `radialGradient`（中心 `GOLD_PALE` 3%，向外透明），模拟"星钥微亮"。品牌文字下方加金色光晕。 |
| **起源** | 加载动画：星钥图标从 0→1 opacity 渐显（1200ms），副标题在星钥亮起后 400ms 淡入。叙事："混沌→星光汇聚→星钥点亮"。 |
| **日食** | 隐私弹窗出现时背景微暗 `brightness(0.8)`（300ms），弹窗从下方滑入。同意后背景恢复+淡出。 |
| **空间感** | 后续引入插画时四层 Stack：前景湖面→中景星澜背影→远景月亮→深空。当前可叠加微弱星空粒子。 |

### 4.2 星澜语录页（HomePage / XinglanQuotesTabContent）

| 理念 | 落地方案 |
|:---|:---|
| **卡片式设计** | 气泡 borderRadius 16→20，左侧加 2vp 金色竖线（`GOLD_AMBER` 40%）作为视觉锚点。 |
| **悬浮** | 气泡 `shadow({ radius: 8, offsetY: 2 })` + 金色微光边缘；当前阅读气泡比已读高 2vp。 |
| **轻拟物** | 气泡背景改为 `linearGradient(180deg, rgba(27,27,48,0.72), rgba(27,27,48,0.64))` + 上边缘微亮。 |
| **沉浸光感** | 优化渐变遮罩：顶部 48vp 过渡，底部 24vp。中间区降低密度，让星空图更可见。 |
| **日食** | 语录切换：opacity 1→0(300ms)→停顿 200ms→新 0→1(400ms)。停顿期间背景微暗+星光微闪。 |

### 4.3 占卜中心页（DivinationCenterTabContent）

| 理念 | 落地方案 |
|:---|:---|
| **图标卡片形状** | 卡片圆角 12→20；图标 28→32px，加统一圆形底托（24vp 圆 + 图标居中）。 |
| **吸引** | 中心卡片（圣三角）比其余略亮，列表中央叠加 `radialGradient` 微光。 |
| **悬浮** | 卡片 `shadow({ radius: 6, offsetY: 1 })`；按压时下沉 2vp+阴影缩小。 |
| **轻拟物** | 卡片顶部高光线：`border({ width: { top: 1 }, color: 'rgba(255,255,255,0.06)' })`。 |

### 4.4 旋转牌轮页（CardWheelDrawPage）

| 理念 | 落地方案 |
|:---|:---|
| **平衡** | 中央留空区保持"空"的平衡感——有微弱星光的虚空，不是空无一物。 |
| **吸引** | 吸附后中心牌金色光晕从无到有（200ms 渐显）。 |
| **物理动力学** | 松手后前 200ms 摩擦系数 0.95→之后 0.85；弹性吸附 `cubic-bezier(0.34, 1.56, 0.64, 1)`。 |
| **悬浮** | 中心牌比两侧高 4vp + 金色描边发光；两侧微透明 `opacity(0.7)`。 |

**防赌场化规则**：绝对禁止闪烁边框、金币特效、中奖音效、"恭喜"文案、概率展示。牌轮不转超 3 圈；惯性停止节奏缓慢。

### 4.5 卡牌总览页（TarotCardsTabContent）

| 理念 | 落地方案 |
|:---|:---|
| **卡片式设计** | 列表项背景 `rgba(27,27,48,0.52)`→`rgba(27,27,48,0.68)`，加金色细边框。 |
| **图标卡片形状** | 筛选标签 borderRadius 16→12；列表项 borderRadius 6→12。 |
| **悬浮** | 列表项 `shadow({ radius: 4, offsetY: 1 })`。 |
| **空间感** | 列表顶部/底部 `linearGradient` 渐隐（16vp），暗示"还有更多牌在下方"。 |

### 4.6 卡牌详情页（TarotCardDetailPage）

| 理念 | 落地方案 |
|:---|:---|
| **立体空间设计** | 牌图在前（`scale(1.02)` + 柔阴影），文字在后。 |
| **沉浸光感** | 全屏预览背景改为 `radialGradient`（中心微亮 `rgba(20,20,40,0.96)`，边缘深暗）。 |
| **物理动力学** | 双击缩放从跳变改为 `animateTo` 缓动（300ms）；双指缩放跟手率 1:1。 |

### 4.7 星澜聊天页（XinglanChatPage）

| 理念 | 落地方案 |
|:---|:---|
| **沉浸光感** | Header 下方到第一条消息之间加金色光感过渡（`linearGradient` 12vp）。星澜气泡有极淡暖光材质（`radialGradient` 从左上角）。 |
| **轻拟物** | 星澜气泡：轻微纸张质感（上边缘微亮+底部微影）。用户气泡：保持简洁但加 1px 金色描边。 |
| **物理动力学** | 气泡出现 `scale(0.92→1.0)`+`opacity(0→1)`，`cubic-bezier(0.34,1.56,0.64,1)`，350ms。Typing 三点微弹动。 |
| **悬浮** | 输入框区域轻微悬浮（`shadow` + `translateY(-2)`），与键盘分离。 |

**防过度拟人规则**：
- 星澜头像保持 28×28vp 圆形，不做眨眼/表情动画
- 气泡不做爱心/花瓣/星星装饰；不使用粉红色系
- Typing 状态仅用三点弹动，不用"星澜正在想你…"

### 4.8 设置/关于/素材与授权页

| 理念 | 落地方案 |
|:---|:---|
| **系统一致性** | 设置项样式参考 HarmonyOS 系统设置：左图标+标题+右箭头，间距统一。 |
| **清晰层级** | 分组标题用 `FONT_CAPTION` + `MOON_MUTED` + 全大写间距。分组间用 1px `DEEP_STARRY` 分割线。 |
| **长文本** | 素材与授权页 9 部分改为可折叠手风琴。长文本行间距 1.6，段落间距 16vp。 |
| **规范文本** | 版本信息、隐私政策、用户协议链接用 `GOLD_AMBER` + 下划线。外部链接加"↗"图标。 |
