## 一、用户场景分析

### 1.1 核心用户画像

| 维度 | 描述 |
|------|------|
| **人口特征** | 18-28 岁，以 00 后为主（Z 世代），女性占比预估 > 60% |
| **生活状态** | 求学/初入职场/独居或合租，处于人生过渡期 |
| **情绪基调** | 非病理性孤独感——日常、可承受、但反复出现 |
| **技术素养** | 数字原住民，习惯 App 陪伴，对 AI 无过高期待 |
| **审美敏感** | 对"AI 味"高度敏感，能识别模板化/批量化表达 |

### 1.2 使用时机 × 情绪状态矩阵

| 时段 | 使用频率 | 典型触发情绪 | 用户核心诉求 | 对星澜的期待 |
|------|---------|------------|-------------|------------|
| **深夜 22:00-02:00** | ⭐⭐⭐⭐⭐ 最高 | 失眠、孤独、轻度崩溃、周日晚上焦虑 | "有人听我说，但不要打扰我" | 安静陪伴，不催促睡眠 |
| **周末白天** | ⭐⭐⭐ | 无事空虚、关系内耗、自我怀疑 | "帮我理一理想不通的事" | 接住情绪，给出可行动下一步 |
| **工作日碎片时间** | ⭐⭐⭐ | 工作/学习压力、拖延、委屈 | "给我一点光" | 一句不鸡汤的鼓励 |
| **节日/生日** | ⭐⭐ | 节日孤独、生日低落 | "有人记得我" | 轻轻一句在意话，不过度庆祝 |
| **情绪突发时刻** | ⭐⭐⭐⭐ | 关系内耗、轻度崩溃、想被安慰 | "立刻有一个地方可以去" | 秒回应，不追问，不审判 |

### 1.3 用户行为心理模型

用户使用星钥塔罗的**真实心理路径**：

```
"我有点不舒服，但说不清楚"
→ "不想打扰别人，也不想发朋友圈"
→ "打开星钥塔罗，抽一张牌"
→ "星澜帮我把混乱的感觉说出来了"
→ "我觉得被理解了"
→ "我可以继续过今晚了"
```

**关键洞察**：用户不是来找答案的，是来找一个**安全的地方放置情绪**。塔罗牌是"借口"——让用户允许自己脆弱。星澜是"容器"——接住但不溢出的陪伴。

## 三、核心用户路径（7 条关键流程）

### 路径 1：深夜孤独——"来一个人陪"

```
App 启动（深夜）
→ Tab「星澜聊天」
→ 星澜聊天入口页（看到动态文案"慢慢说，星澜会先听你说完"）
→ 进入聊天页 → 选择相处方式「陪我静」
→ 星澜回应短句 → 用户随意打字或不发 → 星澜不追问
→ 用户可能收到一张"夜信"（次日早晨）
→ 离开（没有被追问的压力）
```

**成败关键**：入口页文案能否在 2 秒内传递"这里安全"。聊天页是否不催促、不追问。

### 路径 2：情绪日常——"抽一张牌看看"

```
App 启动（碎片时间）
→ Tab「占卜」
→ 每日一占 / 单牌快占
→ 输入一句话心事
→ 旋转牌轮（滑动选牌）
→ 卡牌结果页（情绪陪伴式解读）
→ 完成 → 点亮情绪星（成功页）
→ （可选）分享卡 / 追问
→ 情绪星出现在星澜星历上
```

**成败关键**：卡牌结果页面是"镜子"而非"宣判"。点亮情绪星是"温柔收据"，非打卡。

### 路径 3：关系内耗——"我到底该不该……"

```
App 启动（可能多次）
→ Tab「占卜」→ 关系探索
→ 输入困扰 → 抽取三张牌（我/对方/走向）
→ 结果页：不判断对错，帮看见自己的感受
→ （可选）进入星澜聊天「帮我理」模式讨论
→ 不触发"复合概率""对方心意"等计算
```

**成败关键**：结果页是否严守"不替用户决定"的边界。避免任何"你应该分手/应该主动"的表达。

### 路径 4：星澜星历回顾——"看看我走过的日子"

```
App 启动
→ Tab「我的」
→ 星澜星历入口
→ 日历视图中看到哪些天点亮了情绪星
→ 点击某天 → 看到当天的牌面/情绪
→ 可能触发"轻记得"——星澜对某天的轻轻回顾
→ 感受：我认真走过的日子被温柔记录了
```

**成败关键**：没有任何连签数字、断签警告、打卡天数。空白日期不制造焦虑。

### 路径 5：星钥等级旅程——"我走了多远"

```
不经意间触发升级判定
→ 升级弹窗出现（安静高级）
→ 展示新称号（如：从"初见"→"星火初亮"）
→ 星澜给出一句对应等级的文案（不说"恭喜"）
→ 用户关闭弹窗
→ 在"我的"页面可以看到当前等级
```

**成败关键**：弹窗是"轻轻的提醒"而非"成就解锁炫耀"。升级文案围绕"陪伴的时间"而非"使用了多久"。

### 路径 6：卡牌探索——"翻翻星钥之书"

```
Tab「卡牌总览」
→ 78 张牌网格/列表展示
→ 点击某张 → 卡牌详情页
→ 正位/逆位 情绪陪伴式解读
→ 感受：像翻看一本安静的图书馆藏书
→ 可能跳转到"用这张牌做个单牌快占"
```

**成败关键**：总览页避免"卖牌货架"感，详情页避免"说明书"感。应是"星钥之书"的翻阅体验。

### 路径 7：星澜语录——"只看一句话"

```
Tab「星澜语录」
→ 随机/分类浏览陪伴文案
→ 1018 条，可上下滑动
→ 用户看到触动的一句话 → 可收藏/可跳转到星澜聊天展开
→ 感受：像翻一本星澜留下的诗集，不要求任何互动
```

**成败关键**：不强制互动，不引导转化。每条文案本身即完整价值。

## 五、设计风险点（7 类高危区域）

### 风险 1：算命感泄漏 ⚠️⚠️⚠️

**风险描述**：塔罗牌天然携带"占卜/预测"的文化符号，任何设计细节都可能将星钥塔罗推向传统算命。

**具体危险信号**：
- 卡牌使用"死神""塔"等传统恐怖意象
- 结果页出现"运势""命运""注定"等词
- 翻开牌的动画使用神秘光效/魔法阵
- 金钱/爱情/事业三段式解读（算命经典套路）

**防范原则**：所有牌面解读必须以"这张牌可能在提醒你……"开头，以"星澜更想陪你看见的是……"收尾。

### 风险 2：恋爱绑定感 ⚠️⚠️⚠️

**风险描述**：星澜是温柔陪伴者，但用户可能将感情投射到星澜上形成依赖。

**具体危险信号**：
- 星澜自称"我"时使用恋爱关系式表达（"我会一直陪着你"）
- 聊天页设计有"亲密度""好感度"等数值
- 星澜使用暧昧/撒娇/恋爱感语气
- 视觉上星澜有强烈的人类拟人化形象（避免乙游感）

**防范原则**：星澜保持"月光下的倾听者"角色边界。视觉呈现上可考虑非人形、光影/符号化表达。

### 风险 3：打卡焦虑 ⚠️⚠️

**风险描述**：情绪陪伴如果变成"每日任务"，将彻底背离产品理念。

**具体危险信号**：
- 连签天数、断签惩罚、签到排行榜
- "你已经 3 天没来了"的推送
- 点亮情绪星显示积分/分数
- 星澜星历空白日期用灰色/警告色

**防范原则**：认真感受过的日子才被标记。空白日是"还没感受的日子"，不是"缺席"。

### 风险 4：AI 工具感 ⚠️⚠️

**风险描述**：用户高度反感 AI 味文案，而星澜恰是本地 AI 驱动，设计需避免"AI 感"。

**具体危险信号**：
- 机械化的"正在分析……""生成中……"
- 回复格式高度统一/模板化
- 使用"根据您的输入，我为您……"的客服腔
- 深色背景 + 绿色代码字体的终端感

**防范原则**：星澜不展示"处理过程"。回复节奏有变化——有时快、有时慢、有时只有一句话。

### 风险 5：博彩/赌场视觉 ⚠️⚠️

**风险描述**：暖金色如果使用不当，极易滑向赌场金/土豪金。

**具体危险信号**：
- 大面积金色渐变 + 发光
- 金色边框/描边过多
- 翻牌动画如老虎机
- 牌背设计如扑克牌

**防范原则**：金色仅作为"月光反射"的点缀——细线、星点、文字高亮，面积不超过 5%。

### 风险 6：强光污染/玻璃拟态过度 ⚠️

**风险描述**：夜间使用场景占比高，光污染直接影响用户舒适度。

**具体危险信号**：
- 大面积纯白/高亮元素
- 玻璃拟态（Glassmorphism）多层叠加
- 荧光霓虹色调
- 动画频率过高（可能触发光敏）

**防范原则**：主背景始终深蓝夜空基调，最亮元素为月光白（非纯白 #FFFFFF），使用暖光而非冷光。

### 风险 7：信息密度失衡 ⚠️

**风险描述**：8 个核心页面承载的功能差异大，信息密度需要差异化处理。

**具体危险信号**：
- 卡牌总览 78 张牌密集排列 → 压迫感
- 聊天页空白过多 → 空洞感
- 星历日历信息过载 → 焦虑感
- 结果页文字过多 → 说教感

**防范原则**：
- 卡牌总览：稀疏网格 + 暗色背景，减少 78 张牌的压迫
- 聊天页：对话式留白，但确保内容区域有足够呼吸感
- 星历：只显示有情绪星的日期，其余淡化到几乎不可见
- 结果页：主牌面 + 3 段短解读（每段 ≤ 3 句），更多内容折叠

## 六、情绪体验目标

### 6.1 核心流程 → 体验目标对应表

| 核心流程 | 进入时情绪 | 完成后目标情绪 | 体验关键动作 |
|---------|----------|-------------|------------|
| 深夜孤独 → 星澜聊天 | 无人可说的孤独 | "我知道你在"——被轻轻接住 | 入口页 2 秒安全信号 → 聊天不催促 |
| 碎片情绪 → 每日一占 | 说不清的烦乱 | "原来是这样"——被具象化理解 | 牌面解读像镜子 → 点亮星星是温柔确认 |
| 关系内耗 → 关系探索 | 反复纠结 | "我的感受是重要的"——被确认 | 不判断对错 → 帮用户看见自己 |
| 回顾 → 星澜星历 | 想看看走过的日子 | "那些日子都被记下了"——温暖但不沉重 | 只标点亮日 → 空白不制造焦虑 |
| 等级触发 → 升级弹窗 | 不经意间触发 | "原来我走了这么久"——安静的自豪 | 轻轻地出现 → 轻轻地离开 |
| 探索 → 卡牌总览 | 好奇/想看看 | "这里有一整本书"——宁静翻阅 | 翻书感 → 可深入可离开 |
| 随时 → 星澜语录 | 只想看一句话 | "这句话好像在说我"——被击中 | 不要求互动 → 每条自足 |

### 6.2 全局体验目标

| 体验维度 | 目标 | 反面 |
|---------|------|------|
| **安全感** | 用户进入 App 后感觉"这里不需要表演" | 需要塑造人设、担心被评判 |
| **被理解** | 星澜说出的比用户写出的更精准 | 感觉星澜在套模板、AI 味 |
| **留白感** | 可以随时离开，不被挽留 | 被推送/红点/任务拉扯 |
| **仪式感** | 认真感受过的日子被轻轻记下 | 打卡压力、错过焦虑 |
| **陪伴感** | 像月光在窗外——在，但不侵入 | 像阳光一样热切、黏腻 |
| **角色边界** | 星澜是倾听者，不是恋人/医生/老师 | 恋爱感、诊断感、说教感 |

## 八、给后续 Phase 的建议

1. **先设计「星澜聊天入口页」和「星澜聊天页」**——这是星澜人格的第一接触点，是整个陪伴体系的"门"和"房间"。其他页面的调性从这两个页面推导。

2. **卡牌结果页需要最严格的文案审计**——这是塔罗文化符号最密集的页面，任何一个词汇偏差都会滑向算命感。

3. **「点亮情绪星成功页」是最容易被低估的页面**——它体积最小，但承载"仪式感但不打卡"这条铁律的核心检验。

4. **星澜星历页的设计是最大挑战**——日历 UI 天然携带"打卡/签到"的符号联想，需要全新的视觉隐喻来打破这一惯性。

5. **不要在设计初期就引入动效方案**——先确定静态页面的情绪基调正确，再考虑动效如何增强而非破坏这种基调。
