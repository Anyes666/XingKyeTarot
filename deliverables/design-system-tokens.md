## 四条修正铁律（全局优先级最高）

> 以下四条规则覆盖所有后续规范。任何与铁律冲突的 Token，以铁律为准。

### 铁律 1：星澜形象规则

- ✅ **允许使用**：星澜头像、月光环、半透明剪影、局部人物特征、星钥/月牙发饰符号
- ❌ **绝对禁止**：全屏恋爱立绘、亲密姿态、暧昧凝视、好感度式角色互动
- **执行原则**：星澜可以有人物形象，但要低恋爱化、低乙游化

### 铁律 2：卡牌总览规则

- ✅ **允许使用**：78 张牌完整开放，可做「已浏览 / 未浏览」「最近查看」「今日相关」区分
- ❌ **绝对禁止**：未解锁、锁图标、付费解锁、解锁卡牌
- **执行原则**：当前 App 没有收藏功能，暂只做完整卡牌总览

### 铁律 3：星钥等级阈值（固定不可更改）

| 天数 | 称号 |
|------|------|
| 0 天 | 初见 |
| 1 天 | 星火初亮 |
| 7 天 | 七日星轨 |
| 21 天 | 月下自省者 |
| 30 天 | 星钥守望者 |
| 90 天 | 星河同行者 |
| 180 天 | 半年听心者 |
| 365 天 | 长夜星澜 |

### 铁律 4：动效性能降级规则

- **标准版**：月光呼吸 + 少量星点 + 涟漪点击反馈
- **降级版**：静态光斑 + 无粒子 + 无连续动画
- **HarmonyOS ArkTS 落地时默认优先稳定**

```
背景层级递进（由深到浅）：
  bg-base (#060B1A ~ #0A0D1A)  →  bg-card  →  bg-floating  →  bg-hover

文字层级递进（由亮到暗）：
  text-primary (#EBE3CD ~ #F2EDDD)  →  text-secondary  →  text-tertiary

金色层级递进（由纯到柔）：
  gold-primary (#C5A050 ~ #D4AC4D)  →  gold-soft  →  gold-dim(透明度)
```

# §2 字体系统

## 2.1 统一基础字体栈

```css
/* ── HarmonyOS 首选字体栈 ── */

/* 无衬线（正文通用） */
--font-sans: "HarmonyOS Sans", "Source Sans 3", "Inter", -apple-system, "PingFang SC", "Microsoft YaHei", sans-serif;

/* 衬线（C 方案标题/卡牌名专用） */
--font-serif: "HarmonyOS Serif", "Source Serif 4", "Noto Serif SC", Georgia, "Songti SC", serif;

/* 等宽（A 方案辅助/时间戳专用） */
--font-mono: "HarmonyOS Sans Mono", "JetBrains Mono", "Cascadia Code", "Consolas", monospace;
```

### 字体可用性兜底策略

| 层级 | 首选 | 系统兜底 | 最终兜底 |
|------|------|---------|---------|
| 无衬线正文 | HarmonyOS Sans | Source Sans 3 / Inter | sans-serif |
| 衬线标题 | HarmonyOS Serif | Source Serif 4 / Noto Serif SC | Georgia / Songti SC |
| 等宽辅助 | HarmonyOS Sans Mono | JetBrains Mono | Consolas / monospace |

> **注意**：若 HarmonyOS 系统无预装衬线中文字体，需将 Noto Serif SC 打包进 App（包体积预估增加约 4MB）。备选方案：用无衬线 Bold + letter-spacing 0.08em 模拟庄重感。

|-----------|---------|-----------|

### 降级检测与恢复

```
App 启动
  ├── 默认降级模式（优先稳定）
  ├── 检测设备性能等级
  │     ├── 高性能 → 切换标准模式
  │     └── 低性能 → 保持降级模式
  ├── 运行时帧率监控
  │     ├── 连续 2s < 45fps → 强制降级
  │     └── 连续 10s > 55fps → 尝试恢复标准
  └── 电量/省电模式监听
        ├── 电量 < 20% 或省电模式 → 降级
        └── 电量 > 30% 且非省电 → 可恢复标准
```

## 10.5 星澜形象使用规范（铁律 1 落地）

### 允许使用的形象元素

| 元素 | 尺寸/位置 | 使用场景 | 样式规范 |
|------|----------|---------|---------|
| **星澜头像** | 40×40（聊天列表）/ 48×48（聊天页） | 聊天页、导航栏 | 圆形裁切，可选月光光环（B 方案） |
| **月光环** | 头像外 12px 扩散 | 星澜正在回复时 | `--halo-avatar-active`，仅表示"正在回复" |
| **半透明剪影** | 最大 120×200 | 入口页装饰 | opacity ≤ 0.3，仅轮廓，无面部细节 |
| **局部人物特征** | 手部/发饰/衣角 | 卡片装饰、空状态 | 线性描边，`--text-secondary` |
| **星钥/月牙发饰** | 16×16 ~ 24×24 | 品牌符号、装饰 | `--gold-primary`，线性图标 |

### 绝对禁止的形象元素

| 禁止元素 | 原因 |
|---------|------|
| 全屏恋爱立绘 | 过度乙游化，偏离情绪陪伴定位 |
| 亲密姿态（拥抱、握手、贴近） | 暗示恋爱关系 |
| 暧昧凝视（直面镜头的深情注视） | 乙游好感度互动感 |
| 好感度式角色互动（心跳、脸红、害羞） | 恋爱模拟机制 |
| 任何暗示恋爱关系的视觉元素 | 违反产品定位 |

### 星澜头像在不同方案中的表现

| 方案 | 头像光环 | 头像边框 | 说明 |
|------|---------|---------|------|
| B（月湖映心） | ✅ 月光光环（标准）/ 金色细边框（降级） | 无 | B 方案可使用光环，但仅表示"正在回复" |
| C（星钥之书） | ❌ 无光环 | 可选 1px 烫金边框 | C 方案保持书页克制感 |
| A（星图档案） | ❌ 无光环 | 无 | A 方案最克制，纯头像 |
