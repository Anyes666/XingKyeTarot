## 1. 行业基准数据

以下基准数据综合自公开行业报告（2024-2026）、应用分析平台数据及同类产品常识推断，作为星钥塔罗指标设定的参考锚点。

### 1.1 留存率基准

| 指标 | 行业类别 | 中位数 | 优秀水平（75分位） | 数据来源 |
|------|---------|--------|-------------------|---------|
| **次日留存（D1）** | 健康与健身 | 25% | 35-45% | UXCam 2026 |
| **次日留存（D1）** | 生活方式/工具 | 25-30% | 40-45% | UXCam 2026 |
| **7日留存（D7）** | 健康与健身 | 10% | 15-22% | UXCam 2026 |
| **7日留存（D7）** | 生活方式/工具 | 8-12% | 18-22% | UXCam 2026 |
| **30日留存（D30）** | 全类别平均 | 4% | 5-8% | UXCam 2026 |

**塔罗/占星类App专项推断**：
- 该类App具有"日常仪式感"特征（每日运势/每日抽牌），天然适合培养使用习惯
- 如果Onboarding成功引导用户完成首次抽牌并理解"每日一牌"价值，D1可期达到 **30-35%**，D7可期达到 **12-16%**
- 但情绪陪伴类App存在"新鲜感衰退"挑战，D30通常回落至 **5-8%**

### 1.2 分享功能使用率基准

| 指标 | 基准范围 | 说明 |
|------|---------|------|
| 分享功能触达率（看到分享按钮的用户占比） | 60-80% | 若入口设计明显，大多数用户会注意到 |
| 分享行为发生率（生成过分享卡的用户/活跃用户） | 5-12% | 生活方式类App中位数；优秀App可达15-20% |
| 分享卡保存成功率（保存成功/点击保存） | 85-95% | 取决于权限申请流程和设备兼容性 |
| 单用户月均分享次数 | 1.2-2.5次 | 高频分享通常来自核心用户群 |

**塔罗场景特别说明**：塔罗/占卜类App的分享动机天然较强——用户倾向于在社交媒体展示"今日运势"或"抽到的牌"，参考同类产品（如Co-Star、The Pattern），优质产品的分享率可达 **15-20%**。

### 1.4 应用市场转化率基准

| 漏斗阶段 | 行业基准 | 数据来源 |
|---------|---------|---------|
| **有效曝光点击率**（应用市场搜索结果/推荐位 → 详情页） | 6-12% | 通用ASO基准 |
| **详情页转化率**（详情页 → 下载） | 23-35% | 生活方式类App Store 23.3%；工具类 40.6%；华为市场建议 >35% |
| **安装成功率**（下载 → 成功安装打开） | 90-95% | 华为AGC基准 |
| **整体漏斗**（曝光 → 安装打开） | 1.5-4% | 综合漏斗 |

**华为应用市场专项说明**：
- 华为AGC（AppGallery Connect）为开发者提供**同品类Benchmark数据**，可直接对比曝光点击率、详情页转化率、安装成功率
- 华为应用市场定位为"品质应用平台"，用户下载决策相对谨慎，详情页转化率通常低于Google Play 3-5个百分点
- 评分低于3.5星的应用，详情页转化率会显著下降（降幅可达40-60%）

### 1.5 隐私协议同意率基准

| 设计模式 | 同意率 | 说明 |
|---------|-------|------|
| **无拒绝选项**（仅"同意"按钮） | 90-95% | ⚠️ 不推荐，正被逐步禁止 |
| **弹窗 + 明确拒绝按钮** | 72-78% | ✅ 推荐，合规 |
| **弹窗 + "不同意但继续"链接** | 77-85% | ✅ 推荐 |
| **页脚式 + "继续不同意"链接** | 85-86% | 用户习惯性忽略 |
| **App场景（非Web Cookie）** | 80-92% | App场景下用户对隐私弹窗的容忍度更高 |

**特别提示**：星钥塔罗作为**完全离线本地运行**的App，隐私协议同意率预期应**显著高于行业均值**。因为：
1. 用户无需担心数据上传/分享
2. "本地处理"是一个强信任信号
3. 可在隐私协议中突出"无需联网、数据仅存本地"的差异化卖点

**预期基准：85-92%**

### 2.1 下载量

| 维度 | 内容 |
|------|------|
| **为什么重要** | 最基础的漏斗入口指标。没有下载就没有后续一切行为。对于新上架App，下载量直接反映"是否存在PMF的微弱信号" |
| **当前如何估算** | ① 华为AGC后台「分发分析 → 下载安装」可直接查看「新下载成功次数」；② 结合抖音推广内容的曝光量×行业CPM转化率估算（视频曝光→详情页点击约0.5-1.5%，详情页→下载约25-35%） |
| **未来如何采集** | AGC后台持续监控，分渠道（自然搜索 vs 推广位 vs 外部链接） |
| **健康阈值** | 新上架首月：**>100次/月**（有机下载） |
| **预警阈值** | **<30次/月** 持续3个月 → 需评估ASO策略和推广投入 |
| **优化方向** | ASO优化（标题含"塔罗/占卜/情绪"关键词）、华为应用市场CPD投放、抖音/小红书内容引流 |

### 2.2 首次打开率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 衡量"下载即流失"问题。部分用户下载后从不打开，说明应用市场详情页与真实体验的期望差过大，或安装过程存在摩擦 |
| **当前如何估算** | ① AGC后台「新下载成功次数」与预发布测试中的首次打开计数对比；② 小范围用户访谈询问"下载后多久打开" |
| **未来如何采集** | 如有本地计数：应用首次启动时写入flag；无埋点：AGC后台DAU/下载量比值粗略估算 |
| **健康阈值** | **>90%**（大部分应用接近100%） |
| **预警阈值** | **<80%** → 说明有严重的安装/启动问题（权限弹窗过多、启动崩溃、图标无吸引力） |
| **优化方向** | 检查安装包完整性、减少启动权限请求、优化应用图标（增加辨识度） |

### 2.3 隐私协议同意率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 用户对隐私安全的信任投票。对于强调"离线/本地/无数据收集"的App，此指标也是"产品差异化承诺"是否被用户感知到的检验 |
| **当前如何估算** | ① 用户访谈中询问"看到隐私协议时的第一反应"；② 预发布测试中模拟首次启动流程，记录同意/拒绝比例 |
| **未来如何采集** | 本地计数器：同意次数 / 隐私页展示总次数 |
| **健康阈值** | **>80%**（行业App场景平均72-92%，离线App应偏高） |
| **预警阈值** | **<65%** → 协议文字太长/过于法律化/用户不信任 |
| **优化方向** | 精简协议文案（控制在3屏内）、在首页用更大字号强调"所有数据仅存储在您的设备上"、增加可视化图标说明 |

### 2.4 Onboarding完成率

| 维度 | 内容 |
|------|------|
| **为什么重要** | Onboarding是用户从"陌生人"变成"理解产品价值的人"的桥梁。行业数据显示，完成Onboarding的用户其D30留存率是未完成者的 **2-3倍** |
| **当前如何估算** | ① 预发布测试：邀请5-10名目标用户走完整Onboarding流程，计时+观察+事后访谈；② 计算完成率并标记各步骤流失点 |
| **未来如何采集** | 本地计数器：Onboarding开始人数 / 完成最后一步人数。每步均设置计数，定位最大流失步骤 |
| **健康阈值** | **>60%**（行业平均激活率36%，但优秀Onboarding可达60%+） |
| **预警阈值** | **<40%** → Onboarding步骤过多/复杂/用户感受不到价值 |
| **优化方向** | 控制在3-5步内、每步传递一个明确价值点（"今天的心情，一张牌就能读懂"）、步骤间流失>30%需单独排查 |

### 2.5 首次抽牌完成率

| 维度 | 内容 |
|------|------|
| **为什么重要** | **这是整个产品的"啊哈时刻"**。用户第一次抽到牌并看到解读，是产品核心价值兑现的瞬间。此指标直接预测D1留存 |
| **当前如何估算** | 预发布测试中观察：Onboarding结束→点击"抽牌"→看到结果的全流程完成率 |
| **未来如何采集** | 本地计数器：进入抽牌页 / 抽牌结果展示页 |
| **健康阈值** | **>75%**（Onboarding完成者中） |
| **预警阈值** | **<50%** → 抽牌流程存在交互障碍（动画太慢、按钮不明显、加载出错） |
| **优化方向** | 减少抽牌到结果的等待时间、确保动画流畅、加微交互反馈（震动/音效） |

### 2.6 今日星钥点击率

| 维度 | 内容 |
|------|------|
| **为什么重要** | "今日星钥"是产品设计的**日活引擎**——每天更新的运势/牌面推荐。此功能点击率直接反映用户是否养成了"每天来App看一眼"的仪式感习惯 |
| **当前如何估算** | ① 用户访谈中询问"如果App每天给你推送一张新牌，你每天会来看吗"；② 预发布测试中观察用户对"今日"入口的注意程度 |
| **未来如何采集** | 本地计数器：首页「今日星钥」入口展示次数 / 入口点击次数 |
| **健康阈值** | **>55%**（活跃用户中） |
| **预警阈值** | **<30%** → 入口不够显眼、内容吸引力不足、用户对"每日更新"无感知 |
| **优化方向** | 增大首页入口视觉权重、考虑桌面卡片同步每日牌面、增加"连续打卡"激励 |

### 2.7 单牌快占完成率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 最轻量的塔罗占卜体验——1张牌+1句话解读。是降低使用门槛、提高使用频率的关键功能。高完成率表明产品满足了"快速情绪安慰"的场景需求 |
| **当前如何估算** | 预发布测试：观察用户从首页→选择快占→看到单牌结果的完成路径流畅度 |
| **未来如何采集** | 本地计数器：点击"单牌快占"→ 看到结果页 |
| **健康阈值** | **>50%**（活跃用户中使用过） |
| **预警阈值** | **<25%** → 入口位置不佳、用户不理解"快占"含义、解读文案吸引力弱 |
| **优化方向** | 优化功能命名（如"快速抽一张"）、入口放首页首屏、解读文案增加共鸣感 |

### 2.8 圣三角使用率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 经典3张牌阵，代表用户深度使用意愿。使用此功能的用户通常有较强的塔罗知识或更复杂的情绪问题需要探索，是**核心用户识别信号** |
| **当前如何估算** | 用户访谈中了解目标用户对"多张牌阵"的认知和兴趣度；预发布测试观察是否有用户主动发现此功能 |
| **未来如何采集** | 本地计数器：进入圣三角页面 / 完成3张牌展示 |
| **健康阈值** | **>10%**（活跃用户中使用过） |
| **预警阈值** | **<3%** → 功能隐藏太深、引导不足、用户不理解牌阵价值 |
| **优化方向** | 在单牌解读结果页添加"想深入了解？试试圣三角牌阵"的引导链接、增加牌阵教学引导 |

### 2.9 关系探索使用率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 关系主题是塔罗占卜的高频场景（感情/人际/家庭）。此功能的渗透率反映了产品是否覆盖了用户最关心的情感维度 |
| **当前如何估算** | 用户访谈中了解目标用户最关心的占卜主题（感情、事业、健康等），评估关系主题的优先级 |
| **未来如何采集** | 本地计数器：进入关系探索页面 / 完成流程 |
| **健康阈值** | **>8%**（活跃用户中使用过） |
| **预警阈值** | **<2%** → 需求不匹配或入口隐蔽 |
| **优化方向** | 在首页增加场景化入口（如"关于TA的困惑？"）、结合情绪星点推荐关系主题 |

### 2.10 追问星澜点击率

| 维度 | 内容 |
|------|------|
| **为什么重要** | "追问"是产品**陪伴深度**的体现——用户从"看完解读"到"想进一步追问"，说明解读内容引发了真实的情感共鸣。此指标反映了AI解读质量和用户信任度 |
| **当前如何估算** | 预发布测试：观察用户看完解读后的反应——是否有用户主动问"还能继续问吗？" |
| **未来如何采集** | 本地计数器：解读结果页「追问星澜」按钮展示次数 / 点击次数 |
| **健康阈值** | **>25%**（看到解读结果页的用户中） |
| **预警阈值** | **<10%** → 解读内容质量不足、追问按钮不显眼、用户不确定追问是否免费/有效 |
| **优化方向** | 优化解读文案深度和个性化程度、在解读结尾添加引导性提问提示、明确标注"免费追问" |

### 2.11 情绪星点点击率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 情绪星点是用户**情感状态的轻量记录入口**。此功能既帮助用户自我觉察，也为产品理解用户情绪状态提供信号。点击率反映用户对"情绪工具"属性的认可度 |
| **当前如何估算** | 用户访谈中询问"你会用什么方式记录自己的情绪状态？"；预发布测试观察情绪星点的视觉吸引力 |
| **未来如何采集** | 本地计数器：情绪星点入口展示次数 / 点击次数 |
| **健康阈值** | **>20%**（活跃用户中） |
| **预警阈值** | **<8%** → 入口设计不够情感化、用户不理解功能价值 |
| **优化方向** | 增加emoji/色彩情绪表达、与每日抽牌结果联动（"根据你今天的情绪星点，为你推荐这张牌"） |

### 2.12 分享卡生成率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 分享是**零成本增长引擎**。精美的塔罗牌面+解读文案天然适合社交分享。高分享率意味着产品具备自传播能力，是低成本获客的关键杠杆 |
| **当前如何估算** | ① 用户访谈中展示分享卡样式，询问"你会愿意把这个发到朋友圈/小红书/抖音吗？"（意愿测试）；② 预发布测试观察用户是否有分享冲动 |
| **未来如何采集** | 本地计数器：点击"生成分享卡" / 活跃用户数 |
| **健康阈值** | **>10%**（活跃用户中生成过分享卡） |
| **预警阈值** | **<3%** → 分享卡设计不够精美、分享流程繁琐、生成后不知道如何引导分享 |
| **优化方向** | 优化卡片视觉设计（品牌水印、牌面艺术感）、一键跳转微信/朋友圈/小红书、增加分享后的正向反馈（如"你的朋友也在看这张牌"） |

### 2.13 保存图片成功率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 分享流程的"最后一公里"。用户点击保存→成功存储到相册，中间可能因权限问题、存储空间不足等原因失败。失败率过高会直接扼杀分享行为 |
| **当前如何估算** | 预发布测试：在不同HarmonyOS版本设备上测试保存流程，记录成功/失败 |
| **未来如何采集** | 本地计数器：点击"保存图片" / 系统返回保存成功 |
| **健康阈值** | **>90%** |
| **预警阈值** | **<75%** → 系统权限兼容性问题、存储路径异常 |
| **优化方向** | 预先检查并引导授予存储权限、增加保存失败后的引导提示、适配不同HarmonyOS版本 |

### 2.14 历史记录打开率

| 维度 | 内容 |
|------|------|
| **为什么重要** | 历史记录反映了用户对"回顾与反思"的需求。塔罗用户常有回看"上次抽到了什么牌"的习惯。此指标也是用户对产品数据持久化能力的信任投票 |
| **当前如何估算** | 用户访谈中询问"你希望保留之前的抽牌记录吗？多久回看一次？" |
| **未来如何采集** | 本地计数器：历史记录入口展示次数 / 点击进入次数 |
| **健康阈值** | **>20%**（活跃用户中） |
| **预警阈值** | **<8%** → 入口不显眼、历史记录展示方式缺乏吸引力 |
| **优化方向** | 增加"时间线"式展示（情绪变化曲线+对应牌面）、在首页展示"上次抽牌回顾"小卡片 |

### 2.15 桌面卡片添加率

| 维度 | 内容 |
|------|------|
| **为什么重要** | HarmonyOS的桌面卡片能力是**留存护城河**——用户每次解锁手机都能看到牌面/运势，大幅降低"忘记这个App"的概率。研究显示52%的用户认为Widget对快速获取信息有价值 |
| **当前如何估算** | 预发布测试中引导用户添加卡片，观察成功率；用户访谈中展示卡片样式，询问添加意愿 |
| **未来如何采集** | 本地计数器：引导添加卡片弹窗展示次数 / 用户确认添加（通过卡片更新回调） |
| **健康阈值** | **>5%**（活跃用户中已添加） |
| **预警阈值** | **<1%** → 引导不足、卡片设计不够吸引人、用户不理解卡片价值 |
| **优化方向** | 在Onboarding最后一步引导添加、卡片设计展示每日牌面+简短解读、卡片定期自动刷新 |

### 2.16 次日留存（D1 Retention）

| 维度 | 内容 |
|------|------|
| **为什么重要** | D1留存是产品价值的**第一道检验**——用户今天用了，明天还会来吗？行业数据表明，D1是D30留存的最强预测指标 |
| **当前如何估算** | ① 预发布测试后，间隔24小时联系测试用户询问"今天还想打开吗"；② AGC后台可查看新增用户留存（若有集成HMS Core） |
| **未来如何采集** | 如有本地计数：记录启动日期，计算相邻两日均有启动的用户占比；无埋点：通过AGC分析服务的留存报表 |
| **健康阈值** | **>30%**（生活方式类D1中位数25-30%，塔罗应偏高） |
| **预警阈值** | **<20%** → 首次体验未击中"啊哈时刻"、Onboarding失败、无次日回访动机 |
| **优化方向** | 强化"每日一牌"的仪式感设计、桌面卡片提醒、首次抽牌后给出"明天同一时间，星钥会为你准备好新的牌面"的预期 |

### 2.17 7日留存（D7 Retention）

| 维度 | 内容 |
|------|------|
| **为什么重要** | D7留存是"习惯养成"的初步信号。对于"每日仪式感"定位的产品，7天内至少回来2-3次的用户才算进入了使用节奏 |
| **当前如何估算** | 暂无有效方法（周期太长，预发布测试无法覆盖）；可参考同类产品公开数据推断 |
| **未来如何采集** | AGC分析服务留存报表 / 本地每周启动计数 |
| **健康阈值** | **>12%**（健康与健身类D7中位数10%，塔罗应介于健身与社交之间） |
| **预警阈值** | **<6%** → 产品缺乏"每日回来"的理由、新鲜感快速消退 |
| **优化方向** | 每日内容差异化（不同牌面/不同解读角度）、连续使用激励、周度回顾（"本周你的情绪轨迹"） |

### 2.18 用户评论关键词

| 维度 | 内容 |
|------|------|
| **为什么重要** | 在当前无埋点阶段，**用户评论是最重要的定性数据来源**。评论关键词直接反映用户对产品价值的认知和情感连接。也是ASO权重因子 |
| **当前如何估算** | ① 华为应用市场评论区逐条阅读+关键词标注（正向：准/好看/治愈/好玩；负向：不准/无聊/闪退/广告多）；② 微博/小红书搜索"星钥塔罗"提取UGC内容 |
| **未来如何采集** | 持续人工标注+简易NLP分类（正向/中性/负向），按周追踪情感分数变化 |
| **健康阈值** | ① 评分 > 4.0星；② 正向关键词占比 > 60%；③ "治愈/精准/好看"等核心价值词出现频次 > 30% |
| **预警阈值** | ① 评分 < 3.5星；② 负向关键词 > 30%；③ 出现"闪退/卡顿/不准"等体验类负面词集中爆发 |
| **优化方向** | 针对负面关键词定位问题（如"不准"→优化解读模型和prompt、"闪退"→修复崩溃）、在评论回复中引导用户更新版本 |

### 2.19 抖音引流效果

| 维度 | 内容 |
|------|------|
| **为什么重要** | 塔罗/玄学/情绪类内容的短视频传播力极强。抖音是此类产品最重要的外部流量来源之一。此指标衡量内容营销的投入产出比 |
| **当前如何估算** | ① 抖音创作者后台查看视频播放量、互动量（点赞/评论/分享/收藏）；② 华为AGC后台按渠道查看下载来源；③ 在抖音评论区/简介中使用短链或引导搜索词，间接追踪引流效果 |
| **未来如何采集** | AGC分发渠道归因；抖音企业号后台「转化追踪」；HarmonyOS支持Deferred Deep Link时实现安装归因 |
| **健康阈值** | ① 单条爆款视频 → 带来50+下载；② 抖音渠道下载占总下载 > 15%；③ 视频互动率 > 3%（点赞+评论+收藏/播放量） |
| **预警阈值** | ① 月均抖音内容 < 4条 → 内容供给不足；② 视频平均播放 < 500 → 内容质量或选题问题；③ 抖音渠道下载占比 < 5% → 引流链路断裂 |
| **优化方向** | 优化视频选题（"测测你今天运势"、"塔罗告诉你TA怎么想"等情绪钩子）、在视频中展示精美牌面和解读效果、评论区置顶搜索引导词 |

### 2.20 应用市场搜索转化

| 维度 | 内容 |
|------|------|
| **为什么重要** | 华为应用市场内的搜索是**最高意图的流量**——用户已经知道自己想要什么。搜索转化率直接影响自然获取用户效率，也是ASO策略有效性的核心检验 |
| **当前如何估算** | ① AGC后台「分发分析→搜索分析」查看搜索曝光次数和搜索下载次数；② 计算：搜索下载/搜索曝光 |
| **未来如何采集** | AGC搜索分析报表持续监控；分关键词追踪转化率 |
| **健康阈值** | ① 搜索→详情页点击率 > 8%；② 详情页→下载转化率 > 30%；③ 搜索词覆盖 > 10个相关关键词（塔罗、占卜、运势、星座、情绪、治愈...） |
| **预警阈值** | ① 详情页转化率 < 15% → 应用详情页质量差（截图/描述/评分不足以说服用户）；② 搜索词覆盖 < 5个 → ASO关键词策略严重不足 |
| **优化方向** | 优化应用标题（含核心关键词）、应用截图前三张展示核心体验、争取早期优质评论提升评分 |

## 3. 数据采集策略建议

### 3.1 第一阶段：无埋点时代（当前 — v1.0.0 上架后）

在当前完全离线本地运行、无任何埋点SDK的情况下，数据采集依赖 **外部可触达的信号源**：

#### 3.1.1 华为应用市场后台（AGC）— 核心数据源

| 可获取指标 | 报表位置 | 对应框架指标 |
|-----------|---------|-------------|
| 新下载成功次数、更新下载次数 | 分发分析 → 下载安装 | #1 下载量 |
| 有效曝光次数、有效曝光点击率 | 分发分析 → 下载安装 | #20 搜索转化 |
| 详情页转化率 | 分发分析 → 下载安装 | #20 搜索转化 |
| 安装成功率 | 分发分析 → 下载安装 | #2 首次打开率 |
| 卸载次数 | 分发分析 → 下载安装 | 间接反映流失 |
| 新增用户留存（D1/D7/D30） | 运营分析 → 用户维系（需集成HMS Core基础分析） | #16 #17 |
| 评分与评论 | 质量分析 → 评分评论 | #18 评论关键词 |
| 同行业Benchmark对比 | 行业风向标 | 所有基准对标 |

**关键动作**：
- 每周五下午固定查看AGC后台，记录关键指标变化
- 每新增100条评论时做一次关键词情感分析
- 在AGC中确认已开启"同行业Benchmark"对比功能

#### 3.1.2 用户评论分析 — 定性核心数据源

```
频率：每周一次
方法：
1. 导出华为应用市场全部评论
2. 人工标注每条评论：情感（正向/中性/负向）+ 关键词标签
3. 使用简易标签体系：
   - 体验类：界面/动画/音效/流畅度
   - 内容类：准确/不准/有趣/无聊
   - 功能类：闪退/卡顿/功能缺失/想要XX功能
   - 情感类：治愈/安心/焦虑/失望
4. 周报呈现关键词云 + 情感分数变化趋势
```

#### 3.1.3 小范围用户访谈（10-15人/轮）

```
频率：每2-3周一轮
方法：
1. 招募目标用户（HarmonyOS设备 + 对塔罗/情绪陪伴有兴趣）
2. 观察式测试：用户首次打开App，不给予任何指导，观察自然行为
3. 关键提问：
   - "你看到隐私协议时的第一反应？"
   - "Onboarding结束后，你第一件事想做什么？"
   - "你会把这个App推荐给朋友吗？为什么？"
   - "如果你明天还会打开，是什么原因？"
4. 记录：
   - Onboarding每步耗时
   - 首次抽牌到看到结果的时间
   - 各类功能被"自然发现"的概率
   - NPS评分
```

#### 3.1.4 社交媒体监听

```
频率：每周一次
渠道：
- 抖音：搜索"星钥塔罗"相关视频，记录播放量、互动量、评论区关键词
- 小红书：搜索"星钥塔罗"笔记，记录情感倾向
- 微博：品牌词搜索
- 华为应用市场评论区

记录指标：
- 自然UGC数量（用户自发分享）
- UGC情感分布
- 高频关联词（如"准"、"不准"、"好看"、"治愈"）
```

#### 3.1.5 预发布测试（QA环境）

```
范围：5-10名内部/亲友测试用户
设备：覆盖不同HarmonyOS版本（2.0/3.0/4.0）
记录：
- 首次启动到Onboarding完成的耗时和完成率
- 首次抽牌完成率
- 各功能模块的"自然发现率"（不给予任何提示）
- 分享卡生成→保存到相册的全流程完成率
- 发现的Bug和体验问题
```

### 3.2 第二阶段：轻量本地埋点方案（可选未来）

如果需要更精确的行为数据，可以在**完全不联网**的前提下，在本地App内增加最小可行的行为计数方案：

#### 3.2.1 设计原则

> ⚠️ **硬约束**：不联网、不上传、不引入任何第三方SDK、所有数据仅存储在用户设备本地。

#### 3.2.2 最小可行埋点方案

```
数据存储：HarmonyOS Preferences（轻量KV存储）/ 本地JSON文件
数据结构：事件名 + 计数 + 时间戳

埋点事件清单（精简到15个以内）：

启动事件：
  app_launch              — 应用启动次数
  app_first_launch        — 首次启动（写入后不再更新）

漏斗事件：
  privacy_show            — 隐私协议展示
  privacy_accept          — 隐私协议同意
  onboarding_start        — Onboarding开始
  onboarding_complete     — Onboarding完成
  first_draw_start        — 首次抽牌开始
  first_draw_complete     — 首次抽牌完成

功能使用事件：
  daily_key_click         — 今日星钥点击
  quick_draw_complete     — 单牌快占完成
  saint_triangle_use      — 圣三角使用
  relation_explore_use    — 关系探索使用
  deep_ask_click          — 追问星澜点击
  emotion_star_click      — 情绪星点点击

分享事件：
  share_card_generate     — 分享卡生成
  share_card_save_success — 保存成功
  share_card_save_fail    — 保存失败

留存事件：
  history_open            — 历史记录打开
  widget_add              — 桌面卡片添加
```

#### 3.2.3 数据导出方式

```
方案A：设置页隐藏入口 → 导出JSON到本地文件 → 用户手动发送给开发者（仅限自愿参与的用户测试）
方案B：通过HarmonyOS应用内"意见反馈"入口 → 用户可选择附带使用数据
方案C：定期测试版本 → 预置数据导出到指定目录 → QA手动收集
```

#### 3.2.4 隐私合规要点

- 在隐私协议中明确说明"本地计数数据的内容、目的、存储方式"
- 提供"关闭数据收集"的开关（默认开启）
- 所有数据导出必须由用户主动触发，不可静默自动执行
- 标注"此数据仅用于产品体验优化，不会离开您的设备（除非您主动分享）"
