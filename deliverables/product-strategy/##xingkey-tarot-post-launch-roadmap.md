## 1. 当前阶段判断

### 1.1 核心诊断

星钥塔罗已于华为应用市场上架 v1.0.0。基于三份研究输入的交叉分析，当前阶段判断如下：

| 判断 | 依据 | 置信度 |
|------|------|:---:|
| **定位正确但落地度约60%** | 竞品分析显示"不是算命"定位在市场唯一，但牌义文本仍有预测色彩、无心理学框架背书、品牌话术中"情绪陪伴"不突出 | ⭐⭐⭐⭐⭐ |
| **当前最大短板是内容深度和留存机制** | 用户研究指出留存主战场在D1→D7；竞品塔罗气泡68%留存靠内容+CBT行动指引；星钥当前牌义基础、无行动建议、追问缺乏情绪差异化 | ⭐⭐⭐⭐⭐ |
| **华为生态先发红利窗口仍在** | 竞品分析确认HarmonyOS塔罗类App数量明显更少，桌面卡片、元服务等鸿蒙特色交互竞品无法直接复制；但窗口不会永远开放 | ⭐⭐⭐⭐ |
| **合规是最大红利而非包袱** | 竞品分析显示测测861条投诉、315曝光、监管收紧；星钥离线+无付费+无账号天然合规，应主动放大此优势 | ⭐⭐⭐⭐⭐ |
| **不应被"做大做全"诱惑** | 测测6000万用户但合规风险极高；预见塔塔功能丰富但解读空泛。星钥当前最大优势恰恰是"克制"——无账号、无付费、无广告、离线运行 | ⭐⭐⭐⭐⭐ |
| **D1→D7是唯一留存战场** | 用户研究指出情绪类App天然"好了就不来了"；数据报告显示D30暴跌至4-5%。首要目标是把用户留在D1→D7区间，而非追求D30 | ⭐⭐⭐⭐⭐ |

### 1.2 一句话判断

> 星钥塔罗当前最需要做的不是增加功能，而是把"情绪陪伴"定位从60分拉到90分——让每一个已经在用的人感受到"星澜真的懂我"，并因此愿意明天再来。

## 4. v1.0.1 稳定性补丁

### 4.1 定位说明

v1.0.1是上架后的首个补丁版本，**唯一目标是修复已发现的问题**，不使用户可感知的新功能。这是用户信任的第一道关卡——上架后如果出现明显Bug而不快速修复，将直接打击留存和评分。

### 4.2 应观察的用户反馈

以下反馈应在上架后第一周密切监控：

| 反馈类型 | 监控渠道 | 响应优先级 |
|----------|----------|:---:|
| 闪退/崩溃报告 | 华为AGC质量分析 → 崩溃率 | P0 |
| 应用无法启动 | 华为应用市场评论区 | P0 |
| 白边/黑边/沉浸式问题 | 评论区 + 用户访谈 | P0 |
| 隐私协议看不懂 | 评论区 + 用户访谈 | P0 |
| 分享卡保存失败 | 评论区 | P1 |
| 桌面卡片不更新 | 评论区 | P1 |
| "不准"类评价 | 评论区 | P2（非v1.0.1修复，记录为内容改进需求） |
| "功能太少"类评价 | 评论区 | P2（非v1.0.1修复，记录为路线图参考） |

## 5. v1.1.0 情绪星轨留存计划

### 5.1 核心定位

v1.1.0的目标是把情绪星点从"结果页附属入口"变为"用户每天主动回来的钩子"。这是留存战役的核心阵地。

### 5.2 用户价值主张

> 不是打卡，是每天给自己一个被理解的时刻。7天后，你会看见自己的情绪地图。

### 5.3 每天点亮机制

#### 5.3.1 点亮触发

| 触发条件 | 点亮行为 |
|----------|----------|
| 完成任意一种抽牌（单牌快占/圣三角/关系探索/今日星钥） | 自动点亮当天情绪星，无需用户额外操作 |
| 用户当天已点亮 | 不重复点亮，显示"今日已点亮 ✨" |
| 当天未打开App但桌面卡片有展示 | 不自动点亮（只在用户主动进入App时点亮） |

#### 5.3.2 点亮视觉

- 当天未点亮：灰色星点（半透明）
- 当天已点亮：金色星点（带微光辉动效）
- 连续2天+：星点间出现金色星轨连线
- 第6天：星点出现特殊微动效提示"明天就满一周了"
- 第7天：全部星点完成跃动动画，生成7日回顾入口

#### 5.3.3 点亮后的引导

| 用户状态 | 引导内容 |
|----------|----------|
| 第一次点亮 | "你留下了一颗星。明天它还会在这里等你。" |
| 连续3天 | "星澜看见你连续来了三天——你正在养成一个温柔的习惯。" |
| 连续6天 | "明天就满一周了。星澜会为你准备一份情绪回顾。" |
| 连续7天 | "七天，七颗星。来看看这周你留下了什么。" → 跳转7日回顾 |
| 断签后重新点亮 | "欢迎回来。中间断了的那些天也没关系——星澜更在乎你回来了。" |

### 5.4 今日星钥联动

今日星钥是D1留存的核心引擎。v1.1.0中：

| 联动点 | 行为 |
|--------|------|
| 用户打开App首页 | 今日星钥卡片醒目展示今日牌面缩略图 + 一句话情绪引导 |
| 用户点击今日星钥 | 进入当日牌面详情 + 自动触发情绪星点点亮 |
| 今天已完成今日星钥 | 再次点击显示"今天的指引已送达，你随时可以重新看" |
| 桌面卡片 | 同步展示今日星钥牌面缩略图 + "今天星澜想对你说..."引导文案 |

### 5.5 7日轨迹展示

#### 5.5.1 展示页面

从情绪星点入口进入，新增「7日情绪星轨」子页（或在现有情绪星点页升级）。

#### 5.5.2 展示内容

| 元素 | 数据来源 | 展示方式 |
|------|----------|----------|
| 7天星轨图 | 7天内每天是否点亮 | 水平星轨线，已点亮的金色星连线，未点亮灰色 |
| 每天抽到的牌 | 当天第一次抽牌记录 | 每颗星点上方悬浮该日首张牌的缩略图 |
| 最常出现的情绪主题 | 7天内追问/选择中出现的情绪标签 | 一句话总结"本周你反复在思考..." |
| 星澜一句话 | 基于本周数据生成 | "星澜看见：你这周比上周更愿意表达自己的真实感受" |

#### 5.5.3 不展示的内容

- 不做心理诊断（"你有抑郁倾向"等）
- 不做情绪等级评定（"本周焦虑指数7.5"）
- 不做与别人的对比（"你比85%的用户更焦虑"）
- 不预测下周（"下周运势将会..."）

### 5.6 断签温柔处理

| 用户行为 | 产品响应 |
|----------|----------|
| 断签1-3天后回来 | "星澜看见你回来了。中间空了的星不是缺陷——是你在忙，或在休息。" |
| 断签4-6天后回来 | 正常点亮今天的星，之前空白的星保持灰色（可追溯） |
| 断签7天+后回来 | 重新开启新一轮7天周期，上一轮数据作为"历史星轨"可查看 |
| 连续多次断签 | 不做任何负面反馈——永远欢迎，永远不催促 |

**核心原则**：断签不是用户的错，是产品还不足够让用户每天都想回来。

### 5.7 历史记录联动

| 联动点 | 实现 |
|--------|------|
| 7日星轨中每颗星 | 点击可跳转到该日最早一次抽牌的历史详情 |
| 历史详情中有情绪星点标记 | 每条记录标记"⭐ 那天你点亮了一颗星" |

## 6. v1.2.0 内容厚度升级

### 6.1 核心定位

v1.2.0的目标是让用户**多次使用仍然有新鲜感**，并且每次使用都感受到"星澜真的懂我"。

### 6.2 为什么优先22张大阿卡纳而不是78张

| 理由 | 说明 |
|------|------|
| **成本可控** | 22张牌×每张牌4个场景维度×每种≥2条追问 = 约200条内容，可在2-4周内高质量完成。78张牌同样的深度需要至少3倍时间 |
| **大阿卡纳是用户高频感知的核心** | 单牌快占、今日星钥、圣三角（3张）中，大阿卡纳的出现概率远高于小阿卡纳，用户对大牌的情感连接更强 |
| **"少而精"远胜"多而泛"** | 预见塔塔的教训：解读空泛比解读少更致命。22张深度牌义的留存价值远大于78张表面牌义 |
| **荣格分析心理学主要对应大阿卡纳** | 大阿卡纳的"愚人之旅"与荣格"个体化进程"高度对应，心理学框架背书天然聚焦于22张大牌 |
| **78张留到v2.0+** | 当D7留存稳定>12%、MAU稳定>2000时，再考虑扩展小阿卡纳 |

### 6.3 每张牌字段设计

每张大阿卡纳牌（以"愚人"为例）应包含以下字段：

| 字段 | 内容说明 | 示例（愚人 The Fool） |
|------|----------|----------------------|
| **牌名** | 中文牌名 + 英文名 | 愚人 · The Fool |
| **牌面意象** | 2-3句描述牌面视觉元素，帮助用户建立画面感 | "悬崖边一个背着行囊的人正要迈出脚步，身旁有一只小狗在提醒什么。日光正亮，前路未知。" |
| **荣格心理原型** | 该牌对应的荣格原型概念（v1.2.0新增） | 孩童原型 / 全新开始的原型——卡尔·荣格 |
| **核心隐喻** | 一句话提炼本牌的心理学隐喻 | "这不是无知者的莽撞，而是相信未来的人敢于踏出第一步。" |
| **情绪场景解读** | 当用户情绪低落/焦虑/迷茫时，这张牌在对TA说什么 | **[见6.4节]** |
| **关系场景解读** | 当用户关注感情/人际关系时 | **[见6.4节]** |
| **选择场景解读** | 当用户面临选择/决策困难时 | **[见6.4节]** |
| **行动场景解读** | 从牌义延伸出低压力、可执行的行动建议 | **[见6.4节]** |
| **正位/逆位差异** | 核心解读的不同倾向（非命运断言） | 正位：勇敢出发的能量被打开；逆位：提醒你在冲动前先看看脚下 |
| **"星澜的一小步"** | 基于本牌的行动建议（固定2-3句） | "如果最近有什么你一直想做但犹豫的事，今天可以先花10分钟写下第一步。" |
| **共时性提问** | 引导用户思考的开放式问题（v1.2.0新增） | "如果此刻的你不是在害怕结果——你最想开始的改变是什么？" |

### 6.4 区分四个场景维度

每个场景维度的解读文案必须**从该场景的视角出发**，而非同一段牌义的四种改写。

#### 情绪场景解读

| 要素 | 要求 |
|------|------|
| 目标 | 让焦虑/迷茫/低落的用户感到被接住 |
| 话术方向 | "当你觉得______的时候，这张牌可能在提醒你______" |
| 关键词 | 理解、允许、存在、慢慢来 |
| 示例（愚人） | "当你觉得憋了很久想挣脱什么的时候——这张牌像在轻轻推你一下：恐惧本身不是问题，让恐惧决定你每一步才是。" |

#### 关系场景解读

| 要素 | 要求 |
|------|------|
| 目标 | 让纠结关系的用户看见自己的真实期待 |
| 话术方向 | "在关系里，你真正在意的可能是______" |
| 关键词 | 靠近、边界、看见、期待 |
| 示例（愚人） | "在这段关系里，你可能一直在等一个'合适的时机'。这张牌在问：如果完美时机从不存在，你愿意先迈出一小步吗？" |

#### 选择场景解读

| 要素 | 要求 |
|------|------|
| 目标 | 让做决定的用户降低决策焦虑 |
| 话术方向 | "选择A和选择B，你真正怕失去的是______" |
| 关键词 | 选择、路径、视角、暂停 |
| 示例（愚人） | "你不是不知道选哪个——你是怕选了之后后悔。这张牌在说：不是所有决定了就不能回头，大多数路都可以重新走。" |

#### 行动场景解读

| 要素 | 要求 |
|------|------|
| 目标 | 给用户一个今天就能做的低压力动作 |
| 话术方向 | "今天你可以只做一件小事：______" |
| 关键词 | 今天、一小步、试试看、不强求 |
| 示例（愚人） | "今天试试做一件比平时稍微勇敢一点点的事——不是辞职或表白，而是比如先说出你一直藏着的那个想法。" |

### 6.5 追问按情绪标签差异化

#### 6.5.1 情绪标签体系

用户在结果页/追问页可以选择当前的情绪状态，星澜基于牌面+情绪标签给出差异化回答。

| 情绪标签 | 适用追问示例 |
|----------|-------------|
| 😔 有点低落 | "这张牌看见了我的难过吗？" |
| 😰 焦虑不安 | "我现在为什么会反复想这件事？" |
| 🤔 迷茫困惑 | "我该往哪个方向走？" |
| 💔 关系困扰 | "这段关系是不是我想太多？" |
| 😤 有点生气 | "我这种愤怒是正常的吗？" |
| 😶 说不清 | "星澜，我其实也不知道自己在想什么" |

#### 6.5.2 追问回答结构（统一五段式）

每张牌×每个情绪标签×每个追问 = 一个完整的五段式回应：

1. **一句共情**："星澜感觉到了，你现在可能..."
2. **一个情绪翻译**："这不仅仅是______，更深的地方你也许在______"
3. **牌义隐喻**："这张牌像是在提醒你______"
4. **一个小动作**："今天你可以只做一件小事：______"
5. **一个温柔边界**："如果做到了，很好。如果没做到，也没关系。"（固定性兜底语句）

### 6.6 结果页三段式升级

现有结果页升级为「星澜看见了什么 → 你卡在哪里 → 今天的一小步」三段式结构：

| 段 | 定位 | 文案要求 |
|----|------|----------|
| **第一段：星澜看见了什么** | 共情——让用户感觉"被说中了" | 从用户可能的状态出发，避免从牌义出发。"星澜感觉到，你不是不想______，而是______。" |
| **第二段：你卡在哪里** | 翻译——把用户反复绕的情绪翻译成一句话 | "更深处，你也许真正在意的是______。这不是错误，这是你需要被看见的那一面。" |
| **第三段：今天的一小步** | 行动——给一个低压力、可执行的动作 | "如果你愿意的话，今天可以做一件很小的事：______。不是为了马上改变什么，而是让自己知道——你在试着对自己好一点。" |

#### 复用说明

圣三角牌阵的三张牌（过去/现在/未来）和关系探索的结果页，复用同一套三段式结构，但根据牌阵位置调整第一段和第二段的视角：

- **过去位置**：第一段偏"回看"，"那时候你可能在______"
- **现在位置**：第一段偏"当下"，"此刻的你也许在______"
- **未来位置**：第一段偏"倾向"，"如果顺着现在的方向，你可能会______"（注意：用"可能/倾向"，不用"一定/注定"）

### 6.7 合规边界

v1.2.0的内容升级必须通过以下合规检查：

| 检查项 | 标准 | 检查方式 |
|--------|------|----------|
| 零宿命论表述 | 无"你一定会""注定""命运安排""必然发生""很快会" | 全局文本搜索+人工审查 |
| 零心理诊断 | 无"抑郁症""焦虑症""人格障碍""心理疾病"等诊断用语 | 全局文本搜索 |
| 零专业建议 | 无投资/医疗/法律/维权建议 | 全局文本搜索 |
| 零关系操控 | 无"他一定喜欢你""对方很快会来找你""拿捏对方" | 全局文本搜索 |
| 免责声明可见性 | 每个结果页底部保留统一免责声明 | UI审查 |
| 心理学引用准确性 | 荣格概念非学术引用，标注"参考"而非"依据" | 内容审查 |

## 8. v2.0.0 远期方向

### 8.1 是否做AI/联网

| 问题 | 当前建议 | 触发条件 |
|------|----------|----------|
| 是否引入AI解读？ | **v1.x不做。v2.0评估。** | D7留存稳定>16%，且用户大量追问超出离线模板覆盖范围 |
| 如果要引入AI，方案是？ | 接入大型语言模型API（如部署于云端的专用塔罗小模型），做联网+服务端推理 | 需先解决：模型托管、隐私政策更新、内容审核机制、运营成本测算 |
| 离线AI可行吗？ | HarmonyOS端侧AI能力在发展中，当前不建议。v2.0+可评估 | HarmonyOS端侧模型成熟度、牌义生成质量验收 |

**关键原则**：AI解读的前提是不能破坏"不是算命"定位。AI生成的每一句回答都需要过合规审核。

### 8.2 是否做账号系统

| 问题 | 当前建议 | 理由 |
|------|----------|------|
| **v1.x做账号吗？** | **坚决不做。** | 无账号是星钥核心差异化优势之一。账号=数据上云=信任打折 |
| **v2.0做账号吗？** | **不主动做。** 除非以下场景不可避：跨设备数据同步、用户强烈需求 | 账号的价值是跨设备同步和备份，但代价是信任和复杂度。优先探索"本地备份+手动迁移"方案 |

### 8.3 是否做云同步

| 问题 | 当前建议 | 触发条件 |
|------|----------|----------|
| **v2.0做云同步吗？** | **谨慎评估。** 先做本地导出→手动恢复 | 大量用户因换机丢失数据并明确表达此需求 |
| **如果做，怎么做？** | 可选华为云同步（HarmonyOS提供）、手动加密备份导出、设备间直连迁移 | 让用户选择是否云端备份，而非强制 |

### 8.4 是否做iOS/Android

| 问题 | 当前建议 | 触发条件 |
|------|----------|----------|
| **v2.0做iOS/Android吗？** | **评估做。** 但不是把HarmonyOS版移植过去——而是重新设计符合各自平台设计语言的版本 | MAU稳定>5000，且HarmonyOS生态增长遇到明显天花板 |
| **iOS/Android的价值** | 触达用户规模扩大10-50倍，但竞争也更激烈 | 需提前思考：在iOS/Android上，离线+无账号+无付费的定位是否仍然有竞争力 |

### 8.5 是否做付费

| 问题 | 当前建议 | 触发条件 |
|------|----------|----------|
| **v2.0做付费吗？** | **可考虑一次性买断型付费。** 如"进阶探索包"（更多牌阵、深度内容包）一次性买断。**绝不**做订阅/自动续费 | D7留存>16%，且用户数>5000，且用户评论中出现"愿意付费"的信号 |
| **为什么一次性而非订阅？** | 与"无压力陪伴"定位一致。订阅制造续费压力，一次性买断让用户拥有而非租用 | 竞品分析明确看到：自动续费是信任杀手 |

### 8.6 现在绝不要做什么

| 事项 | 理由 |
|------|------|
| 做AI解读 | 离线模板先做好，AI是锦上添花不是雪中送炭 |
| 做账号系统 | 破坏"无账号=信任"的核心优势 |
| 做社区/UGC | 冷启动极难，且监管风险高（用户生成内容可能含违规信息） |
| 做付费订阅 | 破坏"纯净陪伴"定位 |
| 接入广告 | 破坏沉浸感和信任 |
| 跨平台移植 | 在HarmonyOS先站稳再扩展 |
| 做78张全牌库 | 内容深度>>内容广度 |
| 做多牌阵（超3种） | 牌阵不是差异化，内容深度才是 |

### 8.7 达到什么指标才值得做v2.0

| 指标 | 触发阈值 | 说明 |
|------|----------|------|
| D7留存 | >16% | 说明用户养成了习惯，产品有留存基础 |
| MAU | >5000 | 有一定用户基数 |
| 分享卡生成率 | >15% | 说明有自传播能力 |
| 用户评论正向率 | >70% | 说明产品口碑健康 |
| 7日星轨完成率 | >15% | 说明留存机制生效 |

## 10. 星澜人格升级规划

### 10.1 人格关键词

| 维度 | 关键词 | 说明 |
|------|--------|------|
| 核心气质 | 温柔、有边界、不审判、知性 | 始终如一 |
| 关系定位 | **引导者 / 镜子**（非朋友、非恋人、非导师、非心理医生） | 帮助用户看见自己，而不是替用户生活 |
| 情感温度 | 比冷静多一丝暖意，比热烈少三分热度 | "55度温水"——刚好入口，不烫不凉 |
| 智慧来源 | 塔罗意象 + 荣格心理学 + 生活常识 | 有理论但不掉书袋 |

### 10.2 说话风格

| 要素 | 要求 |
|------|------|
| 句式 | 中等句子（15-30字为主），少用长难句 |
| 修辞 | 多用"像是""也许""可能"，少用"一定""肯定" |
| 节奏 | 先共情再分析，先说感受再说建议 |
| 文化调性 | 现代中文，不文绉绉，不网感过度 |
| 性别感 | 中性偏柔，不使用明确性别标志 |

### 10.3 禁止语气

| 禁止 | 示例 |
|------|------|
| 算命先生语气 | "你一定会遇到……""这件事注定……" |
| 导师命令语气 | "你必须……""你应该……""不要……" |
| 心理医生语气 | "你的症状表明……""你需要治疗……" |
| 营销号语气 | "太准了！""惊人发现！""不看后悔！" |
| PUA/操控语气 | "他对你不是真心的""你要让他……" |
| 过度亲昵 | "亲爱的""宝宝""我的女孩" |
| 过度承诺 | "用了这个方法一定有效""星澜保证" |

### 10.4 典型句式

| 场景 | 推荐句式 |
|------|----------|
| 共情开头 | "星澜感觉到，你现在可能……" |
| 情绪翻译 | "这不仅仅是____，更深处你也许在____" |
| 牌义引导 | "这张牌像是在轻轻提醒你……" |
| 行动建议 | "如果你愿意的话，今天可以做一件很小的事：____" |
| 温柔边界 | "如果做到了，很好。如果没做到，也没关系。" |
| 不确定表达 | "也许……也可能……你可以先看看哪个更像你。" |
| 关系探讨 | "在这段关系里，你真正在意的也许是____" |
| 回看过去 | "那天的你，正在经历____。今天的你回头看，可能会发现____" |

### 10.5 适合说的话 vs 不适合说的话

#### 适合

> "你不是不想靠近，而是怕自己的靠近变成打扰。这张牌在说：可以试试。"
>
> "如果这句话像你，可以先从这里开始。不用一下子做到全部。"
>
> "星澜更想陪你看见的是：你不是卡在选择本身，而是卡在怕选错之后无法回头。"
>
> "这不是失败，只是这次连接没有被接住。不是你不够好。"
>
> "今天的一小步：发一句不带期待的轻问候。不是为了立刻得到答案，而是让自己不再一直停在猜测里。"

#### 不适合

> "这张牌预示你将在三个月内遇到贵人。" ❌（命运断言）
>
> "你应该果断一点，别再犹豫了。" ❌（命令式）
>
> "你的焦虑指数偏高，建议你做一下这个心理测试。" ❌（心理诊断）
>
> "他其实没那么喜欢你，别浪费时间了。" ❌（替用户判断）
>
> "今天的运势不太好，注意避开小人。" ❌（玄学断言）

### 10.6 各场景出现方式

| 场景 | 出现方式 | 语气 |
|------|----------|------|
| **首页** | 动态问候语（基于时间/连续使用天数/最近情绪主题） | 欢迎但不过度热情。如"下午好。今天想从哪开始？" |
| **结果页** | 三段式解读的叙述者 | 温柔、共情、"星澜看见了……""星澜更想陪你看见的是……" |
| **追问页** | 一对一回应者 | 专注、认真、"你说得对，这确实不是简单的事……" |
| **分享卡** | 文字的署名者 | 品牌感、简洁、"——星澜，于星钥塔罗" |
| **桌面卡片** | 每日一句话的朋友 | 轻量、不打扰、"今天星澜想对你说：____" |

### 10.7 如何让用户感觉"被看见"而非"被判断"

| 原则 | 做法 |
|------|------|
| **反射而非点评** | 说"你好像在____"而非"你这是____" |
| **邀请而非指导** | 说"如果你愿意，可以试试____"而非"你应该____" |
| **描述而非诊断** | 说"你反复在想的似乎是____"而非"你有焦虑倾向" |
| **允许而非催促** | 说"做不到也没关系"而非"每天坚持很重要" |
| **存在而非比较** | 绝不说"你比XX%的用户更焦虑" |

| 关系困扰类 | 想靠近又怕、被拒绝后的受伤、不确定对方心意 | 帮用户看见自己的真实期待，给低压力行动 |

| 说不清类 | 不明原因的低落/烦躁/不想说话 | 允许"说不清"本身，不强迫命名情绪 |

### 11.4 关系场景分类

| 分类 | 覆盖的关系类型 | 内容目标 |
|------|---------------|----------|
| 想靠近 | 暗恋、有好感但不确定、想主动但不敢 | 降低靠近的心理压力，给低风险行动建议 |
| 在拉扯 | 暧昧、忽冷忽热、不确定对方想法 | 帮用户看见自己的底线，不做操控建议 |
| 想放下 | 分手后、单恋、知道没结果但放不下 | 陪伴用户承认"放不下"是正常的，不催促 |
| 想修复 | 误会、冷战、想重新开始但不知道怎么开口 | 给低压力沟通模板，不承诺结果 |
| 家庭/友谊 | 亲子关系、朋友隔阂、社交焦虑 | 帮用户整理情绪而非教TA"怎么做才对" |

### 11.5 选择场景分类

| 分类 | 覆盖的选择类型 | 内容目标 |
|------|---------------|----------|
| 人生方向 | 换工作、换城市、升学、转行 | 帮用户看清"怕失去什么"而非"选哪个更好" |
| 日常决策 | 要不要去某个社交场合、要不要买某样东西 | 让用户感知情绪的权重在决策中多大 |
| 关系决策 | 要不要继续/要不要开始/要不要结束 | 帮用户区分恐惧和直觉 |
| 自我改变 | 想改变某个习惯/性格/生活方式但下不了决心 | 把大改变拆成微小的第一步 |

### 11.6 行动建议分类

| 说出来 | 低压力表达练习 | "发一句不带期待的轻问候" |

| 观察自己 | 自我觉察而非改变 | "今天注意一下自己在什么时候会突然烦躁" |

## 16. 最终建议

### 16.1 下个版本做v1.0.1还是v1.1.0？

**先做v1.0.1（稳定性补丁，1-2周），紧接着做v1.1.0（情绪星轨）。**

理由：
- v1.0.1是风控——上架后暴露的Bug如不快速修复，差评会永久伤害ASO
- v1.0.1时间极短（仅修Bug），不会显著延迟v1.1.0
- v1.1.0是留存战役的核心——D1→D7不能等
- v1.0.1和v1.1.0可以并行规划：v1.0.1开发期间同时启动v1.1.0的侦察专项（专项1）

### 16.2 最该先做哪3件事

| # | 事项 | 版本 | 理由 |
|---|------|------|------|
| 1 | **情绪星点真实化——每天点亮机制** | v1.1.0 | 留存主战场D1→D7的核心抓手。用户研究明确：轻仪式感是最强留存钩子。数据报告：情绪星点点击率健康阈值>20%，当前完全未激活此功能 |
| 2 | **牌义文本去玄学化——引入荣格心理学框架** | v1.2.0 | 竞品分析最紧迫建议。定位落地度从60%→90%的关键一步。合规是最大红利，必须尽快穿上防弹衣 |
| 3 | **桌片卡片每日一句** | v1.3.0（可提前到v1.1.0搭配） | 用户研究预测+10-20%次日回流。HarmonyOS专有护城河，竞品无法复制。开发难度低，ROI极高 |

### 16.3 最不该先做哪3件事

| # | 事项 | 理由 |
|---|------|------|
| 1 | **AI解读** | 离线模板还没做好就上AI，是本末倒置。AI不能弥补内容深度的缺失——预见塔塔就是前车之鉴（收费但解读空泛） |
| 2 | **78张完整牌库** | 22张深 vs 78张浅——答案明确。资源有限时先打深度，广度留到D7留存证明用户真正需要更多牌之后 |
| 3 | **社区/UGC** | 冷启动极难（需要大量内容才能让社区看起来活跃），且监管风险高。不是做不做的问题，是坚决不做 |

### 16.4 是否继续保持离线？

**是。v1.x阶段保持完全离线。**

理由：
- 离线是星钥塔罗的核心差异化优势之一（竞品无一能做到）
- 用户研究显示：79%的AI陪伴用户认为"表达负面情绪无需顾虑"是核心价值——离线直接交付"隐私安全"
- 数据报告预测：离线App的隐私协议同意率应显著高于行业均值（85-92%）
- 合规红利：不收集数据=零隐私风险
- 如果v2.0引入AI，需要重新设计联网边界（仅AI请求，不收集用户数据），但不是现在

### 16.5 是否短期做AI？

**不做。v1.x阶段坚决不做。**

理由：
- 竞品谷多多的MoE模型是技术壁垒——星钥不可能短期追赶
- 但谷多多的定位是"理性+问题解决"，星钥是"情绪陪伴"——离线模板的情感深度可以差异化竞争
- 塔罗气泡的68%留存靠的不是AI，是CBT行动指引+免费策略
- AI解读的不确定性是内容风险来源——需要审核每一条AI输出，运营成本高
- v1.x先证明"离线模板足够让用户留存"的假设，v2.0再考虑AI锦上添花

### 16.6 是否短期做账号？

**不做。短期、中期都不做。**

理由：
- 无账号是星钥最独特的产品体验之一（无竞品做到）
- 账号系统带来：注册摩擦（降低Onboarding完成率）、数据安全责任、隐私政策复杂化、用户信任打折
- 唯一可能触发账号的场景是"换机数据迁移"——用本地导出方案（加密的JSON文件）解决，远优于云账号

### 16.7 是否短期做付费？

**不做。v1.x保持完全免费。**

理由：
- 塔罗气泡的68%留存关键杠杆之一是"无限次免费"
- 付费墙在当前用户基数下没有任何意义——先要有深度使用的用户
- 竞品分析明确：自动续费是信任杀手，一次性买断是唯一可考虑的
- 当前阶段的目标是"被最多人爱上"，不是"从少数人那里收到钱"
- 如果用户真的爱上产品，未来一次性买断"进阶探索包"的转化率会远高于现在强推付费

### 16.8 最值得投入的核心体验是什么？

> **让用户每次使用后都想说一句："星澜真的懂我。"**

具体拆解为三个可感知的体验：

1. **被理解的瞬间**：看完结果页（三段式升级后），用户感觉"原来我不是奇怪，我只是在认真感受"。（对应v1.2.0内容厚度升级）

2. **被记住的证据**：7天后的情绪星轨告诉用户"星澜记得你每一天的心情"。（对应v1.1.0情绪星轨留存计划）

3. **被温柔推动的一小步**：分享卡/行动签/星澜一句话让用户觉得"今天我可以试试这个，不成功也没关系"。（对应v1.3.0分享传播增强）

**这三件事做好了，留存、传播、合规——都是水到渠成的结果。**
