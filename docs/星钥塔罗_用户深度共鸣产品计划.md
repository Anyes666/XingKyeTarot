# 星钥塔罗：让用户真正深有感触的产品计划

版本：v1.0  
适用项目：HarmonyOS / ArkTS「星钥塔罗」  
核心定位：塔罗式情绪陪伴 + 自我探索，不做宿命预言，不做专业建议  
陪伴者：星澜  
统一免责声明：结果仅供娱乐与自我探索参考，不构成任何专业建议。

---

## 1. 这次对话样本带来的核心洞察

从塔罗师对话样本里可以看到，用户表面上是在问“塔罗准不准”“未来会不会遇到谁”“该不该主动”，但真实需求并不是想要一段神秘预测，而是希望有人帮他把心里的不确定感讲清楚。

用户真正关心的是：

- 我是不是应该主动一点？
- 我靠近别人会不会显得很尴尬？
- 我该怎么开口，才不会太冒犯？
- 如果对方不接话，我是不是很失败？
- 我想要关系，但我不知道怎么自然地开始。

这说明「星钥塔罗」后续真正要做的，不是单纯补更多牌义，而是把塔罗牌变成一个心理隐喻入口，帮用户完成三件事：

1. **被理解**：星澜能说出用户没说出口的情绪。
2. **被接住**：星澜不吓人、不审判、不做确定性预言。
3. **有下一步**：星澜给用户一个低压力、可执行的小动作。

一句话总结：

> 塔罗只是入口，真正让用户留下来的，是星澜帮用户把混乱的心事整理成一句能做的事。

---

## 2. 产品总方向

「星钥塔罗」不应该做成传统算命 App，而应该强化为：

> 用塔罗牌作为心理隐喻的情绪陪伴工具。

产品重点不是告诉用户“未来一定会怎样”，而是帮助用户看见：

- 我现在为什么反复想这件事？
- 我真正害怕的是什么？
- 我有没有把对方的反应想得太重？
- 我能不能先做一点点，而不是逼自己马上做重大决定？

因此后续所有结果页、追问页、分享卡、历史记录都应围绕一个核心体验：

> 星澜不是替你决定，而是陪你把心里的结慢慢解开。

---

## 3. 星澜的人格与表达原则

星澜的语气要稳定，所有功能都要统一这套人格。

### 3.1 星澜应该像什么

星澜应该是：

- 温柔的
- 有边界的
- 不审判的
- 不吓人的
- 不神神叨叨的
- 能把复杂情绪说清楚的
- 能给用户一个低压力行动的

### 3.2 星澜不应该像什么

星澜不应该：

- 像算命先生一样断言未来
- 像导师一样命令用户
- 像心理医生一样诊断用户
- 像营销号一样制造焦虑
- 像操控话术教程一样教用户拿捏别人

### 3.3 禁止表达

后续文案中应避免：

- 你一定会……
- 这件事注定……
- 对方肯定……
- 马上会发生……
- 你必须……
- 这是命运安排……
- 这段关系必然……
- 这张牌说明你有心理疾病……
- 这张牌建议你投资 / 治疗 / 维权……

### 3.4 推荐表达

推荐使用：

- 这张牌可能在提醒你……
- 如果这句话像你，可以先从这里开始……
- 星澜更想陪你看见的是……
- 你不需要马上做决定，可以先做一个很小的动作……
- 这不是失败，只是这次连接没有被接住。
- 你可以主动，但不用把自己推到很危险的位置。

## 4. 结果页升级方向：从“牌义解释”到“情绪翻译”

当前塔罗类产品常见问题是：只解释牌义，不解释用户的处境。

未来「星钥塔罗」结果页应从下面这种表达：

> 圣杯骑士代表浪漫、靠近、表达。

升级为：

> 这张牌像是在说，你不是不想靠近，而是很怕自己的靠近变成打扰。你需要的不是一个完美开场，而是一个让自己不那么紧张的小入口。

### 4.1 每次解读建议固定为 5 个模块

每次结果页可以逐步统一成以下结构：

1. **星澜看见了什么**  
   先共情用户当前状态，让用户感觉“被说中了”。

2. **你真正卡住的地方**  
   把问题从表层事件翻译成内在情绪。

3. **这张牌的提醒**  
   用牌义作为心理隐喻，不做命运预言。

4. **星澜给你的一小步**  
   给用户一个今天就能做的低压力动作。

5. **温柔边界提醒**  
   提醒用户尊重自己，也尊重他人，不把结果看成专业建议。

### 4.2 示例：关系问题

用户问题：我该不该主动？

星澜式回答：

> 你想靠近，但又怕自己显得太用力。星澜觉得，这不是矛盾，而是你在保护自己的自尊。  
> 这张牌更像是在提醒你：可以主动，但不要用“证明自己”的方式主动。  
> 今天你可以只做一件小事：发一句不带期待的轻问候，看看对方是否愿意接住这个话题。

## 5. 新增核心机制：每次结果都给一个“低压力行动”

用户看完解读后最容易出现的问题是：

> 然后呢？我该怎么办？

所以后续所有结果页都应增加一个稳定模块：

> 星澜给你的一小步

### 5.1 行动建议的原则

行动建议必须满足：

- 具体
- 轻量
- 不冒犯别人
- 不逼用户马上做重大决定
- 不制造焦虑
- 不承诺结果
- 可以今天就做

### 5.2 不推荐的行动建议

不要写：

- 勇敢一点
- 相信自己
- 主动出击
- 放下一切
- 立刻表白
- 别再想了

这些太空，用户不会真正被帮助。

### 5.3 推荐的行动建议

关系场景：

> 今天只做一件小事：给一个你想靠近的人发一句不带期待的问候。不是为了立刻得到答案，而是看看这段连接有没有自然延伸的空间。

选择场景：

> 先把这个选择写成两个版本：如果我选 A，我最怕失去什么？如果我选 B，我最怕后悔什么？

情绪场景：

> 今晚先不要逼自己想明白。只写一句：我现在最累的不是事情本身，而是……

## 6. 追问星澜升级方向：从“补充解释”到“继续陪我想”

「追问星澜」是最适合让用户深有感触的功能之一。

它不应该只是解释牌义，而应该像一次温柔的继续对话。

### 6.1 关系探索后的追问模板

关系探索后可以提供以下固定追问：

1. 我该主动一点吗？
2. 我怎么开口才不尴尬？
3. 如果我怕被拒绝怎么办？
4. 这段关系是不是我想太多？
5. 我该继续靠近，还是先退一步？

用户点击后：

- 其他问题隐藏
- 只显示用户选中的问题
- 只显示星澜对应回答
- 回答文字更大、更像认真回应
- 保留「回到结果页」按钮
- 不新增聊天框
- 不接 AI 接口

### 6.2 回答结构

每个追问回答建议固定为：

1. 一句共情
2. 一个情绪翻译
3. 一个边界提醒
4. 一句可复制话术或低压力行动
5. 一句不把结果压在自己身上的提醒

示例：

> 你想主动，但又怕自己显得太用力。星澜觉得，这不是矛盾，而是你在保护自己的自尊。  
> 你可以主动，但不用一下子把心意摊开。更适合你的方式，是先创造一个轻松的交集。  
> 可以试试这句话：  
> “刚刚看到这个，感觉你可能会喜欢，就想发给你。”  
> 如果对方愿意接话，再慢慢往下聊。如果对方回应很淡，也不是你失败了，只是这次连接没有被接住。

## 7. 新功能方向：星澜帮我想一句话

这是从样本对话中最明确提炼出的功能机会。

用户真正想问的是：

> 我该怎么说？

所以可以在关系探索结果页或追问星澜页增加轻功能：

> 星澜帮我想一句话

### 7.1 功能定位

当用户想主动靠近某个人，但不知道怎么开口时，星澜给出低压力、可复制、不过度承诺的关系话术。

本功能先做离线模板，不联网，不接 AI。

### 7.2 入口建议

入口可以放在：

- 关系探索结果页
- 追问星澜页
- 行动签分享卡前

按钮文案：

- 星澜帮我想一句话
- 给我一句自然开口
- 我想靠近，但不想冒犯

### 7.3 三个场景选项

进入后提供三个选项：

1. 我想先开启聊天
2. 我想自然邀约
3. 我想表达好感但不想吓到对方

点击后只展示一个回答，不做聊天输入框。

### 7.4 场景一：我想先开启聊天

回答模板：

> 可以从“分享”开始，而不是从“盘问”开始。  
> 试试这句：  
> “刚刚看到这个，突然想到你可能会喜欢。”  
> 它不会给对方太大压力，也不会让你显得太刻意。对方如果愿意接住，你们就有了自然延伸的话题。

### 7.5 场景二：我想自然邀约

回答模板：

> 不要一上来就把邀约说得太重。你可以把它变成一个轻一点的提议。  
> 试试这句：  
> “这周我想找个地方坐坐，如果你也想放松一下，可以一起。”  
> 这句话表达了邀请，也给了对方拒绝的空间。

### 7.6 场景三：我想表达好感但不想吓到对方

回答模板：

> 你可以表达兴趣，但不要急着要求对方回应。  
> 试试这句：  
> “我发现和你聊天的时候，我会比较放松，所以想多了解你一点。如果你也愿意，我们可以慢慢来。”  
> 这不是逼对方表态，而是在诚实地靠近。

### 7.7 合规边界

这个功能必须避免：

- 不教操控话术
- 不鼓励骚扰
- 不制造对方一定喜欢你的暗示
- 不承诺关系结果
- 不PUA
- 不让用户反复打扰对方
- 强调尊重对方回应和边界

## 8. 分享卡升级方向：从“结果卡”到“可保存的情绪证据”

分享卡不只是传播工具，也可以变成用户保存情绪的方式。

当前已有类型：

- 情绪签
- 人设签
- 行动签

后续重点不一定马上新增类型，而是先让「行动签」更有用。

### 8.1 行动签应该更具体

不建议只写：

> 今天适合主动一点。

建议写成：

> 今天的一小步：  
> 给一个你想靠近的人发一句不带期待的轻问候。  
> 不是为了立刻得到答案，而是让自己不再一直停在猜测里。

### 8.2 可选新增类型：话术签

未来可以新增分享卡类型：

> 话术签

内容结构：

- 适合你的开场
- 适合你的邀约
- 星澜提醒
- 免责声明

示例：

> 适合你的开场：  
> “刚刚看到这个，突然想到你可能会喜欢。”  
> 星澜提醒：好的靠近不是用力证明自己，而是让对方感到舒服，也让自己不委屈。

---

## 9. 历史记录升级方向：从“记录列表”到“情绪回看”

历史记录未来不应该只是记录列表，而应成为用户回看自己的入口。

目标体验：

> 原来我这几天一直在同一个问题上绕。

### 9.1 历史详情建议结构

每条历史详情可以逐步升级为：

1. 那天我问的是
2. 星澜当时看见的是
3. 那张牌提醒我的事
4. 那天给我的一小步
5. 今天回看，我可能可以补一句

示例：

> 那天我问的是：  
> 我想知道这段关系值不值得。  
> 星澜当时看到的是：  
> 你不是不清醒，而是舍不得。  
> 那天给我的一小步：  
> 先停止猜测，观察对方是否真的行动。

### 9.2 情绪回看方向

未来可以在历史记录中增加：

- 反复出现的问题类型
- 最近常见情绪
- 7 日情绪星轨
- 最近的关系主题
- 最近最常出现的提醒词

这会比单纯列表更有留存价值。

## 10. 情绪星点升级方向：让 7 日陪伴更像仪式

情绪星点是留存功能，不只是结果页附属入口。

后续可以把它做成：

> 7 日情绪星轨

核心体验：

- 今天我点亮了一颗星
- 这一周我反复出现了什么情绪
- 我不是每天都要打卡，而是在慢慢看见自己

### 10.1 情绪星点文案方向

按钮：

- ⭐ 点亮情绪星
- ⭐ 查看情绪星点

结果页引导语：

> 如果你愿意，可以把今天的心情留成一颗星。不是为了坚持打卡，而是为了慢慢看见自己。

## 11. 优先级计划

### P0：先修稳定性与明显体验问题

目标：保持 App 可编译、流程可走通、不要让用户在基础体验中受阻。

包括：

1. HistoryPage 空状态按钮按筛选类型跳转正确。
2. AskHeartPage 首次体验支持“暂时跳过，先逛首页”。
3. ShareCardPage 单牌分享卡增加卡牌面视觉。
4. 全屏深色背景适配，减少上下白边。
5. 页面顶部和底部间距轻量修正。

### P1：增强“深有感触”的核心体验

目标：让用户看完结果后觉得“星澜懂我”。

包括：

1. 结果页增加“星澜给你的一小步”。
2. 追问星澜回答升级为“共情 + 边界 + 具体话术”。
3. 关系探索后新增“星澜帮我想一句话”。
4. 行动签从抽象建议改为具体低压力行动。

### P2：增强留存和回看

目标：让用户愿意回来看看过去的自己。

包括：

1. 历史详情升级为“情绪回看”。
2. 情绪星点强化 7 日星轨。
3. 历史记录按情绪、关系、选择做更清晰分类。
4. 增加“最近反复出现的心事”提示，但不做诊断。

### P3：后续扩展

目标：产品更完整，但不急于第一阶段实现。

包括：

1. 话术签分享卡。
2. 更多关系场景模板。
3. 更丰富的离线追问模板。
4. 未来如需 AI，再单独规划联网、备案、隐私政策和安全边界。

## 12. 第一阶段推荐实施顺序

为了避免 Agent 混改，建议每次只做一个小专项。

### 第 1 轮：修明确 Bug

任务：HistoryPage 空状态按钮按筛选类型变化。  
只允许修改：HistoryPage.ets。  
验收：圣三角显示“去圣三角”，关系显示“去关系探索”，ERROR = 0。

### 第 2 轮：修分享卡视觉

任务：ShareCardPage 单牌分享卡增加卡牌面视觉。  
只允许修改：ShareCardPage.ets。  
禁止修改：ShareImageService、保存图片底层逻辑。  
验收：分享卡有黑金卡牌面，保存图不包含按钮和状态栏，ERROR = 0。

### 第 3 轮：修首屏和沉浸式

任务：EntryAbility 深色窗口背景 + 核心页面轻量 padding 调整。  
优先页面：HistoryPage、AskHeartPage、TriangleIntroPage、RelationIntroPage。  
验收：上下白边减少，内容不被状态栏和手势条遮挡，ERROR = 0。

### 第 4 轮：升级追问星澜文案

任务：把追问回答从“解释牌义”升级为“共情 + 具体行动”。  
只允许修改：FollowUpPage 或追问数据源文件。  
验收：用户点击一个追问后，只显示选中问题和星澜回答，回答更落地，ERROR = 0。

### 第 5 轮：新增“星澜帮我想一句话”轻功能

任务：关系探索后提供三种离线话术模板。  
建议入口：RelationResultPage 或 FollowUpPage。  
验收：不联网、不接 AI、不新增复杂 Store，话术合规，ERROR = 0。

## 13. 功能验收标准

后续所有“深有感触”相关功能，都用以下标准验收：

### 13.1 用户感受标准

用户看完后应该感觉：

- 它不是在吓我
- 它没有命令我
- 它好像说中了我为什么纠结
- 它给了我一个不会太尴尬的小动作
- 如果我做不到，也不会觉得自己失败

### 13.2 合规标准

必须满足：

- 不预测确定未来
- 不承诺关系结果
- 不涉及医疗、法律、投资建议
- 不做心理诊断
- 不鼓励骚扰别人
- 不教操控话术
- 不制造焦虑
- 保留统一免责声明

### 13.3 工程标准

必须满足：

- 每轮只做一个专项
- 每轮只改指定文件
- 不使用 any
- 不使用匿名对象类型
- router 参数必须显式声明 RouterOptions
- UI 闭包不写 const / let / 普通赋值
- 修改后必须 assembleHap
- ERROR 必须为 0
- WARN 可以暂时保留

## 14. 一句话产品北极星

> 星钥塔罗不是告诉用户未来会发生什么，而是让用户在混乱、犹豫、想靠近又害怕的时候，被星澜温柔地接住，并得到一个今天就能做到的小动作。

后续给 Agent 的任务都应加上：

- 本轮只做指定专项。
- 不要混改。
- 不要清 WARN。

- 不要改 EmotionStarPage，除非任务就是情绪星点。
- 不要改 FollowUpPage，除非任务就是追问星澜。
