## 第一部分：项目概览

星钥塔罗是一款运行在 HarmonyOS NEXT 手机上的本地离线塔罗应用，当前重点不在“复杂算法”或“联网能力”，而在于：

- 让用户可以快速完成每日一占与多种基础牌阵占卜；
- 让结果能够稳定保存、回看与分享；
- 让产品在合规前提下具备足够的内容丰富度与持续迭代能力；
- 保持工程结构简单、明确、可维护。

应用当前不依赖网络、不依赖账号系统、不依赖云端服务，所有核心数据都保存在本地。

## 第二部分：当前已实现功能

### 一、首页

当前首页由三块主要内容组成：

- 顶部品牌区：展示“星钥塔罗”名称与副标题；
- 中部每日一占卡片：直接展示今日牌名、正逆位和关键词；
- 下部功能区：展示多种占卜入口与历史查询入口。

首页当前已提供以下入口：

| 入口 | 当前状态 | 说明 |
|------|------|------|
| 每日一占 | ✅ | 进入当日运势详情页 |
| 单牌快占 | ✅ | 进入页面后自动抽 1 张牌 |
| 圣三角 | ✅ | 过去 / 现在 / 未来三张牌 |
| 关系探索 | ✅ | 我的状态 / 对方状态 / 关系走向 |
| 决策牌阵 | ⏳ | 仅占位，当前未开放 |
| 历史查询 | ✅ | 查看最近 50 条占卜记录 |

### 二、每日一占

每日一占是当前最稳定、最轻量的日常功能。

已实现能力：

- 每天固定抽取 1 张牌；
- 同一用户同一天结果稳定；
- 展示牌名、正逆位、关键词、摘要解读；
- 支持从首页查看详情；
- 支持最近每日记录回看；
- 支持桌面卡片同步展示。

实现特性：

- 基于 `userId + 日期` 生成确定性种子；
- 结果按日期缓存到 Preferences；
- 每日记录单独归档到本地历史中。

### 三、单牌快占

单牌快占已经实现为真正的“快占”体验。

当前行为：

- 用户点击首页“单牌快占”后直接进入占卜页；
- 进入页面即自动抽出 1 张牌；
- 默认使用 `general` 主题；
- 不需要额外选择主题；
- 不需要点击翻牌；
- 直接进入可查看解读的状态。

这一功能适合快速灵感、随手抽一张或轻量问答场景。

### 四、圣三角占卜

圣三角是当前的主牌阵之一，三张牌分别表示：

- 过去
- 现在
- 未来

已实现流程：

- 用户先选择主题；
- 系统抽取 3 张不重复的牌；
- 用户按顺序翻牌；
- 翻完后进入结果页查看完整解读。

### 五、关系探索牌阵

关系探索是基于三张牌结构扩展出来的关系向牌阵。

当前牌位：

- 我的状态
- 对方状态
- 关系走向

它复用了三牌抽取与结果展示逻辑，仅替换牌位含义和显示文案，因此维护成本较低。

### 六、占卜结果页

结果页当前支持所有已开放牌阵的统一展示。

已实现能力：

- 根据牌阵类型动态显示页面标题；
- 根据牌阵类型动态显示牌位副标题；
- 展示每张牌的牌名、正逆位、关键词与解读；
- 支持保存分享图片；
- 进入结果页后自动将本次占卜结果归档到本地历史；
- 左上角统一返回按钮。

### 七、历史查询

历史查询已实现为完整可用功能。

当前规则：

- 保存最近 50 条占卜记录；
- 按时间倒序排列；
- 每条记录展示：
  - 牌阵类型（如单牌快占、圣三角、关系探索）
  - 占卜时间
  - 占卜主题
  - 抽到的牌摘要
- 点击记录可直接进入该次结果详情页；
- 支持一键清空历史记录；
- 首页“历史查询 >”入口会显示当前已保存的记录条数。

### 八、分享图片保存到相册

当前支持将占卜结果生成为分享图并保存到系统相册。

已实现能力：

- 通过隐藏的 `SharePreview` 组件生成分享预览；
- 使用 `UIContext.getComponentSnapshot()` 截图；
- 将生成的 PixelMap 保存到系统相册；
- 保存成功后提示“已保存到相册”；
- 如果保存失败，则降级为复制文字内容，不静默失败。

### 九、桌面卡片

项目当前带有 2×2 桌面万能卡片（FormExtension）。

已实现内容：

- 展示每日一占牌面与关键词；
- 点击卡片可跳转到每日一占详情；
- 由系统 Form 与应用前台逻辑共同兜底刷新。

### 十、新手引导与法务确认

当前具备基础引导与合规入口：

- 首次启动显示用户协议与隐私政策确认；
- 未完成新手引导时，自动进入引导页；
- 引导页为 3 屏介绍；
- 完成状态写入本地 Preferences；
- 各主要二级页面统一增加左上角返回按钮。

### 一、`common/`

放置核心业务模型。

关键文件：

- `TarotCard.ets`：塔罗牌、牌组、正逆位相关结构
- `Spread.ets`：牌阵定义
- `DivinationRecord.ets`：一次完整占卜记录
- `DailyFortune.ets`：每日一占结果结构

当前牌阵模型包括：

- `single_card`
- `sacred_triangle`
- `relationship`

### 三、`data/`

放置静态数据与本地持久化底层。

关键文件：

- `TarotCardData.ets`：78 张牌定义
- `InterpretationData.ets`：正逆位解读文本

- `PreferenceStore.ets`：Preferences 封装

### 四、`service/`

放置核心业务逻辑，是当前项目最重要的业务层。

关键文件与职责：

- `DailyFortuneService.ets`
  - 生成每日一占

  - 保存每日一占历史

- `DivinationService.ets`
  - 按牌阵抽牌
  - 控制抽牌数量
  - 生成 `DivinationRecord`

- `DivinationHistoryService.ets`
  - 保存占卜历史
  - 读取最近 50 条占卜记录
  - 清空占卜历史

- `TarotDataService.ets`
  - 提供牌面数据、关键词、解读内容

- `ShareImageService.ets`
  - 分享图文件命名
  - 截图后保存相册

- `ShareService.ets`
  - 分享失败时的剪贴板兜底

- `UserStateService.ets`
  - 用户协议接受状态

## 第五部分：数据与本地存储

应用当前所有业务数据都保存在本地，不联网、不上传。

### 一、核心本地存储项

| 数据项 | 存储方式 | 说明 |
|------|------|------|
| 用户协议接受状态 | Preferences | 首次启动合规确认 |
| 新手引导完成状态 | Preferences | 决定是否进入 onboarding |
| 本地用户 ID | Preferences | 用于每日一占确定性种子 |
| 每日一占缓存 | Preferences | 保证同日同结果 |
| 每日一占历史 | Preferences | 当前最多 30 条 |
| 占卜历史 | Preferences | 当前最多 50 条 |

### 二、占卜历史存储内容

每条占卜历史当前保存：

- `id`
- `questionType`
- `spread`
- `cards`
- `createdAt`

其中每张牌还包含：

- 牌对象
- 牌位名称
- 正逆位状态
- 解读文本
- 关键词

这意味着历史页可以直接恢复原始结果页，不需要重新抽牌。

## 第六部分：权限与相册能力

### 一、当前申请的权限

项目当前只申请以下权限：

```text
ohos.permission.WRITE_IMAGEVIDEO
```

配置位置：

[module.json5](D:/XingKeyTarot/entry/src/main/module.json5)

### 二、权限用途

该权限仅用于：

- 用户主动点击“保存分享图片”时；
- 将应用本地生成的占卜结果图片写入系统相册。

### 三、这项权限不代表什么

这项权限**不是读取图库权限**，当前应用不会：

- 浏览用户已有相册内容；
- 读取图库中的既有图片；
- 分析图库；
- 上传图片到服务器；
- 在后台自动保存图片。

也就是说，当前项目的“图库相关功能”本质是：

> **只向系统相册写入应用生成的图片，不读取用户相册。**

### 四、未申请的权限

当前明确未申请：

- 网络权限
- 相机权限
- 麦克风权限
- 位置权限
- 通讯录权限
- 后台常驻权限

## 第七部分：页面与交互流

### 一、启动流

1. 启动 App
2. 检查用户协议是否已同意
3. 若未同意，展示法务确认弹窗
4. 若同意但未完成 onboarding，跳转引导页
5. 完成后进入首页

### 二、单牌快占流

1. 首页点击“单牌快占”
2. 写入 `selectedSpreadId = single_card`
3. 跳转占卜页
4. 进入页面自动抽 1 张牌
5. 直接进入可查看解读状态
6. 点击“查看解读”进入结果页
7. 结果页自动写入历史

### 三、圣三角 / 关系探索流

1. 首页点击对应入口
2. 占卜页读取对应牌阵
3. 用户选择主题
4. 系统抽 3 张不重复的牌
5. 用户按顺序翻牌
6. 进入结果页查看完整解读
7. 结果页自动写入历史

### 四、历史回看流

1. 首页点击“历史查询 >”
2. 进入历史页
3. 浏览最近 50 条记录
4. 点击某条记录
5. 将该记录写入 `currentDivinationRecord`
6. 跳转结果页并直接回显
