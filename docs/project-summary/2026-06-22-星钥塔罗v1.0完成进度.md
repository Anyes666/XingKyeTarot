## 第二部分：版本路线图

### v1.1 — 工程稳固 + 卡片真刷新（建议 1-2 天）

> 目标：让上架更新零风险，桌面卡片真正有用。

| 任务 | 说明 |
|------|------|
| 🔧 修 deprecated API | 29 个 WARN：`getContext()` → `getContext(this)`、`router.pushUrl` → `pushNamedRoute`、`back` → `pop`、`showToast` → 新 API |
| 🃏 桌面卡片自动刷新 | 目前卡片只在 App 冷启动/回前台时刷新。加系统定时回调，设置合理的 `updateDuration` |
| 🧪 测试修复 | 确认每日运势确定性、抽牌不重复、卡片数据正确性都有断言覆盖 |

### v1.2 — 每日一占历史记录（建议 2-3 天）

> 目标：用户每天都想回来看看。塔罗应用最核心的留存点。

| 任务 | 说明 |
|------|------|
| 📅 运势历史列表 | 每天自动存档运势结果，按日期排列，可回溯翻阅 |
| 💭 心情记录 | 每日运势页加心情选择（😊😐😢），和当日运势绑定 |
| 🔥 连续打卡 | 轻量仪式感：显示"已连续打卡 N 天"，不做强打扰 |
| 🏠 首页卡片强化 | 把"已连续 N 天"放进每日运势卡片，增加回来动力 |

### v1.3 — 牌库 · 牌义学习（建议 3-4 天）

> 目标：从"抽完就走"变成"可以逛"。低风险高价值。

| 任务 | 说明 |
|------|------|
| 📖 78 张牌详情页 | 每张牌独立页面：牌名/编号/关键词/正位解读/逆位解读 |
| 🔍 分类浏览 | 大阿卡纳 / 权杖 / 圣杯 / 宝剑 / 星币 五大分类 |
| 🔎 关键词搜索 | 输入"爱情""工作"等关键词，匹配相关牌 |
| ⭐ 收藏牌 | 收藏常看的牌，方便回顾 |
| 🔗 结果页跳转 | 占卜结果中每张牌名可点击跳转到牌义详情 |

### v1.4 — 分享体验升级（建议 1-2 天）

> 目标：让分享图成为自然传播入口。

| 任务 | 说明 |
|------|------|
| 🖼️ 每日运势卡片生成 | 每日运势页也支持生成精美分享卡片 |
| 📤 分享渠道 | 在相册保存基础上，调起系统分享面板 |
| 🎨 分享图视觉优化 | 1080×1920 布局字体/间距继续打磨 |

### v1.5 — 新增牌阵（建议 2-3 天）

> 目标：给不同场景的用户提供匹配的工具。不要一口气加太多，每个要有清晰使用场景。

| 牌阵 | 场景 | 牌数 |
|------|------|------|
| 🔮 单牌占卜 | 快速问答，日常灵感 | 1 张 |
| 💑 关系牌阵 | 双方立场 + 关系走向 | 4 张 |
| ⚖️ 决策牌阵 | "选 A 还是 B" | 4 张 |
| 🌕 月度牌阵 | 4 周运势概览 | 4 张 |

### v2.0 — 商业化探索（后续评估）

| 方向 | 说明 |
|------|------|
| 🎨 主题皮肤 | 解锁不同视觉风格的牌背/背景 |
| 🔓 高级牌阵 | 凯尔特十字等复杂牌阵 |
| 💎 会员 | 无限历史记录、更多牌阵、去广告（如果有） |
| 🤖 AI 解读 | 结合用户问题和主题做个性化解读（需评估合规） |

### 推荐迭代节奏

```
v1.0（已发布）→ v1.1 → v1.2 → v1.3 → v1.4 → v1.5 → v2.0
```

**v1.1 + v1.2 性价比最高**：卡片自动刷新让桌面小组件真正有用，连续打卡 + 历史记录让用户每天回来。两个版本做完就可以发一次更新，根据留存数据再决定后续优先级。
