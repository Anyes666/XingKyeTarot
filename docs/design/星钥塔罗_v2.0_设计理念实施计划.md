# 星钥塔罗 v2.0 设计理念实施计划

> 基于《星钥塔罗 App 设计方向》文档，结合当前 v1.1 项目进度，拆解为可执行的实施路线图。
>
> **核心理念**: 不是冷冰冰抽一张牌，而是一次安静、私密、被理解的内心对话。

---

## 一、差距分析：v1.x 现有能力 vs v2.0 目标体验

### 1.1 用户旅程对比

| 阶段 | 当前 v1.1 | 目标 v2.0 | 差距 |
|------|-----------|-----------|------|
| **进入** | 首页直接展示每日运势+占卜入口 | 先出现一句"问心"引导，情绪化选项 | 缺「问心」交互层 |
| **准备** | 选主题 → 直接进入翻牌 | 5 秒静心仪式动画 | 缺仪式感动画 |
| **抽牌** | 点击翻牌（逐张） | 保留翻牌交互，增加心理暗示文字 | 当前基本满足 |
| **结果** | 牌阵展示 + 解读文字 | 5 模块结构：一句话牌意 → 状态解读 → 情绪安抚 → 行动建议 → 结尾温柔句 | 解读结构需重构 |
| **沉淀** | 占卜历史列表 | 星钥日记、情绪星图、牌面回响 | 缺长期陪伴能力 |
| **人格** | 无 | 虚拟塔罗师"月见"人格贯穿 | 全新能力 |

### 1.2 牌阵对照

| v1.1 已有 | v2.0 新增 | 说明 |
|-----------|-----------|------|
| 单牌快占 | 今日情绪牌（深度版） | 拆成"今日能量+今日提醒+今日行动"三层 |
| 圣三角（过去/现在/未来） | ✅ 保留，解读侧重重构 | 不变 |
| 关系探索（我/对方/走向） | 关系温度牌阵（我/对方/保护自己） | 调整位置定义，语气更温柔 |
| — | 职场选择牌阵（4张） | 「要不要离职？」情绪化命名 |
| — | 考学焦虑牌阵（3张） | 「上岸前的自我校准」 |
| — | 财富稳定牌阵（4张） | 「钱从哪里漏掉了？」 |
| 决策牌阵（已预留未启用） | 融入"纠结选择"问心场景 | 可以和问心联动 |

## 二、实施分阶段路线图

### Phase 2.0-alpha：体验地基（v1.2 ~ v1.3）

> **目标**: 把"被接住"的感觉做出来，核心是结果页重构 + 问心入口。

#### 2.0.1 解读数据结构重构

当前 `CardInterpretation` 只有 `upright` / `reversed` 两段文字。需要扩展为结构化解读。

**新增类型** (`model/TarotCard.ets`):

```typescript
// 结构化解读（替代扁平字符串）
export interface StructuredInterpretation {
  oneLineCardMeaning: string;     // 一句话牌意（金句）
  emotionLabel: string;           // 情绪标签：「你最近可能处在…」
  situationAnalysis: string;      // 当前状态解读
  emotionalComfort: string;       // 情绪安抚
  actionSteps: string[];          // 行动建议列表（2-4条）
  closingGentleWords: string;     // 结尾温柔句
  realWorldTask?: string;         // 今日现实任务
}

// 扩展 CardInterpretation
export interface CardInterpretation {
  cardId: string;
  upright: string;                 // 保留向后兼容
  reversed: string;                // 保留向后兼容
  uprightKeywords: string[];
  reversedKeywords: string[];
  uprightStructured?: StructuredInterpretation;   // v2.0 新增
  reversedStructured?: StructuredInterpretation;  // v2.0 新增
}
```

**影响范围**: `TarotCard.ets`（类型）、`InterpretationData.ets`（新增 156 条结构化解读）、`DivinationResultPage.ets`（结果页 UI 重构）、`SharePreview.ets`（分享图模板更新）

**工作量**: 156 × 2 条解读，每条约 400-600 字。可分批进行——先用 AI 批量生成初稿，人工校对优化。

### Phase 2.0-beta：陪伴能力层（v1.4 ~ v1.6）

> **目标**: 让用户抽完不走，能持续沉淀和回看。

#### 2.0.4 静心仪式动画

**设计规格**:
- 深蓝星空背景缓慢流动
- 金色星尘聚集成一把钥匙形状
- 动画时长 5 秒
- 文字逐行浮现：
  > "请把注意力放回自己身上。"
  > "不需要许愿，也不用害怕答案。"
  > "这张牌只是一面镜子，帮你看清此刻的内心。"
- 按钮在文字结束后淡入：「我准备好了，开启牌面」

**技术方案**: ArkUI 显式动画（`animateTo`）+ Canvas 绘制星空粒子 + 路径动画

**新增文件**:
- `CalmRitualPage.ets`：静心仪式页（独立页面）
- `StarKeyAnimation.ets`：星尘聚合成钥匙动画组件

#### 2.0.5 星钥日记 + 情绪星图

**数据结构** (`model/StarKeyDiary.ets`):

```typescript
export interface DiaryEntry {
  id: string;
  recordId: string;          // 关联的占卜记录
  userNote: string;          // 用户一句话笔记
  emotionTags: string[];     // 情绪标签：「关系边界」「职场压力」「自我怀疑」
  cardId: string;            // 关联牌
  cardOneLine: string;       // 牌的一句话牌意
  createdAt: number;
}

export interface EmotionStarMap {
  period: string;            // 如「最近30天」
  topThemes: Array<{         // 高频主题
    theme: string;
    count: number;
    percentage: number;
  }>;
  dominantEmotion: string;   // 主导情绪
  suggestion: string;        // 综合建议
}
```

**页面设计**:

**星钥日记列表页**：时间线形式，每条显示：
- 日期 + 牌名 + 用户的一句话笔记
- 情绪标签小圆标
- 点击展开完整记录

**情绪星图页**：
- 星图式可视化（Canvas 绘制散点/星座连线）
- 中心是用户最常出现的主题
- 外围是各情绪维度的分布
- 底部是一段综合解读文字

**新增文件**:
- `model/StarKeyDiary.ets`
- `service/DiaryService.ets`
- `pages/StarKeyDiaryPage.ets`
- `pages/EmotionStarMapPage.ets`
- `components/DiaryEntryCard.ets`
- `components/StarMapCanvas.ets`（Canvas 组件）

#### 2.0.6 牌面回响

**设计**: 每次抽牌结果页底部，弹出：

> "这张牌里，哪句话最像你现在的状态？"

3 个选项（根据牌意自动生成的贴近用户场景的描述），选择后生成更精准的个性化解读。

**数据**:
- 每张牌 3 条回响选项 + 对应定向解读（可先覆盖 22 张大阿卡纳）
- 需要 `InterpretationData.ets` 增加 `echoOptions` 字段

**涉及文件**:
- `InterpretationData.ets`（新增回响数据）
- `DivinationResultPage.ets`（新增回响弹窗）

#### 2.0.7 今日现实任务

**设计**: 每次解读后给一个具体可执行的小任务。

> 「今日任务：不急着证明自己。今天只完成一件你真正能掌控的小事。」

**数据**: 22 张大阿卡纳 + 部分高频小阿卡纳，每张牌正逆位各一条任务。

**呈现**: 结果页行动建议区的最后一条，放大、加金色边框，可标记"完成"。

### Phase 2.0-rc：深度差异化能力（v1.7 ~ v2.0）

#### 2.0.8 虚拟塔罗师「月见」人格

**身份**: 温柔、专业、边界清晰的女性塔罗师

**贯穿触点**:
| 场景 | 月见出现方式 |
|------|-------------|
| 首次进入 | "你好，我是月见。我不会替你决定答案，但我会陪你看清心里的方向。" |
| 问心时 | "今晚，你想让哪一部分的自己被看见？" |
| 结果页 | 星钥提醒 → "月见提醒" |
| 星钥日记 | "这些情绪不是你的弱点，是你对自己的诚实。" |
| 连续 7 天记录 | "你已经记录了一周的情绪轨迹，来看看你的星图吧。" |

**技术实现**: 不需要 NLP，本质是预设的文案触发器，根据场景和上下文拼接。

**新增文件**:
- `service/TarotReaderPersona.ets`：月见人格文案系统

#### 2.0.9 月相陪伴计划

**设计**: 结合真实月相，提供节奏型情绪陪伴。

| 月相 | 内容 |
|------|------|
| 🌑 新月 | 适合设定目标 · "写下这个月你最想滋养的 3 件事" |
| 🌒 上弦月 | 适合推进计划 · "上次设定的目标，哪件事可以今天迈出第一步？" |
| 🌕 满月 | 适合复盘关系 · "看看最近的星钥日记，什么主题在重复出现？" |
| 🌘 下弦月 | 适合放下消耗 · "有没有一个人或一件事，你准备好慢慢松手了？" |

**页面**: 新月 → 月相卡片 + 对应引导文字 + 关联抽牌建议

**新增文件**:
- `service/MoonPhaseService.ets`（月相计算）
- `pages/MoonPhasePage.ets`
- `components/MoonPhaseCard.ets`

```
高影响 ┤                 ┌──────────────────────┐
       │                 │ 结果页五模块重构       │ ← DO FIRST

       │                 │ 首页问心入口           │
       │                 ├──────────────────────┤
       │                 │ 星钥日记               │ ← DO SECOND
       │                 │ 牌面回响               │
       │                 │ 情绪星图               │
       │                 │ 今日现实任务           │
       ├─────────────────┼──────────────────────┤

       │                 │ 月见人格系统           │
       │                 │ 月相陪伴计划           │

       │                 │ 品牌焕新 Token        │
低影响 ┤                 └──────────────────────┘
        低复杂度                              高复杂度
```

- **红区（先做）**: 结果页重构 + 问心入口 —— 最小投入、最大体验提升

- **黄区（后做）**: 深度差异化（仪式动画、人格、月相、新牌阵）—— 竞争壁垒

## 六、文案资产规划

### 6.1 解读文案调整原则

当前 156 条解读已经写完了，v2.0 需要做的不是推翻，而是**结构化和场景化**：

- **保留**现有的正逆位解读文字 → 作为 `situationAnalysis` 主体
- **新增**一句话牌意 → 精简成 15-25 字金句（如"你不是想太多，你只是还没看清真相"）
- **新增**情绪安抚段落 → 针对恐惧牌（高塔、死神、恶魔、宝剑十等）定制安抚文案
- **新增**行动建议 → 每张牌 2-4 条可执行步骤
- **新增**结尾温柔句 → 22 张大阿卡纳各有专属收尾

### 6.2 月见人格文案池

约 30-50 条预置文案，覆盖 8 个场景（首次问候、每日开始、问心引导、抽牌鼓励、结果共情、低谷陪伴、连续记录、月相提醒）。

建议先写 30 条核心文案，后续随场景扩展。

```
v2.0 → 月见 ← 完整人格
```
