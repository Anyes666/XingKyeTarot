# 星钥塔罗 v2.0 爆款 MVP 计划

> **产品定义**: 塔罗式情绪陪伴 App，不是占卜工具。
>
> **一句话**: 抽一张牌，看见你没说出口的心事。
>
> **评分基准**: 8.2/10 → 方向正确，但不是"完整功能"，而是先打穿传播闭环。

---

## 一、爆款公式

```
问心入口 → 扎心金句 → 月见追问 → 分享卡 → 7天情绪星图
```

这个闭环里，每一个节点都是用户**主动打开**的理由。

| 节点 | 用户行为 | 核心指标 |
|------|----------|----------|
| 问心入口 | 30秒内被一句话击中 | 首次爽感时间 |
| 扎心金句 | "她怎么知道我在想什么" | 结果页停留时长 |
| 月见追问 | 继续跟月见对话 | 追问转化率 → 付费转化 |
| 分享卡 | 截图发朋友圈/小红书 | 自然新增 |
| 7天情绪星图 | 连续回来打卡 | D7留存 |

## 三、P0 详细设计

### P0-1：首页「问心」—— 两层追问

**现状**: 冷冰冰的"选择一种占卜方式"网格 + 6 个主题选项

**目标**: 对话式入口，让用户感觉"她在问我问题"

#### 问心流程

```
第一问：你现在，最想被哪件事接住？

┌────────────────────────────────┐
│ 💔 他/她忽冷忽热，我很内耗       │
│ 💕 我想知道这段关系值不值得       │
│ 😮‍💨 我最近工作很累，想逃离        │
│ 🔀 我正在纠结一个选择             │
│ 🫥 我感觉自己能量很低             │
│ 🫂 我只是想被安慰一下             │
└────────────────────────────────┘

↓ 用户选择后 ↓

第二问（针对选择的追问）：
以"关系内耗"为例 →

"这件事最消耗你的地方是？"

┌────────────────────────────────┐
│ 忽冷忽热，没有安全感              │
│ 对方不承诺，我在等一个答案         │
│ 我总是想太多，反复内耗             │
│ 对方不解释，我自己猜               │
└────────────────────────────────┘

↓ 用户选择后 → 自动带入选牌+解读上下文
```

#### 两层追问的价值

- **用户觉得"准"**: 不是随机匹配的，是我选出来的
- **解读更精准**: 同一个牌阵+同一个牌，结合用户两层选择调整解读侧重点
- **月见追问的语境**: 后续追问可以直接引用用户的选择

#### 技术实现

**现有代码改动点** (`Index.ets`):
- 替换 `FeatureGrid()` Builder → 新的 `HeartAskEntry()` Builder
- 第一问 6 个选项（替代现有 4 个 FeatureEntry Grid）
- 选择后弹出第二问覆盖层（3-4 个情绪子选项）
- 将两层选择结果传入 `AppStorage`，供 `DivinationPage` 和 `ResultPage` 使用

**新增存储 Key**:
```typescript
// AppStorage
'heartAskTopic'      // 第一问选择 (如 'love_anxiety')
'heartAskPainPoint'  // 第二问选择 (如 'hot_cold_no_security')
```

**影响 `DivinationPage.ets`**:
- 原来选牌阵+主题 → 现在如果从问心进入，自动推荐最匹配的牌阵
  - "关系内耗" → 推荐关系温度牌阵（3张：我/对方/保护自己）
  - "工作太累" → 推荐职场选择牌阵（可先降级为决策牌阵 3 张，后续扩展 4 张）
  - "纠结选择" → 决策牌阵
  - "能量低/想被安慰" → 单牌快占
- 跳过原来的"选择主题"步骤（`SELECT_TOPIC` phase），直接进入抽牌

#### 文案要求

第一问的选项不要分类名词（"情感关系""学业事业"），要用**场景化情绪描述**：

| 旧文案（冷） | 新文案（热） |
|------------|------------|
| 情感关系 | 💔 他/她忽冷忽热，我很内耗 |
| 学业事业 | 😮‍💨 我最近工作很累，想逃离 |
| 自我成长 | 🫥 我感觉自己能量很低 |
| 两难决策 | 🔀 我正在纠结一个选择 |
| 人际社交 | — （融入关系选项） |
| 通用综合 | 🫂 我只是想被安慰一下 |

#### 数据结构变更

**扩展 `InterpretationData.ets`**：为 22 张大阿卡纳新增结构化字段。

```typescript
// model/TarotCard.ets 新增
export interface OneLineMeaning {
  love_anxiety?: string;       // 关系内耗场景
  work_exhaustion?: string;    // 工作疲惫场景
  choice_dilemma?: string;     // 纠结选择场景
  energy_low?: string;         // 能量低落场景
  comfort_seeking?: string;    // 寻求安慰场景
  default: string;             // 通用
}

export interface StructuredInterpretation {
  oneLineMeanings: OneLineMeaning;   // 一句话牌意（按场景）
  emotionalComfort: string;           // 情绪安抚
  actionSteps: string[];             // 行动建议（3-4条）
  closingWords: string;              // 月见提醒收尾
  realityTask?: string;              // 今日现实任务
}
```

**关键**: `oneLineMeanings` 按问心场景拆分，让同一张牌在不同场景下给出不同的金句。例如：

- 月亮正位 + 关系内耗 → "你不是想太多，你只是还没看清真相。"
- 月亮正位 + 工作疲惫 → "你不是能力不够，你只是在一个不透明的环境里消耗自己。"
- 月亮正位 + 能量低落 → "你不是真的低落，你只是太久没有好好休息。"

这会让用户觉得"她真的在针对我的情况讲话"，而不是泛泛的牌意。

### P0-3：月见追问

**这是最重要的新能力，也是最大付费点。**

#### 为什么是 P0

用户看完结果后最自然的行为不是"退出"，而是继续问。追问 = 留存 + 付费转化。

#### 交互设计

结果页第一屏底部，「追问月见」按钮。点击后：

```
┌─────────────────────────────────┐
│                                 │
│  月见想再问你一句：               │
│                                 │
│  ┌─────────────────────────┐   │
│  │ "你更害怕失去这个人，       │   │
│  │  还是害怕承认自己已经很累了？"│   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │ 😔 害怕失去这个人           │   │
│  ├─────────────────────────┤   │
│  │ 😞 承认自己已经很累         │   │
│  ├─────────────────────────┤   │
│  │ 🤷 两个都有                 │   │
│  └─────────────────────────┘   │
│                                 │
└─────────────────────────────────┘
```

选择后 → 生成二次定向解读（1-2 段文字，不抽新牌），以月见口吻直接回应。

**追问不是重新抽牌**，而是基于已抽牌 + 用户两层问心选择 + 追问选择，用半结构化文案拼接生成更个性化的回应。

#### 追问数据设计

每张牌的每个场景（约 5 个场景 × 22 张大阿卡纳 = 110 个组合），预设 1 个追问问题 + 3 个选项 + 3 条回应文案。

```typescript
interface FollowUpQuestion {
  question: string;                     // 追问问题
  options: Array<{
    label: string;                      // 选项文字
    response: string;                   // 月见回应（200-400字）
  }>;
}
```

**工作量**: 22 张大阿卡纳 × 平均 3 个高频场景 = 66 组追问数据。可先用 AI 生成初稿，人工精修。

#### 商业化钩子

- **免费**: 每次占卜后可追问 1 次
- **轻付费（¥6/月 或 ¥1/次）**: 无限追问 + 追问历史
- **引导**: 第一次追问后弹出 "月见可以陪你一直聊下去"

- **行动指引型**: "今天的任务：不解释、不讨好、只整理真实想法。"

每种牌在不同场景下匹配不同模板风格。

- 首次分享后弹出 "你的情绪星图将在7天后解锁" 引导连续使用

### P0-5：7天情绪星图

#### 为什么先不做完整日记，先做 7 天星图

- 完整日记是工具，7 天星图是目标
- 用户需要的是一个"完成 7 天会获得什么"的期待，而不是一个空白的日记本
- 7 天后的结果页本身就是一个极强的分享素材

#### 流程

```
Day 1: 抽牌 → 结果页 → "保存到日记"（一句话笔记） → "已记录 1/7 天"
Day 2-6: 同上，每次显示进度 2/7、3/7...
Day 7: 抽牌 → "🎉 你的情绪星图已解锁！"
       → 星图可视化 + 综合解读
       → [分享我的星图]
```

│                                 │
│  📊 你的高频主题                  │

│  ██░░░░░░░░ 期待回应  1天       │
│                                 │
│  🌙 你的情绪类型：温柔内耗型        │

#### 存储

继续用 `Preferences`，日记按日期索引。7 天内数据量很小（≤ 7 条），无需迁移存储方案。
