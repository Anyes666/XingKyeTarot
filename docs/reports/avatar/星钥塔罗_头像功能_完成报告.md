### Phase 10-15：编译调错 + 最终验证

编译过程中发现并解决的错误：

1. **错误**：`Property 'size' does not exist on type 'Promise<ImageInfo>'`
   - **原因**：`imageSource.getImageInfo()` 返回 Promise，未使用 await
   - **修复**：添加 `await` 关键字

2. **错误**：`In an '@Entry' decorated component, the 'build' method can have only one root node`
   - **原因**：在 `build()` 方法内部使用了 `let`/`const` 局部变量声明，ArkTS 编译器将其解释为多根节点
   - **修复**：移除所有局部变量声明，将计算表达式直接内联到属性值（如 `(this.screenWidth - this.circleDiameter) / 2`、`136` 替代 `56 + 60 + 20`）

## 关键技术决策

1. **PhotoViewPicker 而非 mediaLibrary**：PhotoViewPicker 基于系统 UI，用户主动选择，不需要申请 `ohos.permission.READ_MEDIA` 权限，满足隐私合规"不申请额外权限"的要求
2. **显示层裁剪而非像素级裁剪**：通过 `borderRadius` + `clip(true)` 实现圆形视觉效果，原始图片完整保存在 `filesDir`，避免复杂的 pixelmap 裁剪逻辑，也保证了图片质量
3. **`router.replaceUrl()` 刷新策略**：由于"我的"页是 @Component 子组件（SettingsTabContent），无法使用 `onPageShow()` 生命周期，采用 replaceUrl 重建页面实现头像刷新
4. **内联表达式替代局部变量**：因 ArkTS 编译器限制 `build()` 方法内不允许 `let`/`const` 声明，所有局部计算改为内联表达式
