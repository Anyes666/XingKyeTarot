# 星钥塔罗 P0-P2 产品需求文档（PRD）

---

## 一、项目信息

| 字段 | 内容 |
|------|------|
| **语言** | 中文 |
| **平台/技术栈** | HarmonyOS + ArkTS |
| **项目代号** | star_key_tarot |
| **当前基线** | P0.1 问心单牌流程已跑通（AskHeartPage → CardDrawPage → ResultGoldenPage → ShareCardPage / FollowUpPage / EmotionStarPage） |
| **文档版本** | v1.0 |
| **日期** | 2026-06-30 |

### 原始需求摘要

在 P0.1 单牌占卜基础上，扩展为完整的塔罗情绪陪伴 App：增加启动门禁（隐私政策）、首次体验引导、首页、圣三角牌阵、关系探索牌阵、历史记录系统，以及 ShareCardPage 兼容三种牌阵。产品底线：塔罗是情绪陪伴而非算命，月见人格温柔克制有边界感，不接 AI、不接会员、不接广告、不接云端同步。

## 二、产品定义

### 2.1 产品目标

| # | 目标 | 衡量标准 |
|---|------|---------|
| G1 | **建立可信赖的情绪陪伴入口**：通过启动门禁与月见人格文案，让用户感知塔罗是"自我探索工具"而非"算命工具" | 新用户同意隐私政策后完成首次体验的比例 ≥ 80% |
| G2 | **覆盖三种核心使用场景**：单牌快占（日常情绪速查）、圣三角（过去·现在·未来反思）、关系探索（我·对方·走向洞察），每种场景独立闭环、可分享 | 三种牌阵均能独立走通完整流程（引导→抽牌→结果→分享） |
| G3 | **沉淀个人情绪轨迹**：所有占卜结果自动保存为历史记录，用户可回顾、可查看详情、可重新反思 | 历史记录最多保存 50 条，支持列表浏览和详情查看 |

### 2.2 用户故事

| # | 用户故事 | 对应功能 |
|---|---------|---------|
| US1 | 作为新用户，我首次打开 App 时希望先了解隐私政策，确认后再进入体验，这样我会感到安全可信 | 启动门禁（隐私政策 + 用户协议） |
| US2 | 作为新用户，我希望第一次使用 App 时能自然地体验一次完整的问心单牌流程，不需要做任何选择或配置 | 首次体验模式（FIRST_EXPERIENCE） |
| US3 | 作为日常用户，我打开 App 后希望看到今日运势卡片，并能快速选择我想要的占卜方式（单牌/圣三角/关系探索） | 首页 HomePage |
| US4 | 作为反思型用户，我希望回顾之前做过的所有占卜，重新审视当时的情绪和牌面解读 | 历史记录系统 |
| US5 | 作为分享型用户，无论我做了哪种牌阵的占卜（单牌/圣三角/关系探索），我都希望生成一张精美的分享卡片保存到相册 | ShareCardPage 兼容三种 spreadType |

## 三、技术规范

### 3.1 需求池（Requirements Pool）

#### P0 — 必须交付（MVP 底线）

| ID | 需求 | 说明 |
|----|------|------|
| P0-01 | **启动门禁** | 新用户首次启动时展示隐私政策（`privacy-policy.md`）和用户协议（`user-agreement.md`），用户必须点击"同意"才能进入 App。拒绝则退出 App。同意状态存储在本地 Preferences |
| P0-02 | **首次体验流程** | 新用户同意隐私后，直接进入问心单牌首次体验（FIRST_EXPERIENCE 模式）：AskHeartPage（引导式输入）→ CardDrawPage → ResultGoldenPage。体验结束后标记完成，后续启动进入首页 |
| P0-03 | **首页 HomePage** | 包含：① "今日一占"运势卡片（每日刷新）；② 四大功能入口（单牌快占 / 圣三角 / 关系探索 / 决策牌阵-即将上线）；③ 历史查询入口。点击入口进入对应流程 |
| P0-04 | **单牌快占（NORMAL_SINGLE 模式）** | 复用现有 AskHeartPage，新增 NORMAL_SINGLE 模式（区别于 FIRST_EXPERIENCE）。流程：AskHeartPage → CardDrawPage → ResultGoldenPage。结束后可选择分享 / 追问 / 情绪星图 / 返回首页 |
| P0-05 | **圣三角牌阵** | 新页面组：TriangleIntroPage（牌阵介绍）→ TriangleDrawPage（依次抽取 3 张牌：过去·现在·未来）→ TriangleResultPage（三牌解读）。支持进入 ShareCardPage |
| P0-06 | **关系探索牌阵** | 新页面组：RelationIntroPage（牌阵介绍）→ RelationDrawPage（依次抽取 3 张牌：我·对方·走向）→ RelationResultPage（三牌解读）。支持进入 ShareCardPage |
| P0-07 | **ShareCardPage 兼容三种 spreadType** | ShareCardPage 根据 `spreadType` 参数（SINGLE / TRIANGLE / RELATION）渲染不同的分享卡布局。保存按钮调用系统相册 API 真实保存图片，不做剪贴板退化 |
| P0-08 | **历史记录自动保存** | 每次占卜完成后自动保存一条记录：包含时间戳、spreadType、问题/意图、各牌位牌面数据、解读摘要。最多保留 50 条，超出后删除最早记录 |
| P0-09 | **历史记录列表页** | HistoryPage：按时间倒序展示历史记录列表，每条显示时间、牌阵类型、首张牌名。支持点击进入详情 |
| P0-10 | **历史记录详情页** | HistoryDetailPage：展示单条历史的完整信息，包括牌阵布局、各牌面解读。底部提供"重新反思"入口（可选跳转至对应牌阵流程） |
| P0-11 | **免责声明** | 所有结果页底部统一展示："结果仅供娱乐与自我探索参考，不构成任何专业建议"。所有分享卡保留此声明 |

#### P1 — 应该交付

| ID | 需求 | 说明 |
|----|------|------|
| P1-01 | **首页今日一占动态刷新** | 每日自动刷新运势卡片（基于本地随机种子），展示当日牌面和一句短解读 |
| P1-02 | **历史记录删除** | 支持在历史列表页左滑删除单条记录 |
| P1-03 | **圣三角 / 关系探索的追问页** | 类似问心单牌的 FollowUpPage，支持对三牌结果进行追问式反思 |
| P1-04 | **情绪星图兼容三种牌阵** | EmotionStarPage 根据 spreadType 展示不同的情绪记录维度 |
| P1-05 | **过渡动画** | 首页 → 各功能入口 → 流程内的页面转场动画，保持流畅体验 |

#### P2 — 可以交付

| ID | 需求 | 说明 |
|----|------|------|
| P2-01 | **决策牌阵** | 四大入口中"即将上线"的决策牌阵：DecisionIntroPage → DecisionDrawPage → DecisionResultPage（具体牌位定义待后续 PRD） |
| P2-02 | **分享卡自定义** | 用户可选择分享卡模板风格（如暗色/亮色） |
| P2-03 | **历史搜索/筛选** | 按牌阵类型筛选或按日期范围筛选历史记录 |

### 3.3 数据流约束

| 约束项 | 说明 |
|--------|------|
| 隐私同意状态 | 存储在本地 Preferences，键名 `privacy_agreed`，布尔值 |
| 首次体验完成标记 | 存储在本地 Preferences，键名 `first_experience_done`，布尔值 |
| 历史记录 | 存储在本地关系型数据库（如 RDB Store），单表 `history`，上限 50 条，FIFO 淘汰 |
| 分享图片 | 调用 HarmonyOS `photoAccessHelper` API，写入系统相册，不读取相册内容 |
