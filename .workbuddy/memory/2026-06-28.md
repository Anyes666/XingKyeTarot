### 关键发现
1. **陪伴感核心 = 拟真时序 + 池随机，无 AI**：`simulateReply()`（core.js 1620-1804）是纯概率剧场——typing 指示 + 延迟 + 已读 + 从 customReplies 池随机抽 + emoji/贴纸点缀。星澜可照搬此手法，离线即可获得强陪伴感。
2. **新手引导 = 三层门禁**：splash 6 屏承诺门禁 + disclaimer 弹窗 + 22 步高亮 tour（tourSteps 数组 {element,title,content,position,onBefore}）。22 步过长不适合星钥塔罗，但数据结构可迁移为 ArkTS interface。
3. **无 LICENSE 文件**，disclaimer 声明"绝对禁止商用"+"支持二改二传"。星钥塔罗已上架 → 只能借鉴方法论，不可抄代码/文案/素材。
4. 含塔罗运势占卜（games.js，周运 1 张 + 日运 3 张 78 片含逆位），框架正是星钥塔罗要避开的"算命承诺"。

### 给星钥塔罗的 3 个稳妥优化方向（按性价比）
1. **方向一（最高性价比）**：星澜点击加"正在输入"+0.8~1.5s 延迟 + 文案池随机。改动最小，收益最大。
2. **方向二**：用对话流替换首次引导（OnboardingFlags + GuideStep[]，每场景 1 句星澜话，可跳过可重看，不做浮层）。
3. **方向三**：星澜文案库结构化升级为 CompanionLine[]（带 scene/tone），为星衡预留 NEUTRAL/STRUCTURED 调性。
