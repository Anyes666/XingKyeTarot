- `TarotCardDetailPage.ets`：全屏预览层从 Column 嵌套改为 Stack 扁平结构（背景层→图片手势层→顶部按钮层），关闭按钮从 Text+onClick 改为 Button 组件，顶部 Row 添加 `hitTestBehavior(HitTestMode.Block)` + `zIndex(200)`，确保按钮始终响应点击
- `DivinationPage.ets` / `DivinationResultPage.ets` / `DivinationHistoryPage.ets`：AppBackButton 添加 `btnZIndex: 50`
**编译结果**：BUILD SUCCESSFUL (21s 344ms)，0 ERROR。

### Phase 6.9 补丁 2：全屏预览关闭按钮彻底修复
**问题**：上一次修复（Stack 扁平结构 + Button onClick）后用户仍反馈左上角叉号点击无响应。根因：Stack 中 Image 占满全屏并绑定 PanGesture，手势层优先于子组件 onClick，按钮的 zIndex 无法真正隔离触摸事件。
**最终方案**：
- 预览层改回 **Column 布局**（顶部栏 + 手势图片区上下分区，不重叠）
- **手势从 Image 移到包裹 Image 的 Stack 上**，手势仅在 `layoutWeight(1)` 的图片区域生效
- **关闭按钮从 Button+onClick 改为 Text+onTouch(TouchType.Down)**：onTouch 是更底层的触摸事件，响应优先级高于手势，确保点击不被吞掉
**编译结果**：BUILD SUCCESSFUL (15s 748ms)，0 ERROR。
