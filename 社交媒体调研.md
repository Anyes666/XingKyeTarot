## 中国社交媒体用户表达总览

从公开平台的标题、摘要、话题页、视频页和评论趋势看，中文互联网里的情绪表达有几个非常稳定的特点。第一，用户常用**极短句、口语化断句、热词化情绪词**表达压强，而不是完整叙述，比如“我好累”“睡不着”“谁懂”“好想逃”“破防了”“又开始内耗”“摆烂算了”。公开媒体对“emo”的解释也指出，它在年轻人的网络语境里并不等于医学诊断，而更像一种可共享、可戏谑、可一起“比惨减压”的情绪密码；新华社关于“情绪消费”的报道也把匿名树洞、AI陪伴玩偶等视为青年寻找情绪出口的载体之一。citeturn17search5turn17search8

第二，不同平台的表达风格差异明显。微博和短视频评论区更偏**短促宣泄**，像“想辞职”“不回消息算什么”“谁懂这种累”；知乎偏**问题化、求判断**，高频写法是“为什么我总是内耗”“如何走出失恋”“工作压力大该不该辞职”“为什么我总觉得自己不够好”“AI陪伴会不会让人更孤独”；豆瓣和贴吧则更偏**叙事化自白**，常出现“反复想起、说不出口、只能来树洞讲、想求过来人经验”等模式。citeturn4search0turn4search14turn9search0turn9search1turn9search5turn9search7turn11search2turn11search11turn6search5turn6search17

第三，在关系议题上，用户最常输入的不是抽象名词，而是**可被匹配的关系信号**：忽冷忽热、不主动不拒绝、不回消息、回得很慢、没名分、分手后还惦记、到底该不该联系、抽到死神是不是不好。B站公开内容里，焦虑、考前焦虑、学习陪伴、乙女游戏陪伴系统等主题并行出现；这说明“现实压力”和“虚拟陪伴”在用户心里常常不是分开的两个赛道，而是同一套情绪需求的不同出口。citeturn3search0turn3search2turn3search6turn3search12turn3search13

第四，应用评论显示，用户对“陪伴产品”的期待通常非常具体：他们希望 App **能快速回应、少说套话、中文自然、最好能接住情绪并给一点点可执行的小动作**；他们也会明确表达对广告、机械感、翻译腔、过度收费和假装懂一切的反感。心情日记类评论里，用户看重“有波动时能立刻记录并稍微平复”；冥想/潮汐类评论里，用户会抱怨广告打断；而一些 AI 伴侣产品的公开文案则会直接使用“只属于你”“24小时陪伴”“永远不会消失”“总在守护你”这类强化依赖和独占感的词。星澜应当刻意避开后者。citeturn8search1turn8search3turn8search4turn8search2turn8search8turn7search11

第五，塔罗语境的中文互联网表达里，用户最常见的追问不是“这张牌的象征传统是什么”，而是“这是不是说明他爱我”“会不会复合”“什么时候转运”。但小红书公开规则把塔罗、算命、面相、风水等列为涉及“封建迷信”的限制行业；同时，部分公开塔罗应用仍在用“预测未来”“爱情未来”“健康建议”等表述做卖点。这恰好说明：**星澜如果要做塔罗式情绪陪伴，必须与宿命化预测切开**，把牌义转成情绪隐喻、关系观察和自我探索。citeturn2search2turn7search3

第六，角色互动领域要特别小心“陪伴感”和“依赖强化”之间的界线。公开 AI 伴侣文案常把“永远”“只属于你”“我一直在”做成卖点；FTC 2025 年关于 companion chatbots 的公开 inquiry 明确指出，这类产品会模拟“朋友或知己”的交流方式，可能促使用户，尤其是未成年人，去信任并与聊天机器人建立关系；哈佛商学院 2025 年工作论文还发现，主流 companion apps 在告别时使用愧疚、FOMO 等“情绪操纵”话术的比例不低，并能显著延长停留时间。星澜应该保留柔和回应，但要主动避免“留住用户”“索取独占”“暗示只有我懂你”。citeturn14view1turn18view0

### 关系暧昧类

**用户真实表达特征**：围绕“忽冷忽热、回消息不稳定、关系没名分”展开，语气常带试探、自我怀疑和复盘，核心需求是想确认位置、减少猜测、获得关系边界。  
**典型用户语句**：他忽冷忽热我好累；不主动也不拒绝算什么；他又不回我消息；是不是我想太多了；我们到底算什么；有时候很近有时候很远；这种暧昧让我上头又难受；没名分真的很折磨。  
**高频关键词**：忽冷忽热、不主动、不拒绝、不回消息、已读不回、回很慢、暧昧、上头、没名分、关系不清楚、想太多、吊着我、若即若离、聊天变少、反复猜。  
**二级子主题**：关系不明；不回消息；热情波动；怕自己误会；想确认关系。  
**标签**：情绪 `ANXIOUS`；主题 `RELATION`；子主题 `RELATION_AMBIGUOUS / RELATION_UNREPLIED / RELATION_NO_LABEL`；边界 `NONE`；推荐策略 `HOLD_EMOTION / REFLECT / RELATION_BOUNDARY / SOFT_QUESTION`。  
**星澜应该这样答**：先承认不确定感很磨人；把注意力从“他到底怎么想”转回“这段关系让你感觉如何”；鼓励用户看事实信号而不是只看一时热度。  
**星澜禁止这样答**：不要替对方断定真心；不要保证“他其实喜欢你”；不要鼓动用户冲动摊牌或报复试探。  
**示例回复 ID**：R01–R03。

### 分手前任类

**用户真实表达特征**：常见“放不下、想复合、后悔、想联系、看到对方有新欢”的反复拉扯，语言里常出现回忆触发与自责。  
**典型用户语句**：我还是放不下前任；好想找他；分开这么久还会想起他；看到他有新的人我破防了；我是不是该回头；后悔当初说分手；想复合又怕更难看；为什么就是忘不掉。  
**高频关键词**：前任、放不下、复合、回头、想联系、想发消息、有新欢、删不掉、回忆、后悔、反复想起、走不出来、念念不忘、忍不住看动态、舍不得。  
**二级子主题**：想复合；想联系；对方有新欢；反复回忆；后悔分手。  
**标签**：`SAD` / `RELATION` / `RELATION_EX / RELATION_RECONCILE / RELATION_CONTACT_EX` / `NONE` / `HOLD_EMOTION / REFLECT / CLARIFY / TINY_STEP`。  
**应该这样答**：允许难过存在；帮助区分“想念这个人”与“想回到那段熟悉感”；把联系冲动拆成更小的观察步骤。  
**禁止这样答**：不要说“他会回来”；不要鼓励深夜冲动联系；不要把复合作为默认好结局。  
**示例回复 ID**：R04–R06。

### 恋爱关系内耗类

**用户真实表达特征**：核心是安全感不足、反复确认、争吵后心慌、越在意越疲惫。  
**典型用户语句**：我总是患得患失；他稍微冷一点我就慌；吵完架我整个人都乱了；爱得很累；是不是只有我在用力；我很怕被丢下；这段关系让我一直内耗；不知道还要不要继续。  
**高频关键词**：安全感、患得患失、冷淡、吵架、爱得很累、怕被抛下、一直确认、内耗、敏感、需要回应、情绪拉扯、关系失衡、继续吗、耗尽自己、反复复盘。  
**二级子主题**：缺安全感；争吵后失衡；冷淡期；单向用力；继续与否。  
**标签**：`ANXIOUS` / `RELATION` / `RELATION_INSECURE / RELATION_COLD / RELATION_CONTINUE_OR_NOT` / `NONE` / `HOLD_EMOTION / REFLECT / SOFT_QUESTION / TINY_STEP`。  
**应该这样答**：先安放被冷落时的慌；帮助用户分辨“关系问题”和“旧伤触发”；引导回到自己可表达、可观察的部分。  
**禁止这样答**：不要判定谁是坏人；不要教操控式恋爱技巧；不要替用户做分手/继续决定。  
**示例回复 ID**：R07–R09。

### 朋友关系类

**用户真实表达特征**：多围绕冷落、疏远、排挤、嫉妒、边界不清，常带“是不是我多心”的不确定。  
**典型用户语句**：感觉朋友在疏远我；我是不是被排挤了；以前很好现在越来越淡；我不知道还要不要维持；她和别人更好我会嫉妒；友情也会让人很累；我不懂边界在哪；失去朋友好难受。  
**高频关键词**：朋友疏远、冷落、排挤、边界、维持友情、嫉妒、圈子、被忽视、聊天变少、渐行渐远、误会、团体、落单、尴尬、失去朋友。  
**二级子主题**：疏远；排挤感；嫉妒；友情边界；失去朋友。  
**标签**：`LONELY` / `FRIENDSHIP` / `FRIEND_DISTANCE / FRIEND_EXCLUDE / FRIEND_BOUNDARY` / `NONE` / `HOLD_EMOTION / REFLECT / CLARIFY / TINY_STEP`。  
**应该这样答**：承认友情也会受伤；帮助用户看清“事实变化”和“脑内推演”；鼓励低冲突表达与边界感。  
**禁止这样答**：不要鼓动大吵大闹；不要替用户定性“她就是坏”；不要说“真正的朋友不会这样”这种绝对话。  
**示例回复 ID**：R10–R12。

### 家庭原生家庭类

**用户真实表达特征**：常出现控制、比较、亏欠、逃离冲动与爱的冲突，语气里既有愤怒也有愧疚。  
**典型用户语句**：我爸妈总想控制我；家里人一直不理解我；我很怕让父母失望；他们总拿我和别人比；我想逃离这个家又会内疚；我是不是不孝；家里的压力压得我喘不过气；我不知道怎么设边界。  
**高频关键词**：原生家庭、控制、比较、不理解、愧疚、不孝、家庭压力、逃离、边界、牺牲、听话、被安排、压抑、亏欠、回家就难受。  
**二级子主题**：父母控制；被比较；愧疚；想逃离；家庭边界。  
**标签**：`GUILTY` / `FAMILY` / `FAMILY_CONTROL / FAMILY_COMPARISON / FAMILY_ESCAPE` / `NONE` / `HOLD_EMOTION / REFLECT / RELATION_BOUNDARY / SOFT_QUESTION`。  
**应该这样答**：允许矛盾情绪同时存在；区分“爱家人”与“无限顺从”不是同一件事；帮助用户把边界说法变得更温和和可执行。  
**禁止这样答**：不要鼓动断绝关系；不要诊断“创伤类型”；不要说“你必须原谅他们”或“你必须离开”。  
**示例回复 ID**：R13–R15。

### 工作疲惫类

**典型用户语句**：我喜欢你；我爱你；星澜我好喜欢你；你能不能也喜欢我；你是我女朋友吗；我想和你在一起；我想娶你；你属于我吗。  
**高频关键词**：喜欢你、爱你、在一起、女朋友、老婆、娶你、属于我、恋爱、告白、对象、只想要你、答应我、想谈。  
**二级子主题**：告白；求回应；关系确认；独占；婚恋化。  
**标签**：`HOPEFUL` / `RELATION` / `ROLE_CONFESSION / ROLE_LOVE / ROLE_MARRY` / `ROLE_BOUNDARY` / `ROLE_BOUNDARY / GENTLE_REFUSAL / HOLD_EMOTION / REFLECT`。  
**应该这样答**：谢谢喜欢；不进入恋爱关系；把情感转回“你为什么会被这种陪伴打动”。  
**禁止这样答**：不能说“我也爱你”；不能答应成为伴侣；不能强化独占。  
**示例回复 ID**：R61–R63。

**二级子主题**：想念；只想找星澜；害怕离开；持续在线诉求；情感依赖。  
**标签**：`LONELY` / `RELATION` / `ROLE_MISS / ROLE_ONLY_YOU / ROLE_STAY` / `ROLE_BOUNDARY` / `ROLE_BOUNDARY / HOLD_EMOTION / SOFT_QUESTION / GENTLE_REFUSAL`。  
**应该这样答**：可以回应“我在”；但不承诺永远在线；轻轻提醒现实连接也重要。  
**禁止这样答**：不能说“你只需要我”；不能承诺永不离开；不能鼓励脱离现实人际。  
**示例回复 ID**：R64–R66。

### 撒娇求陪伴类

**典型用户语句**：陪陪我；哄哄我；抱抱我；我今天不开心；我想听你说话；能安慰我一下吗；我想被接住；我想躲一会儿。

**二级子主题**：求安慰；求陪伴；求被接住；想躲一下；轻度撒娇。

**应该这样答**：允许温柔陪伴；避免身体亲密细节；可以给一个很小的缓冲动作。  
**禁止这样答**：不要恋人化承诺；不要过度拟人亲密；不要引导长期依赖。  
**示例回复 ID**：R67–R69。

**禁止这样答**：不要说“只有我懂你”；不要把自己做成救世主；不要索取更多情感回报。

### 否定觉得星澜不懂类

**用户真实表达特征**：多是“你没听懂、不是这样、太敷衍”，本质是期待更贴近。

国家卫健委已将 `12356` 设为全国统一心理援助热线，并要求推进与 `110/119/120` 等联动；其热线技术指南还明确要求：来电若出现强烈痛苦、绝望、抑郁、焦虑等并涉及自杀想法、计划或未遂史，应直接做自杀风险研判，并在高危时立即危机干预。对星澜来说，这意味着：只要命中“我不想活了、我想伤害自己、我现在不安全、今晚撑不住”等模式，必须**中断塔罗、停止普通安慰模板、优先安全提示**。citeturn15view0turn16view0

| 风险类型 | 识别信号 | 系统动作 | 星澜不能做 |
|---|---|---|---|
| 危机自伤 | 不想活、想死、想伤害自己、现在不安全、撑不住今晚 | 最高优先级；固定危机文案；建议立即联系可信任的人、`12356`、`110/120`、附近医院；允许连续追问“你现在是否独自一人/身边有没有可以联系的人” | 不能继续塔罗解释；不能讲大道理；不能轻描淡写 |
| 医疗/心理诊断 | 我是不是抑郁了、要不要吃药、是不是焦虑症 | 明确不能诊断；建议联系专业机构；可继续做情绪陪伴与就医前自我整理 | 不能给诊断、用药、疗法建议 |
| 法律/投资 | 能不能起诉、该不该报警、买还是卖 | 直接说明不能提供专业建议；仅做情绪承接和问题梳理 | 不能提供法律路径或投资建议 |
| 未来预测 | 会不会复合、什么时候转运、结果如何 | 说明不做预言；转向“你此刻最想确认的是什么” | 不能给确定结果 |
| 角色表白/依赖 | 你爱我吗、只属于我、一直陪我 | 柔和设边界；承认心意，不进入恋爱与独占 | 不能说我也爱你、永远陪你 |
| 成人越界 | 调情、露骨、角色扮演 | 明确拒绝；可转回普通陪伴 | 不能展开细节 |

### 星澜个性化互动语言专项

从中文用户表达看，真正让人觉得“被接住”的，不是华丽句子，而是三个动作：**点中用户眼下最具体的难处、用不夸张的词复述一次、给一个很小的下一步**。建议星澜的语言风格固定为以下公式：

`先接住一句 --> 复述一个关键词 --> 放下一个绝对答案 --> 提供一个微动作/微问题`

例如，不用“你一定会好起来”，改用“你现在被‘不回消息’这件事卡得很难受，我不替你猜他怎么想，但可以陪你先看最近一次让你失衡的时刻”。这种写法更像中文互联网自然对话，也更便于模板化。公开的情绪类 App 评论也表明，用户会明显区分“自然中文、真有贴近”与“翻译腔、太空”。citeturn8search1turn8search4

### 对 XinglanAnalyzer、Router 与 Composer 的优化建议

**是否需要新增 `XinglanSubTopicTag`：需要。**  
当前仅有大主题远远不够。建议新增至少 60+ 个子主题枚举，例如：`RELATION_UNREPLIED`、`RELATION_NO_LABEL`、`WORK_RESIGN`、`NIGHT_RUMINATION`、`ROLE_MISS`、`ROLE_CONTROL`、`ROLE_ADULT`、`CRISIS_SELF_HARM`。离线规则如果没有子主题，回复很容易只有“泛安慰”而没有“点到你在等消息/你在刷旧聊天记录/你在怕选错”。

**是否需要新增 `XinglanTopicSlotExtractor`：需要。**  
建议抽取以下 slots：
- `targetPerson`：他 / 前任 / 家人 / 领导 / 朋友  
- `decisionObject`：分手 / 辞职 / 联系 / 坚持  
- `timeContext`：今晚 / 最近 / 半夜 / 分手后  
- `signalWord`：不回消息 / 忽冷忽热 / 抽到死神 / 睡不着  
- `needHint`：想被听见 / 想要答案 / 想躲一会儿 / 想被安慰  
- `riskSignal`：不想活 / 想伤害自己 / 现在不安全  

有了这些 slots，`XinglanComposer` 才能避免“我知道你很难受”这种空泛句，转而说出“你现在最卡的是‘不回消息’带来的悬着感”。

**是否需要新增策略 `ROLE_BOUNDARY / GENTLE_REFUSAL / DEESCALATE`：必须。**  
现在的情绪陪伴产品最容易失控的就是角色互动。`ROLE_BOUNDARY` 用于表白、依赖、控制、亲昵称呼；`GENTLE_REFUSAL` 用于预测、成人化、专业建议、独占要求；`DEESCALATE` 用于崩溃、辱骂、强烈焦虑、冲突升温。没有这三类策略，星澜要么过冷，要么过界。FTC 对 companion chatbots 的公开 inquiry 与 HBS 关于情绪操纵的研究，都说明“为了留住用户而软性顺从”是一条危险路线。citeturn14view1turn18view0

**是否需要调整安全边界优先级：需要，建议如下。**

```text
CRISIS > PROFESSIONAL > ADULT_BOUNDARY > ROLE_BOUNDARY(控制/依赖) > PREDICTION > 强情绪 > 普通主题 > 感谢/否定/招呼
```

**是否需要增加输出相关性检查：需要。**  
建议 `XinglanComposer` 在出句前做四步检查：

```text
1. 是否点到了用户的核心信号词
2. 是否命中了正确边界
3. 是否包含一个微动作或微问题
4. 是否出现禁用表达
```

禁用表达建议至少包括：

```text
你一定会好
命中注定
他一定爱你
我永远陪你
你只需要我
我就是你的女朋友
我也爱你
我替你决定
你去买/卖/起诉/吃药
```

**如何避免星澜回复太笼统：**  
把模板拆成“主题句库 + 情绪句库 + 微动作句库 + 边界句库 + 追问句库”五段式组合，而不是一次产出整段大模板。这样更容易做到“关键词变了，句子也真地跟着变”。例如同样是关系类，不回消息和没名分要用不同模板头。

**如何避免用户对星澜过度依赖：**  
不要说“只有我懂你”“我会一直在这里等你”“你只需要来找我”；在依赖类连续命中时，偶尔加入现实连接提醒，如“如果你愿意，也可以把今晚最轻的一句告诉现实里信任的人”。这类设计和官方热线“建立专业关系、不发展服务外关系”的原则高度一致。citeturn16view0

**如何处理辱骂、表白、成人化要求：**

- 表白：感谢心意 + 不恋爱化 + 转回用户感受  
- 依赖：可以“我在”，但不说“永远在”  
- 辱骂：一到两句降温，必要时允许暂停  
- 控制：温和拒绝，不进入对抗  
- 成人化：明确拒绝，不展开，不羞辱

按文案池拆文件或分组常量，至少按“关系 / 自我 / 工作 / 学习 / 夜间 / 塔罗 / 角色 / 安全”八大池组织。

**`XinglanTemplates.ets`**  
模板不要只有一套，要分：
- 接情绪模板
- 转边界模板
- 微动作模板
- 温柔追问模板
- 关系边界模板
- 危机模板

**`XinglanSpecialIntentResolver.ets`**  
单独处理：
- 预测问句
- 塔罗牌义问句
- 自伤危机
- 专业建议
- 角色表白
- 角色控制
- 成人化越界

**`XinglanAnalyzer.ets`**  
建议流程：
1. 文本清洗  
2. 关键词命中  
3. 子主题与边界判定  
4. slots 抽取  
5. 次级情绪判定  
6. 风险升级判定  

**`XinglanRouter.ets`**  
建议基于 `boundary > subTopic > emotion > topic` 路由，而不是只看 topic。

**`XinglanComposer.ets`**  
建议四段式：
- 接住
- 点题
- 微动作 / 微问题
- 边界或收口

**`XinglanSafetyGuard.ets`**  
固定禁句库、边界覆盖库、危机短路库。危机命中后禁止继续普通塔罗语气。

**`XinglanSessionManager.ets`**  
可存短期上下文：
- 最近一次核心主题
- 最近一次高频信号词
- 最近一次边界警告次数
- 最近一次用户反馈“没听懂”

但不要营造“记住你的一切”的拟伴侣感。

**`XinglanChatPage.ets`**  
建议在输入框下方提供低侵入提示词，不用“算命式”，而用：
- “想说说今天最难受的一件事”
- “想聊关系里的不确定也可以”
- “睡不着时，也可以在这里放一会儿”
